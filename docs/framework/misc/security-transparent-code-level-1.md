## <a name="the-level-1-transparency-model"></a><span data-ttu-id="a785a-114">Düzey 1 saydamlık modeli</span><span class="sxs-lookup"><span data-stu-id="a785a-114">The Level 1 Transparency Model</span></span>  
 <span data-ttu-id="a785a-115">Düzey 1 saydamlık kullandığınızda, güvenlik açısından saydam güvenlik-güvenli-kritik ve kritik güvenlik yöntemleri koddan ayıran bir güvenlik modeli kullandığınız.</span><span class="sxs-lookup"><span data-stu-id="a785a-115">When you use Level 1 transparency, you are using a security model that separates code into security-transparent, security-safe-critical, and security-critical methods.</span></span>  
  
 <span data-ttu-id="a785a-116">Tüm derleme, bir derlemede bazı sınıflar veya bazı yöntemler bir sınıftaki güvenlik açısından saydam olarak işaretleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="a785a-116">You can mark a whole assembly, some classes in an assembly, or some methods in a class as security-transparent.</span></span> <span data-ttu-id="a785a-117">Güvenliği saydam kod ayrıcalıklarını yükseltme yapamazsınız.</span><span class="sxs-lookup"><span data-stu-id="a785a-117">Security-transparent code cannot elevate privileges.</span></span> <span data-ttu-id="a785a-118">Bu kısıtlama, üç sonuçları vardır:</span><span class="sxs-lookup"><span data-stu-id="a785a-118">This restriction has three consequences:</span></span>

 <span data-ttu-id="a785a-128">Güvenlik açısından kritik güvenliği saydam tersidir.</span><span class="sxs-lookup"><span data-stu-id="a785a-128">Security-critical is the opposite of security-transparent.</span></span> <span data-ttu-id="a785a-129">Güvenlik açısından kritik kod tam güven ile yürütülür ve tüm ayrıcalıklı işlemleri gerçekleştirebilir.</span><span class="sxs-lookup"><span data-stu-id="a785a-129">Security-critical code executes with full trust and can perform all privileged operations.</span></span> <span data-ttu-id="a785a-130">Güvenlik-güvenli-kritik kodu erişim iznine sahip değil kaynakları kullanmak, kısmen güvenilmeyen çağrıcılara izin vermeyecek onaylamak için kapsamlı güvenlik denetim alınıp ayrıcalıklı kodudur.</span><span class="sxs-lookup"><span data-stu-id="a785a-130">Security-safe-critical code is privileged code that has been through an extensive security audit to confirm that it does not allow partially trusted callers to use resources they do not have permission to access.</span></span>

> [!IMPORTANT]
>  <span data-ttu-id="a785a-133">Düzey 1 saydamlık, derlemenin kapsamını sınırlıdır; derlemeler arasında zorlanmaz.</span><span class="sxs-lookup"><span data-stu-id="a785a-133">Level 1 transparency is limited to assembly scope; it is not enforced between assemblies.</span></span> <span data-ttu-id="a785a-134">Düzey 1 saydamlık, güvenlik denetimi amacıyla Microsoft'ta öncelikli olarak kullanıldı.</span><span class="sxs-lookup"><span data-stu-id="a785a-134">Level 1 transparency was primarily used within Microsoft for security audit purposes.</span></span> <span data-ttu-id="a785a-135">Kritik güvenlik türleri ve üyeleri bir düzey 1 derleme içinde diğer derlemelerdeki güvenlik saydam kodu tarafından erişilebilir.</span><span class="sxs-lookup"><span data-stu-id="a785a-135">Security-critical types and members within a level 1 assembly can be accessed by security-transparent code in other assemblies.</span></span> <span data-ttu-id="a785a-136">Tüm düzey 1 kritik güvenlik türleri ve üyeleri tam güven için bağlantı isteklerini gerçekleştirmek önemlidir.</span><span class="sxs-lookup"><span data-stu-id="a785a-136">It is important that you perform link demands for full trust in all your level 1 security-critical types and members.</span></span> <span data-ttu-id="a785a-137">Ayrıca güvenlik-güvenli-kritik türler ve üyeler çağıranlar tür veya üye tarafından erişilen korumalı kaynaklara için izinlere sahip olduğunuzu onaylamanız gerekir.</span><span class="sxs-lookup"><span data-stu-id="a785a-137">Security-safe-critical types and members must also confirm that callers have permissions for protected resources that are accessed by the type or member.</span></span>

## <a name="security-transparency-examples"></a><span data-ttu-id="a785a-181">Güvenlik saydamlık örnekleri</span><span class="sxs-lookup"><span data-stu-id="a785a-181">Security Transparency Examples</span></span>  
 <span data-ttu-id="a785a-182">.NET Framework 2.0 saydamlık kuralları (düzey 1 saydamlık) kullanmak için aşağıdaki derleme ek açıklama kullanın:</span><span class="sxs-lookup"><span data-stu-id="a785a-182">To use the .NET Framework 2.0 transparency rules (level 1 transparency), use the following assembly annotation:</span></span>  
  
```  
[assembly: SecurityRules(SecurityRuleSet.Level1)]  
```  
  
 <span data-ttu-id="a785a-183">Tüm derleme derleme herhangi bir kritik kod içermiyor ve herhangi bir şekilde ayrıcalıklarını yükseltme değil belirtmek için saydam hale getirmek isterseniz, saydamlık derleme şu özniteliği ile açıkça ekleyebilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="a785a-183">If you want to make a whole assembly transparent to indicate that the assembly does not contain any critical code and does not elevate privileges in any way, you can explicitly add transparency to the assembly with the following attribute:</span></span>  
  
```  
[assembly: SecurityTransparent]  
```  
  
 <span data-ttu-id="a785a-184">Kritik ve saydam kodu aynı bütünleştirilmiş kodun karışımı istiyorsanız, derleme ile işaretleyerek başlayın <xref:System.Security.SecurityCriticalAttribute> derleme gibi kritik kod içerebilir belirtmek için özniteliği:</span><span class="sxs-lookup"><span data-stu-id="a785a-184">If you want to mix critical and transparent code in the same assembly, start by marking the assembly with the <xref:System.Security.SecurityCriticalAttribute> attribute to indicate that the assembly can contain critical code, as follows:</span></span>  
  
```  
[assembly: SecurityCritical]  
```  
  
 <span data-ttu-id="a785a-185">Güvenlik açısından kritik eylemleri gerçekleştirmek istiyorsanız, başka bir kritik eylemi gerçekleştiren kod açıkça işaretlemelisiniz <xref:System.Security.SecurityCriticalAttribute> aşağıdaki kod örneğinde gösterildiği gibi öznitelik:</span><span class="sxs-lookup"><span data-stu-id="a785a-185">If you want to perform security-critical actions, you must explicitly mark the code that will perform the critical action with another <xref:System.Security.SecurityCriticalAttribute> attribute, as shown in the following code example:</span></span>

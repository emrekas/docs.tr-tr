# <a name="entity-set"></a><span data-ttu-id="e8959-102">Varlık kümesi</span><span class="sxs-lookup"><span data-stu-id="e8959-102">entity set</span></span>
<span data-ttu-id="e8959-103">Bir *varlık kümesi* için mantıksal bir kapsayıcıdır örneklerini bir [varlık türü](../../../../docs/framework/data/adonet/entity-type.md) ve bu varlık türünden türetilmiş herhangi bir tür örneği.</span><span class="sxs-lookup"><span data-stu-id="e8959-103">An *entity set* is a logical container for instances of an [entity type](../../../../docs/framework/data/adonet/entity-type.md) and instances of any type derived from that entity type.</span></span> <span data-ttu-id="e8959-104">(Türetilen türleri hakkında daha fazla bilgi için bkz. [varlık veri modeli: Devralma](../../../../docs/framework/data/adonet/entity-data-model-inheritance.md).) Bir varlık türü ve bir varlık kümesi arasındaki ilişki, satır ve ilişkisel bir veritabanındaki bir tablo arasındaki ilişkiyi benzerdir: Bir satır gibi bir varlık türü, veri yapısını açıklar ve bir tablo gibi belirli bir yapının örneği bir varlık kümesini içerir.</span><span class="sxs-lookup"><span data-stu-id="e8959-104">(For information about derived types, see [Entity Data Model: Inheritance](../../../../docs/framework/data/adonet/entity-data-model-inheritance.md).) The relationship between an entity type and an entity set is analogous to the relationship between a row and a table in a relational database: Like a row, an entity type describes data structure, and, like a table, an entity set contains instances of a given structure.</span></span> <span data-ttu-id="e8959-105">Bir varlık kümesindeki bir veri yapısı modelleme değil; verilerin yapısını açıklamaz.</span><span class="sxs-lookup"><span data-stu-id="e8959-105">An entity set is not a data modeling construct; it does not describe the structure of data.</span></span> <span data-ttu-id="e8959-106">Bir veri deposuna eşlenebilir bunun yerine, bir varlık kümesindeki bir yapısı (örneğin, ortak dil çalışma zamanı veya bir SQL Server veritabanı) barındıran veya depolama ortamı için Grup varlık türü örneklerine sağlar.</span><span class="sxs-lookup"><span data-stu-id="e8959-106">Instead, an entity set provides a construct for a hosting or storage environment (such as the common language runtime or a SQL Server database) to group entity type instances so that they can be mapped to a data store.</span></span>  
  
 <span data-ttu-id="e8959-107">Bir varlık kümesi içinde tanımlanan bir [varlık kapsayıcısı](../../../../docs/framework/data/adonet/entity-container.md), varlık kümelerini mantıksal bir gruplandırması olan ve [ilişki Setleri](../../../../docs/framework/data/adonet/association-set.md).</span><span class="sxs-lookup"><span data-stu-id="e8959-107">An entity set is defined within an [entity container](../../../../docs/framework/data/adonet/entity-container.md), which is a logical grouping of entity sets and [association sets](../../../../docs/framework/data/adonet/association-set.md).</span></span>  
  
 <span data-ttu-id="e8959-108">Bir varlık kümesinde var varlık türü için bir örneği, aşağıdakilerin doğru olması gerekir:</span><span class="sxs-lookup"><span data-stu-id="e8959-108">For an entity type instance to exist in an entity set, the following must be true:</span></span>  
  
-   <span data-ttu-id="e8959-109">Örnek türünün ya da'varlık kümesini alır veya varlık türünün bir alt türdür örneğinin varlık türü olarak aynıdır.</span><span class="sxs-lookup"><span data-stu-id="e8959-109">The type of the instance is either the same as the entity type on which the entity set is based, or the type of the instance is a subtype of the entity type.</span></span>  
  
-   <span data-ttu-id="e8959-110">[Varlık anahtarı](../../../../docs/framework/data/adonet/entity-key.md) için varlık kümesi içinde benzersiz bir örneğidir.</span><span class="sxs-lookup"><span data-stu-id="e8959-110">The [entity key](../../../../docs/framework/data/adonet/entity-key.md) for the instance is unique within the entity set.</span></span>  
  
-   <span data-ttu-id="e8959-111">Örneği başka bir varlık kümesinde yok.</span><span class="sxs-lookup"><span data-stu-id="e8959-111">The instance does not exist in any other entity set.</span></span>  
  
    > [!NOTE]
    >  <span data-ttu-id="e8959-112">Birden çok varlık kümeleri aynı varlık türü kullanarak tanımlanabilir, ancak belirtilen varlık türünün bir örneği yalnızca bir varlık kümesinde var olabilir.</span><span class="sxs-lookup"><span data-stu-id="e8959-112">Multiple entity sets can be defined using the same entity type, but an instance of a given entity type can only exist in one entity set.</span></span>  
  
 <span data-ttu-id="e8959-113">Kavramsal modelde her varlık türü için bir varlık tanımlamak zorunda değildir.</span><span class="sxs-lookup"><span data-stu-id="e8959-113">You do not have to define an entity set for each entity type in a conceptual model.</span></span>

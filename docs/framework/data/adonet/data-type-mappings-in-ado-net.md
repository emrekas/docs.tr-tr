 <span data-ttu-id="48c8f-109">Benzer şekilde, bir `DataReader` sonuç değerini bir veri kaynağından alınan bir değer, .NET Framework türü olan bir yerel değişkende depolanan döndürür.</span><span class="sxs-lookup"><span data-stu-id="48c8f-109">Likewise, when a `DataReader` returns a value from a data source, the resulting value is stored in a local variable that has a .NET Framework type.</span></span> <span data-ttu-id="48c8f-110">Her ikisi için de `Fill` işlemlerini `DataAdapter` ve `Get` yöntemlerinin `DataReader`, .NET Framework türü .NET Framework veri sağlayıcısı tarafından döndürülen değer gösterilir.</span><span class="sxs-lookup"><span data-stu-id="48c8f-110">For both the `Fill` operations of the `DataAdapter` and the `Get` methods of the `DataReader`, the .NET Framework type is inferred from the value returned from the .NET Framework data provider.</span></span>  
  
 <span data-ttu-id="48c8f-111">Çıkarsanan veri türüne işlemine güvenmek yerine, türü belirlenmiş erişimci yöntemlerini kullanabilirsiniz `DataReader` belirli döndürülen değerin türü bildiğinizde.</span><span class="sxs-lookup"><span data-stu-id="48c8f-111">Instead of relying on the inferred data type, you can use the typed accessor methods of the `DataReader` when you know the specific type of the value being returned.</span></span> <span data-ttu-id="48c8f-112">Türü belirtilmiş erişimci metotlarını ek tür dönüştürme ihtiyacını ortadan kaldırır belirli bir .NET Framework türü bir değer döndürerek daha iyi performans verir.</span><span class="sxs-lookup"><span data-stu-id="48c8f-112">Typed accessor methods give you better performance by returning a value as a specific .NET Framework type, which eliminates the need for additional type conversion.</span></span>

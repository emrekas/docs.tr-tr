## <a name="connection-string-syntax"></a><span data-ttu-id="235b8-108">Bağlantı dizesi söz dizimi</span><span class="sxs-lookup"><span data-stu-id="235b8-108">Connection string syntax</span></span>

<span data-ttu-id="235b8-109">Bir bağlantı dizesi anahtar/değer parametresi çiftleri noktalı virgülle ayrılmış bir listesi verilmiştir:</span><span class="sxs-lookup"><span data-stu-id="235b8-109">A connection string is a semicolon-delimited list of key/value parameter pairs:</span></span>
  
    keyword1=value; keyword2=value;
  
<span data-ttu-id="235b8-110">Anahtar sözcükler, büyük küçük harfe duyarlı değildir.</span><span class="sxs-lookup"><span data-stu-id="235b8-110">Keywords are not case-sensitive.</span></span> <span data-ttu-id="235b8-111">Değerleri, ancak veri kaynağına bağlı olarak duyarlı olabilir.</span><span class="sxs-lookup"><span data-stu-id="235b8-111">Values, however, may be case-sensitive, depending on the data source.</span></span> <span data-ttu-id="235b8-112">Hem anahtar hem de değerleri içerebilir [boşluk karakterleri](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span><span class="sxs-lookup"><span data-stu-id="235b8-112">Both keywords and values may contain [whitespace characters](https://en.wikipedia.org/wiki/Whitespace_character#Unicode).</span></span> <span data-ttu-id="235b8-113">Baştaki ve sondaki boşluk anahtar sözcükleri yok sayıldı ve tırnak işareti olmayan değerler.</span><span class="sxs-lookup"><span data-stu-id="235b8-113">Leading and trailing white space is ignored in keywords and unquoted values.</span></span>

<span data-ttu-id="235b8-114">Bir değer virgül içeriyorsa [Unicode denetim karakterlerini](https://en.wikipedia.org/wiki/Unicode_control_characters), veya baştaki veya sondaki boşlukları, bunu tek veya çift tırnak içine alınmalıdır.</span><span class="sxs-lookup"><span data-stu-id="235b8-114">If a value contains the semicolon, [Unicode control characters](https://en.wikipedia.org/wiki/Unicode_control_characters), or leading or trailing white space, it must be enclosed in single or double quotation marks.</span></span> <span data-ttu-id="235b8-115">Örneğin:</span><span class="sxs-lookup"><span data-stu-id="235b8-115">For example:</span></span>

    Keyword=" whitespace  ";
    Keyword='special;character';

<span data-ttu-id="235b8-116">Kapsayan karakter kendisini kapsayan değeri içinde gerçekleşmeyebilir.</span><span class="sxs-lookup"><span data-stu-id="235b8-116">The enclosing character may not occur within the value it encloses.</span></span> <span data-ttu-id="235b8-117">Bu nedenle, tek tırnak işareti içeren bir değeri yalnızca çift tırnak işareti bulunan ve tersi içine alınabilir:</span><span class="sxs-lookup"><span data-stu-id="235b8-117">Therefore, a value containing single quotation marks can be enclosed only in double quotation marks, and vice versa:</span></span>

    Keyword='double"quotation;mark';
    Keyword="single'quotation;mark";

<span data-ttu-id="235b8-118">Aşağıdaki bağlantı dizelerinin geçerli olduğu şekilde eşittir işareti yanı sıra, tırnak işaretleri kaçış, gerek yoktur:</span><span class="sxs-lookup"><span data-stu-id="235b8-118">The quotation marks themselves, as well as the equals sign, do not require escaping, so the following connection strings are valid:</span></span>

    Keyword=no "escaping" 'required';
    Keyword=a=b=c

<span data-ttu-id="235b8-119">Sonraki noktalı virgül veya dizenin sonuna kadar her değer okunur olduğundan, ikinci örnek değer `a=b=c`, ve son noktalı virgül isteğe bağlıdır.</span><span class="sxs-lookup"><span data-stu-id="235b8-119">Since each value is read till the next semicolon or the end of string, the value in the latter example is `a=b=c`, and the final semicolon is optional.</span></span>

<span data-ttu-id="235b8-120">Tüm bağlantı dizeleri, yukarıda açıklanan aynı temel sözdizimi paylaşın.</span><span class="sxs-lookup"><span data-stu-id="235b8-120">All connection strings share the same basic syntax described above.</span></span> <span data-ttu-id="235b8-121">Tanınan bir anahtar kümesini sağlayıcısında ancak bağlıdır ve önceki API'leri yıl boyunca gelişmiştir *ODBC*.</span><span class="sxs-lookup"><span data-stu-id="235b8-121">The set of recognized keywords depends on the provider, however, and has evolved over the years from earlier APIs such as *ODBC*.</span></span> <span data-ttu-id="235b8-122">*.NET Framework* için veri sağlayıcısı *SQL Server* (`SqlClient`) daha eski API'lar, birçok sözcüklerden destekler ancak genellikle daha esnek ve eş anlamlılar pek çok ortak bağlantı dizesini kabul eder. anahtar sözcükler.</span><span class="sxs-lookup"><span data-stu-id="235b8-122">The *.NET Framework* data provider for *SQL Server* (`SqlClient`) supports many keywords from older APIs, but is generally more flexible and accepts synonyms for many of the common connection string keywords.</span></span>

<span data-ttu-id="235b8-123">Yazım hatalarının hatalara neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="235b8-123">Typing mistakes can cause errors.</span></span> <span data-ttu-id="235b8-124">Örneğin, `Integrated Security=true` geçerlidir, ancak `IntegratedSecurity=true` bir hataya neden olur.</span><span class="sxs-lookup"><span data-stu-id="235b8-124">For example, `Integrated Security=true` is valid, but `IntegratedSecurity=true` causes an error.</span></span>

<span data-ttu-id="235b8-125">Bağlantı dizeleri çalışma anında doğrulanmamış kullanıcı girişini el ile oluşturulmuş dize enjeksiyon saldırılarına karşı savunmasız ve veri kaynağındaki güvenliği tehlikeye atabilir.</span><span class="sxs-lookup"><span data-stu-id="235b8-125">Connection strings constructed manually at run time from unvalidated user input are vulnerable to string-injection attacks and jeopardize security at the data source.</span></span> <span data-ttu-id="235b8-126">Bu sorunları gidermeye yönelik *ADO.NET* 2.0 kullanılmaya [bağlantı dizesi oluşturucular](../../../../docs/framework/data/adonet/connection-string-builders.md) her *.NET Framework* veri sağlayıcısı.</span><span class="sxs-lookup"><span data-stu-id="235b8-126">To address these problems, *ADO.NET* 2.0 introduced [connection string builders](../../../../docs/framework/data/adonet/connection-string-builders.md) for each *.NET Framework* data provider.</span></span> <span data-ttu-id="235b8-127">Bu bağlantı dizesi oluşturucular parametreleri kesin türü belirtilmiş bir özellik olarak kullanıma sunmak ve veri kaynağına göndermeden önce bağlantı dizesini doğrulamak mümkün kılar.</span><span class="sxs-lookup"><span data-stu-id="235b8-127">These connection string builders expose parameters as strongly-typed properties, and make it possible to validate the connection string before it's sent to the data source.</span></span>

<span data-ttu-id="10614-103">ADO.NET veri kaynağına OLE DB ve ODBC kullanıma sunulan ve SQL Server ve XML gibi veri kaynaklarına tutarlı erişimi sağlar.</span><span class="sxs-lookup"><span data-stu-id="10614-103">ADO.NET provides consistent access to data sources such as SQL Server and XML, and to data sources exposed through OLE DB and ODBC.</span></span> <span data-ttu-id="10614-104">Veri paylaşımı tüketici uygulamaları, ADO.NET, bu veri kaynağına bağlanın ve almak, işlemek ve içerdikleri verileri güncelleştirmek için kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="10614-104">Data-sharing consumer applications can use ADO.NET to connect to these data sources and retrieve, handle, and update the data that they contain.</span></span>

 <span data-ttu-id="10614-109">ADO.NET sınıflarını System.Data.dll içinde bulunur ve System.Xml.dll içinde bulunan XML sınıfları ile tümleştirilir.</span><span class="sxs-lookup"><span data-stu-id="10614-109">The ADO.NET classes are found in System.Data.dll, and are integrated with the XML classes found in System.Xml.dll.</span></span> <span data-ttu-id="10614-110">Örnek kod, bir veritabanına bağlanan verileri buradan alır ve ardından bu verileri bir konsol penceresinde görüntüler için bkz: [ADO.NET kod örnekleri](../../../../docs/framework/data/adonet/ado-net-code-examples.md).</span><span class="sxs-lookup"><span data-stu-id="10614-110">For sample code that connects to a database, retrieves data from it, and then displays that data in a console window, see [ADO.NET Code Examples](../../../../docs/framework/data/adonet/ado-net-code-examples.md).</span></span>  
  
 <span data-ttu-id="10614-111">ADO.NET, yönetilen kodun yerel Bileşen Nesne Modeli (COM) geliştiricileri için ActiveX Data Objects (ADO) tarafından sağlanan işlevselliği için benzer yazma geliştiriciler için işlevsellik sağlar.</span><span class="sxs-lookup"><span data-stu-id="10614-111">ADO.NET provides functionality to developers who write managed code similar to the functionality provided to native component object model (COM) developers by ActiveX Data Objects (ADO).</span></span> <span data-ttu-id="10614-112">ADO.NET, değil ADO .NET uygulamalarınıza veri erişimi için kullanmanızı öneririz.</span><span class="sxs-lookup"><span data-stu-id="10614-112">We recommend that you use ADO.NET, not ADO, for accessing data in your .NET applications.</span></span>  
  
 <span data-ttu-id="10614-113">ADO.NET, .NET Framework içindeki veri erişiminin en dolaysız yöntem sağlar.</span><span class="sxs-lookup"><span data-stu-id="10614-113">ADO.NET provides the most direct method of data access within the .NET Framework.</span></span> <span data-ttu-id="10614-114">Temel alınan depolama modelinin yerine kavramsal bir modeli karşı çalışmaya uygulamalarına izin veren bir daha yüksek düzeyde soyutlama için bkz. [ADO.NET Entity Framework](../../../../docs/framework/data/adonet/ef/index.md).</span><span class="sxs-lookup"><span data-stu-id="10614-114">For a higher-level abstraction that allows applications to work against a conceptual model instead of the underlying storage model, see the [ADO.NET Entity Framework](../../../../docs/framework/data/adonet/ef/index.md).</span></span>  
  
 <span data-ttu-id="10614-115">**Gizlilik bildirimi**: Bir kullanıcının arasında System.Data.dll, System.Data.Design.dll, System.Data.OracleClient.dll, System.Data.SqlXml.dll, System.Data.Linq.dll, System.Data.SqlServerCe.dll'ye ve System.Data.DataSetExtensions.dll derlemeleri ayırt etmez özel veriler ve özel olmayan veri.</span><span class="sxs-lookup"><span data-stu-id="10614-115">**Privacy Statement**: The System.Data.dll, System.Data.Design.dll, System.Data.OracleClient.dll, System.Data.SqlXml.dll, System.Data.Linq.dll, System.Data.SqlServerCe.dll, and System.Data.DataSetExtensions.dll assemblies do not distinguish between a user's private data and non-private data.</span></span>  <span data-ttu-id="10614-116">Bu derlemeler değil toplamak, depolamak veya herhangi bir kullanıcının özel veri taşıma.</span><span class="sxs-lookup"><span data-stu-id="10614-116">These assemblies do not collect, store, or transport any user's private data.</span></span> <span data-ttu-id="10614-117">Ancak, üçüncü taraf uygulamalar toplamak, depolamak veya bu bütünleştirilmiş kodları kullanan bir kullanıcının özel veri taşıma.</span><span class="sxs-lookup"><span data-stu-id="10614-117">However, third-party applications might collect, store, or transport a user's private data using these assemblies.</span></span>

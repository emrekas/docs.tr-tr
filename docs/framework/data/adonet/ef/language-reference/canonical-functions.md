# <a name="canonical-functions"></a><span data-ttu-id="23778-102">Kurallı İşlevler</span><span class="sxs-lookup"><span data-stu-id="23778-102">Canonical Functions</span></span>
<span data-ttu-id="23778-103">Bu bölümde, tüm veri sağlayıcıları tarafından desteklenir ve tüm sorgulanırken teknolojiler tarafından kullanılan kurallı işlevler açıklanmaktadır.</span><span class="sxs-lookup"><span data-stu-id="23778-103">This section discusses canonical functions that are supported by all data providers, and can be used by all querying technologies.</span></span> <span data-ttu-id="23778-104">Kurallı işlevler bir sağlayıcı tarafından genişletilemez.</span><span class="sxs-lookup"><span data-stu-id="23778-104">Canonical functions cannot be extended by a provider.</span></span>  
  
 <span data-ttu-id="23778-105">Bu kurallı işlevler sağlayıcı için karşılık gelen bir veri kaynağı işlevi çevrilir.</span><span class="sxs-lookup"><span data-stu-id="23778-105">These canonical functions will be translated to the corresponding data source functionality for the provider.</span></span> <span data-ttu-id="23778-106">Bu işlev çağrılarını veri kaynaklarında yaygın biçiminde ifade sağlar.</span><span class="sxs-lookup"><span data-stu-id="23778-106">This allows for function invocations expressed in a common form across data sources.</span></span>  
  
 <span data-ttu-id="23778-107">Bu kurallı İşlevler, veri kaynaklarından bağımsız olduğundan, kurallı işlevler bağımsız değişkeni ve dönüş türleri kavramsal model türleri açısından tanımlanır.</span><span class="sxs-lookup"><span data-stu-id="23778-107">Because these canonical functions are independent of data sources, argument and return types of canonical functions are defined in terms of types in the conceptual model.</span></span> <span data-ttu-id="23778-108">Ancak, bazı veri kaynakları, kavramsal modelde tüm türleri desteklemeyebilir.</span><span class="sxs-lookup"><span data-stu-id="23778-108">However, some data sources might not support all types in the conceptual model.</span></span>  
  
 <span data-ttu-id="23778-109">Ne zaman kurallı işlevler kullanıldığı bir [!INCLUDE[esql](../../../../../../includes/esql-md.md)] sorgu uygun işlevi veri kaynağında çağrılır.</span><span class="sxs-lookup"><span data-stu-id="23778-109">When canonical functions are used in an [!INCLUDE[esql](../../../../../../includes/esql-md.md)] query, the appropriate function will be called at the data source.</span></span>  
  
 <span data-ttu-id="23778-110">Tüm kurallı İşlevler, hem null giriş davranışı hem de açıkça belirtilen hata koşulları vardır.</span><span class="sxs-lookup"><span data-stu-id="23778-110">All canonical functions have both null-input behavior and error conditions explicitly specified.</span></span> <span data-ttu-id="23778-111">Store sağlayıcıları bu davranışı ile uyumlu olması gerekir ancak [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] bu davranışı zorlamaz.</span><span class="sxs-lookup"><span data-stu-id="23778-111">Store providers should comply with that behavior, but [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] does not enforce this behavior.</span></span>  
  
 <span data-ttu-id="23778-112">LINQ senaryoları için karşı sorgular [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] temel alınan veri kaynağına yöntemlere eşleme CLR yöntemlerini içerir.</span><span class="sxs-lookup"><span data-stu-id="23778-112">For LINQ scenarios, queries against the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] involve mapping CLR methods to methods in the underlying data source.</span></span> <span data-ttu-id="23778-113">Kurallı İşlevler, CLR yöntemleri eşlemeye yöntemlerini belirli bir oluşturmak doğru harita, veri kaynağı ne olursa olsun.</span><span class="sxs-lookup"><span data-stu-id="23778-113">The CLR methods map to canonical functions, so that a specific set of methods will correctly map, regardless of the data source.</span></span>  
  
## <a name="canonical-functions-namespace"></a><span data-ttu-id="23778-114">Namespace kurallı İşlevler</span><span class="sxs-lookup"><span data-stu-id="23778-114">Canonical Functions Namespace</span></span>  
 <span data-ttu-id="23778-115">Ad alanı'kurallı işlevinin <xref:System.Data.Metadata.Edm>.</span><span class="sxs-lookup"><span data-stu-id="23778-115">The namespace for canonical function is <xref:System.Data.Metadata.Edm>.</span></span> <span data-ttu-id="23778-116"><xref:System.Data.Metadata.Edm> Ad alanı, tüm sorguları otomatik olarak eklenir.</span><span class="sxs-lookup"><span data-stu-id="23778-116">The <xref:System.Data.Metadata.Edm> namespace is automatically included in all queries.</span></span> <span data-ttu-id="23778-117">Ancak, başka bir ad alanı içe aktarılırsa kurallı işlev ile aynı ada sahip bir işlev içeren (içinde <xref:System.Data.Metadata.Edm> ad alanı), ad alanı belirtilmelidir.</span><span class="sxs-lookup"><span data-stu-id="23778-117">However, if another namespace is imported that contains a function with the same name as a canonical function (in the <xref:System.Data.Metadata.Edm> namespace), you must specify the namespace.</span></span>

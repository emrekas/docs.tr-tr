# <a name="query-results"></a><span data-ttu-id="3fd58-102">Sorgu Sonuçları</span><span class="sxs-lookup"><span data-stu-id="3fd58-102">Query Results</span></span>
<span data-ttu-id="3fd58-103">Bir LINQ to Entities sorgusu komut ağaçlarına dönüştürüldükten ve yürütüldükten sonra sorgu sonuçları genellikle aşağıdakilerden biri olarak döndürülür:</span><span class="sxs-lookup"><span data-stu-id="3fd58-103">After a LINQ to Entities query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>  
  
- <span data-ttu-id="3fd58-104">Sıfır veya daha fazla yazılmış varlık nesnesi koleksiyonu veya kavramsal modeldeki karmaşık türlerin bir projeksiyonu.</span><span class="sxs-lookup"><span data-stu-id="3fd58-104">A collection of zero or more typed entity objects or a projection of complex types in the conceptual model.</span></span>  
  
- <span data-ttu-id="3fd58-105">Kavramsal model tarafından desteklenen CLR türleri.</span><span class="sxs-lookup"><span data-stu-id="3fd58-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="3fd58-106">Satır içi Koleksiyonlar.</span><span class="sxs-lookup"><span data-stu-id="3fd58-106">Inline collections.</span></span>  
  
- <span data-ttu-id="3fd58-107">Anonim türler.</span><span class="sxs-lookup"><span data-stu-id="3fd58-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="3fd58-108">Sorgu veri kaynağına karşı yürütüldüğünde, sonuçlar CLR türlerine getirilir ve istemciye döndürülür.</span><span class="sxs-lookup"><span data-stu-id="3fd58-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="3fd58-109">Tüm nesne gerçekleştirmesi, [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)]tarafından gerçekleştirilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-109">All object materialization is performed by the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="3fd58-110">İle clr arasında eşleme yapılmamasına neden olan herhangi bir hata [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] , nesne materialization sırasında özel durumların oluşturulmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="3fd58-110">Any errors that result from an inability to map between the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] and the CLR will cause exceptions to be thrown during object materialization.</span></span>  
  
 <span data-ttu-id="3fd58-111">Sorgu yürütmesi basit kavramsal model türleri döndürürse, sonuçlar tek başına olan ve bağlantısı kesilen [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)]CLR türlerinden oluşur.</span><span class="sxs-lookup"><span data-stu-id="3fd58-111">If the query execution returns primitive conceptual model types, the results consist of CLR types that are stand-alone and disconnected from the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="3fd58-112">Ancak sorgu, türü ile <xref:System.Data.Objects.ObjectQuery%601>temsil edilen bir tür varlık nesneleri koleksiyonu döndürürse, bu türler nesne bağlamı tarafından izlenir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="3fd58-113">Tüm nesne davranışları (alt/üst koleksiyonlar, değişiklik izleme, çok biçimlilik vb [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].) içinde tanımlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="3fd58-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="3fd58-114">Bu işlev, [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)]içinde tanımlandığı şekilde kapasitesinde kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-114">This functionality can be used in its capacity, as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="3fd58-115">Daha fazla bilgi için bkz. [nesneleriyle çalışma](../working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="3fd58-115">For more information, see [Working with Objects](../working-with-objects.md).</span></span>  
  
 <span data-ttu-id="3fd58-116">Sorgulardan döndürülen yapı türleri (anonim türler ve null yapılabilir karmaşık türler gibi) `null` değer olabilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="3fd58-117">Döndürülen <xref:System.Data.Objects.DataClasses.EntityCollection%601> bir varlığın özelliği de bir `null` değer olabilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="3fd58-118">Bu, herhangi bir öğeye sahip olmayan bir `null` <xref:System.Data.Objects.ObjectQuery%601> ' ın çağrılması <xref:System.Linq.Queryable.FirstOrDefault%2A> gibi, değer olan bir varlığın koleksiyon özelliğinin yansıtıyapılmasının oluşmasına neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="3fd58-119">Belirli durumlarda, yürütme sırasında gerçekleştirilmiş sonuç oluşturmak için bir sorgu görünebilir, ancak sorgu sunucuda yürütülecektir ve varlık nesnesi hiçbir şekilde CLR 'de gerçekleştirilmeyecektir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="3fd58-120">Bu, nesne materialization 'in yan etkileriyle ilgili olarak sorun oluşmasına neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="3fd58-121">Aşağıdaki örnek, `LastName` özelliği ile özel bir sınıfı `MyContact`içerir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="3fd58-122">Özellik ayarlandığında, bir `count` değişken artırılır. `LastName`</span><span class="sxs-lookup"><span data-stu-id="3fd58-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="3fd58-123">Aşağıdaki iki sorguyu çalıştırırsanız, ikinci sorgu bu sırada ilk sorgu artacaktır `count` .</span><span class="sxs-lookup"><span data-stu-id="3fd58-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="3fd58-124">Bunun nedeni, ikinci sorgudaki `LastName` özelliğin sonuçlardan yansıtıldığı `MyContact` ve sınıfın depoda hiçbir şekilde oluşturulamadığından, sorgunun mağazada yürütülmesi gerekmediği için değildir.</span><span class="sxs-lookup"><span data-stu-id="3fd58-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>

# <a name="query-results"></a><span data-ttu-id="07f56-102">Sorgu Sonuçları</span><span class="sxs-lookup"><span data-stu-id="07f56-102">Query Results</span></span>
<span data-ttu-id="07f56-103">Sonra bir [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] sorgu komut ağaçlarını dönüştürülür ve yürütülen, sorgu sonuçları genellikle aşağıdakilerden biri döndürülür:</span><span class="sxs-lookup"><span data-stu-id="07f56-103">After a [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] query is converted to command trees and executed, the query results are usually returned as one of the following:</span></span>  
  
- <span data-ttu-id="07f56-104">Sıfır veya daha fazla yazılan bir varlığın nesnelerin veya projeksiyon kavramsal modeldeki karmaşık türleri koleksiyonu.</span><span class="sxs-lookup"><span data-stu-id="07f56-104">A collection of zero or more typed entity objects or a projection of complex types in the conceptual model.</span></span>  
  
- <span data-ttu-id="07f56-105">Kavramsal model tarafından desteklenen CLR türleri.</span><span class="sxs-lookup"><span data-stu-id="07f56-105">CLR types supported by the conceptual model.</span></span>  
  
- <span data-ttu-id="07f56-106">Satır içi koleksiyonları.</span><span class="sxs-lookup"><span data-stu-id="07f56-106">Inline collections.</span></span>  
  
- <span data-ttu-id="07f56-107">Anonim türler.</span><span class="sxs-lookup"><span data-stu-id="07f56-107">Anonymous types.</span></span>  
  
 <span data-ttu-id="07f56-108">Veri kaynağına karşı sorgu yürütüldü, sonuç CLR türleriyle gerçekleştirilmiş ve istemciye döndürülen.</span><span class="sxs-lookup"><span data-stu-id="07f56-108">When the query has executed against the data source, the results are materialized into CLR types and returned to the client.</span></span> <span data-ttu-id="07f56-109">Tüm nesne gerçekleştirme tarafından gerçekleştirilen [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="07f56-109">All object materialization is performed by the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="07f56-110">Eşleştirilemez arasında sonucunda herhangi bir hata [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] ve CLR özel nesne gerçekleştirme sırasında durum oluşturulmasına neden olur.</span><span class="sxs-lookup"><span data-stu-id="07f56-110">Any errors that result from an inability to map between the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)] and the CLR will cause exceptions to be thrown during object materialization.</span></span>  
  
 <span data-ttu-id="07f56-111">Sorgu yürütme ilkel kavramsal model türü döndürürse, tek başına ve bağlantısı kesilmiş olan CLR türlerini sonuçları oluşur [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="07f56-111">If the query execution returns primitive conceptual model types, the results consist of CLR types that are stand-alone and disconnected from the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="07f56-112">Sorgu yazılan varlığın nesnelerinin bir koleksiyonunu döndürür, ancak tarafından temsil edilen <xref:System.Data.Objects.ObjectQuery%601>, bu türleri nesne bağlamı tarafından izlenir.</span><span class="sxs-lookup"><span data-stu-id="07f56-112">However, if the query returns a collection of typed entity objects, represented by <xref:System.Data.Objects.ObjectQuery%601>, those types are tracked by the object context.</span></span> <span data-ttu-id="07f56-113">Tüm nesne davranışı (örneğin, alt/üst koleksiyonları, değişiklik izleme, çok biçimlilik ve benzeri) olan sınıfında tanımlandığı gibi [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="07f56-113">All object behavior (such as child/parent collections, change tracking, polymorphism, and so on) are as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="07f56-114">Bu işlev, kapasitede sınıfında tanımlandığı gibi kullanılabilir [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span><span class="sxs-lookup"><span data-stu-id="07f56-114">This functionality can be used in its capacity, as defined in the [!INCLUDE[adonet_ef](../../../../../../includes/adonet-ef-md.md)].</span></span> <span data-ttu-id="07f56-115">Daha fazla bilgi için [nesneleriyle çalışma](../../../../../../docs/framework/data/adonet/ef/working-with-objects.md).</span><span class="sxs-lookup"><span data-stu-id="07f56-115">For more information, see [Working with Objects](../../../../../../docs/framework/data/adonet/ef/working-with-objects.md).</span></span>  
  
 <span data-ttu-id="07f56-116">Yapı türleri (örneğin anonim türler ve null olabilir karmaşık türler) sorgularından döndürülen olabilir `null` değeri.</span><span class="sxs-lookup"><span data-stu-id="07f56-116">Struct types returned from queries (such as anonymous types and nullable complex types) can be of `null` value.</span></span> <span data-ttu-id="07f56-117">Bir <xref:System.Data.Objects.DataClasses.EntityCollection%601> döndürülen bir varlığın özelliği biri de olabilir `null` değeri.</span><span class="sxs-lookup"><span data-stu-id="07f56-117">An <xref:System.Data.Objects.DataClasses.EntityCollection%601> property of a returned entity can also be of `null` value.</span></span> <span data-ttu-id="07f56-118">Bu, bir varlık koleksiyon özelliğini yansıtma gelen sonuçlanabilir `null` arama gibi değeri <xref:System.Linq.Queryable.FirstOrDefault%2A> üzerinde bir <xref:System.Data.Objects.ObjectQuery%601> , öğe yok.</span><span class="sxs-lookup"><span data-stu-id="07f56-118">This can result from projecting the collection property of an entity that is of `null` value, such as calling <xref:System.Linq.Queryable.FirstOrDefault%2A> on an <xref:System.Data.Objects.ObjectQuery%601> that has no elements.</span></span>  
  
 <span data-ttu-id="07f56-119">Bazı durumlarda, bir sorgu yürütmesi sırasında gerçekleştirilmiş bir sonuç üretmek için görünebilir ancak sorgu sunucuda yürütülür ve varlık nesnesi hiçbir zaman CLR'de gerçekleştirilmiş.</span><span class="sxs-lookup"><span data-stu-id="07f56-119">In certain situations, a query might appear to generate a materialized result during its execution, but the query will be executed on the server and the entity object will never be materialized in the CLR.</span></span> <span data-ttu-id="07f56-120">Nesne gerçekleştirme, yan etkileri bağlı, bu sorunlara neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="07f56-120">This can cause problems if you are depending on side effects of object materialization.</span></span>  
  
 <span data-ttu-id="07f56-121">Aşağıdaki örnek, özel bir sınıf içerir `MyContact`, ile bir `LastName` özelliği.</span><span class="sxs-lookup"><span data-stu-id="07f56-121">The following example contains a custom class, `MyContact`, with a `LastName` property.</span></span> <span data-ttu-id="07f56-122">Zaman `LastName` özelliği ayarlanmış bir `count` değişkeni artırılır.</span><span class="sxs-lookup"><span data-stu-id="07f56-122">When the `LastName` property is set, a `count` variable is incremented.</span></span> <span data-ttu-id="07f56-123">İki aşağıdaki sorguları yürütün, ilk sorgu artırır `count` karşın ikinci sorgu erişemez.</span><span class="sxs-lookup"><span data-stu-id="07f56-123">If you execute the two following queries, the first query will increment `count` while the second query will not.</span></span> <span data-ttu-id="07f56-124">Bunun nedeni, ikinci sorgu `LastName` özelliği sonuçlardan öngörülen ve `MyContact` sınıf hiçbir zaman oluşturulduğu, mağaza sorguyu yürütmek için gerekli olmadığından.</span><span class="sxs-lookup"><span data-stu-id="07f56-124">This is because in the second query the `LastName` property is projected from the results and the `MyContact` class is never created, because it is not required to execute the query on the store.</span></span>

 <span data-ttu-id="3aee1-110">Tarafından oluşturulan kodu [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] varsayılan olarak ertelenmiş yükleme özellikleri sağlar.</span><span class="sxs-lookup"><span data-stu-id="3aee1-110">The code generated by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] provides deferred loading capabilities by default.</span></span> <span data-ttu-id="3aee1-111">Ertelenen yükleme Orta katmanda çok isteğe bağlı olarak saydam yüklenmesi için uygundur.</span><span class="sxs-lookup"><span data-stu-id="3aee1-111">Deferred loading is very convenient on the mid-tier for transparent loading of data on demand.</span></span> <span data-ttu-id="3aee1-112">Ertelenen yükleme veya istenip istenmediğini ertelenmiş yükleniyor seri hale getirici tetikler. ancak, seri hale getirme için sorunlu olmasıdır.</span><span class="sxs-lookup"><span data-stu-id="3aee1-112">However, it is problematic for serialization, because the serializer triggers deferred loading whether deferred loading is intended or not.</span></span> <span data-ttu-id="3aee1-113">Aslında, bir nesne seri olduğunda, tüm giden erteleme yüklenen başvuruları altında geçişli kendi kapatma seri hale getirilir.</span><span class="sxs-lookup"><span data-stu-id="3aee1-113">In effect, when an object is serialized, its transitive closure under all outbound defer-loaded references is serialized.</span></span>

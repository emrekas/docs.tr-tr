<span data-ttu-id="3fb75-103">Çoğu uygulama, temel tablolar için erişimi kısıtlamak için sahiplik zinciri bağlı olan verilere erişmek için saklı yordamlar kullanır.</span><span class="sxs-lookup"><span data-stu-id="3fb75-103">Many applications use stored procedures to access data, relying on ownership chaining to restrict access to base tables.</span></span> <span data-ttu-id="3fb75-104">Saklı yordamları ÇALIŞTIRMA izinlerini iptal etme veya temel tablolardan izinleri reddetme verebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="3fb75-104">You can grant EXECUTE permissions on stored procedures, revoking or denying permissions on the base tables.</span></span> <span data-ttu-id="3fb75-105">SQL Server tabloları ve saklı yordam sahibi aynıysa çağıranın izinlerini kontrol yapmaz.</span><span class="sxs-lookup"><span data-stu-id="3fb75-105">SQL Server does not check the permissions of the caller if the stored procedure and tables have the same owner.</span></span> <span data-ttu-id="3fb75-106">Ancak, sahiplik zinciri nesneleri farklı sahipleri varsa veya dinamik SQL söz konusu olduğunda çalışmaz.</span><span class="sxs-lookup"><span data-stu-id="3fb75-106">However, ownership chaining doesn't work if objects have different owners or in the case of dynamic SQL.</span></span>  
  
 <span data-ttu-id="3fb75-107">Kullanabileceğiniz EXECUTE AS yan tümcesinde başvurulan veritabanı nesneleri üzerinde çağıran izinlere sahip olmadığında bir saklı yordam.</span><span class="sxs-lookup"><span data-stu-id="3fb75-107">You can use the EXECUTE AS clause in a stored procedure when the caller doesn't have permissions on the referenced database objects.</span></span> <span data-ttu-id="3fb75-108">Geçerli yürütme bağlamı için Ara sunucu kullanıcı anahtarlanır EXECUTE AS yan tümcesi ise.</span><span class="sxs-lookup"><span data-stu-id="3fb75-108">The effect of the EXECUTE AS clause is that the execution context is switched to the proxy user.</span></span> <span data-ttu-id="3fb75-109">İç içe geçmiş saklı yordamları ve tetikleyicileri, çağrıları yanı sıra, tüm kod proxy kullanıcının güvenlik bağlamı altında çalışır.</span><span class="sxs-lookup"><span data-stu-id="3fb75-109">All code, as well as any calls to nested stored procedures or triggers, executes under the security context of the proxy user.</span></span> <span data-ttu-id="3fb75-110">Yürütme bağlamı yalnızca yürütme yordamın veya geri DÖNDÜRME deyimi verildiğinde sonra özgün çağırana döndürülür.</span><span class="sxs-lookup"><span data-stu-id="3fb75-110">Execution context is reverted to the original caller only after execution of the procedure or when a REVERT statement is issued.</span></span>  
  
## <a name="context-switching-with-the-execute-as-statement"></a><span data-ttu-id="3fb75-111">Bağlam değiştirme EXECUTE AS deyimi</span><span class="sxs-lookup"><span data-stu-id="3fb75-111">Context Switching with the EXECUTE AS Statement</span></span>  
 <span data-ttu-id="3fb75-112">Transact-SQL EXECUTE AS deyimi başka bir oturum açma veya veritabanı kullanıcının kimliğine bürünerek bir deyimi yürütme bağlamı geçiş yapmanızı sağlar.</span><span class="sxs-lookup"><span data-stu-id="3fb75-112">The Transact-SQL EXECUTE AS statement allows you to switch the execution context of a statement by impersonating another login or database user.</span></span> <span data-ttu-id="3fb75-113">Bu, sorgular ve yordamları başka bir kullanıcı olarak test etmek için yararlı bir tekniktir.</span><span class="sxs-lookup"><span data-stu-id="3fb75-113">This is a useful technique for testing queries and procedures as another user.</span></span>  
  
```  
EXECUTE AS LOGIN = 'loginName';  
EXECUTE AS USER = 'userName';  
```  
  
 <span data-ttu-id="3fb75-114">Oturum açma veya kullanıcı kimliğine bürünüyorsunuz ve DOĞRULAMASINDAN izinleriniz olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="3fb75-114">You must have IMPERSONATE permissions on the login or user you are impersonating.</span></span> <span data-ttu-id="3fb75-115">Bu izin için örtük `sysadmin` tüm veritabanları için ve `db_owner` oldukları veritabanı rolü üyeleri.</span><span class="sxs-lookup"><span data-stu-id="3fb75-115">This permission is implied for `sysadmin` for all databases, and `db_owner` role members in databases that they own.</span></span>  
  
## <a name="granting-permissions-with-the-execute-as-clause"></a><span data-ttu-id="3fb75-116">İzinleri ile EXECUTE AS yan tümcesi</span><span class="sxs-lookup"><span data-stu-id="3fb75-116">Granting Permissions with the EXECUTE AS Clause</span></span>  
 <span data-ttu-id="3fb75-117">Kullanabileceğiniz EXECUTE AS yan tümcesinde bir saklı yordam, tetikleyici veya kullanıcı tanımlı işlev (hariç, satır içi tablo değerli işlevler) tanımı üstbilgisi.</span><span class="sxs-lookup"><span data-stu-id="3fb75-117">You can use the EXECUTE AS clause in the definition header of a stored procedure, trigger, or user-defined function (except for inline table-valued functions).</span></span> <span data-ttu-id="3fb75-118">Bu kullanıcı adı veya anahtar sözcüğü EXECUTE AS belirtilen bağlamında yürütmek yordamı neden olan yan tümcesi.</span><span class="sxs-lookup"><span data-stu-id="3fb75-118">This causes the procedure to execute in the context of the user name or keyword specified in the EXECUTE AS clause.</span></span> <span data-ttu-id="3fb75-119">Yalnızca gerekli izinleri yordamı tarafından erişilen nesneler üzerinde verme bir oturum açma için eşlenmemiş veritabanında bir proxy kullanıcı oluşturabilir.</span><span class="sxs-lookup"><span data-stu-id="3fb75-119">You can create a proxy user in the database that is not mapped to a login, granting it only the necessary permissions on the objects accessed by the procedure.</span></span> <span data-ttu-id="3fb75-120">Yan tümcesi izinlerine sahip olmanız gerekir olarak yürütme belirtilen proxy kullanıcı tüm nesneleri modülü tarafından erişilir.</span><span class="sxs-lookup"><span data-stu-id="3fb75-120">Only the proxy user specified in the EXECUTE AS clause must have permissions on all objects accessed by the module.</span></span>

>  <span data-ttu-id="3fb75-121">TRUNCATE TABLE gibi bazı eylemleri verilebilen izinlere sahip değilsiniz.</span><span class="sxs-lookup"><span data-stu-id="3fb75-121">Some actions, such as TRUNCATE TABLE, do not have grantable permissions.</span></span> <span data-ttu-id="3fb75-122">Bir yordam içinde deyim ekleme ve ALTER TABLE izni bir ara sunucu kullanıcı belirterek, tabloyu yordamı yalnızca yürütme izinlerine sahip arayanlara kesmek için izinleri genişletebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="3fb75-122">By incorporating the statement within a procedure and specifying a proxy user who has ALTER TABLE permissions, you can extend the permissions to truncate the table to callers who have only EXECUTE permissions on the procedure.</span></span>  
  
 <span data-ttu-id="3fb75-123">EXECUTE yan tümcesi iç içe geçmiş yordamları ve Tetikleyicileri dahil olmak üzere bu yordamın süresince geçerli olduğundan belirtilen bağlam.</span><span class="sxs-lookup"><span data-stu-id="3fb75-123">The context specified in the EXECUTE AS clause is valid for the duration of the procedure, including nested procedures and triggers.</span></span> <span data-ttu-id="3fb75-124">Bağlam, yürütme veya geri DÖNDÜRME deyimi verilen çağırana döner.</span><span class="sxs-lookup"><span data-stu-id="3fb75-124">Context reverts to the caller when execution is complete or the REVERT statement is issued.</span></span>  
  
 <span data-ttu-id="3fb75-125">Üç adımı kullanan EXECUTE AS yan tümcesinde bir yordam.</span><span class="sxs-lookup"><span data-stu-id="3fb75-125">There are three steps involved in using the EXECUTE AS clause in a procedure.</span></span>  
  
1. <span data-ttu-id="3fb75-126">Bir oturum açmayla değil veritabanında bir ara sunucu kullanıcısı oluşturun.</span><span class="sxs-lookup"><span data-stu-id="3fb75-126">Create a proxy user in the database that is not mapped to a login.</span></span> <span data-ttu-id="3fb75-127">Bu gerekli değildir, ancak izinleri yönetirken yardımcı olur.</span><span class="sxs-lookup"><span data-stu-id="3fb75-127">This is not required, but it helps when managing permissions.</span></span>

1. <span data-ttu-id="3fb75-128">Proxy kullanıcı gerekli izinleri verin.</span><span class="sxs-lookup"><span data-stu-id="3fb75-128">Grant the proxy user the necessary permissions.</span></span>

> [!NOTE]
>  <span data-ttu-id="3fb75-130">Denetim gerektiren uygulamalar, arayanın özgün güvenlik bağlamı korunmaz çünkü bozabilir.</span><span class="sxs-lookup"><span data-stu-id="3fb75-130">Applications that require auditing can break because the original security context of the caller is not retained.</span></span> <span data-ttu-id="3fb75-131">SESSION_USER'ı, kullanıcı veya USER_NAME, geçerli kullanıcının kimliğini döndüren yerleşik işlevler EXECUTE AS ile ilişkili kullanıcıyı döndürür yan tümcesi, özgün çağırana.</span><span class="sxs-lookup"><span data-stu-id="3fb75-131">Built-in functions that return the identity of the current user, such as SESSION_USER, USER, or USER_NAME, return the user associated with the EXECUTE AS clause, not the original caller.</span></span>

Devralmada nesne yönelimli programlama dillerinin sınıfların devralma benzer.</span><span class="sxs-lookup"><span data-stu-id="9e107-104">Inheritance in the EDM is similar to inheritance for classes olarak, kavramsal modelde türetilmiş tür her zaman tüm devralır [özellikleri](../../../../docs/framework/data/adonet/property.md) ve [Gezinti  <span data-ttu-id="9e107-108">Kavramsal modelde, türetilmiş bir tür başka bir türetilmiş tür tarafından devralındığında devralma hiyerarşilerini oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="9e107-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="9e107-109">(Bir türü türetilmiş bir tür değil hiyerarşide) hiyerarşinin üst türü olarak adlandırılır *kök türü*.</span><span class="sxs-lookup"><span data-stu-id="9e107-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="9e107-110">Bir devralma hiyerarşisindeki [Varlık anahtarı](../../../../docs/framework/data/adonet/entity-key.md) kök türü üzerinde tanımlanmış olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="9e107-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="9e107-111">Türetilmiş bir tür birden fazla türünden devralan devralma hiyerarşilerini oluşturulamıyor.</span><span class="sxs-lookup"><span data-stu-id="9e107-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="9e107-112">Kavramsal bir modelle, örneğin, bir `Book` varlık türü türetilen türlerin tanımlayabilirsiniz `FictionBook` ve `NonFictionBook` her kaynağından devraldığından `Book`.</span><span class="sxs-lookup"><span data-stu-id="9e107-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="9e107-113">Ancak, daha sonra hem de devralan bir tür tanımlayabilirsiniz değil `FictionBook` ve `NonFictionBook` türleri.</span><span class="sxs-lookup"><span data-stu-id="9e107-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>

# <a name="navigation-property"></a><span data-ttu-id="8911f-102">Gezinme özelliği</span><span class="sxs-lookup"><span data-stu-id="8911f-102">navigation property</span></span>
<span data-ttu-id="8911f-103">A *gezinti özelliği* üzerinde isteğe bağlı bir özellik olan bir [varlık türü](../../../../docs/framework/data/adonet/entity-type.md) olanak tanıyan bir gezinti için [son](../../../../docs/framework/data/adonet/association-end.md) , bir [ilişkilendirme](../../../../docs/framework/data/adonet/association-type.md) için diğer sonu.</span><span class="sxs-lookup"><span data-stu-id="8911f-103">A *navigation property* is an optional property on an [entity type](../../../../docs/framework/data/adonet/entity-type.md) that allows for navigation from one [end](../../../../docs/framework/data/adonet/association-end.md) of an [association](../../../../docs/framework/data/adonet/association-type.md) to the other end.</span></span> <span data-ttu-id="8911f-104">Diğer aksine [özellikleri](../../../../docs/framework/data/adonet/property.md), gezinti özellikleri veri taşıyan değil.</span><span class="sxs-lookup"><span data-stu-id="8911f-104">Unlike other [properties](../../../../docs/framework/data/adonet/property.md), navigation properties do not carry data.</span></span>  
  
 <span data-ttu-id="8911f-105">Bir gezinti özelliği tanımı aşağıdakileri içerir:</span><span class="sxs-lookup"><span data-stu-id="8911f-105">A navigation property definition includes the following:</span></span>  
  
-   <span data-ttu-id="8911f-106">Bir ad.</span><span class="sxs-lookup"><span data-stu-id="8911f-106">A name.</span></span> <span data-ttu-id="8911f-107">(Gerekli)</span><span class="sxs-lookup"><span data-stu-id="8911f-107">(Required)</span></span>  
  
-   <span data-ttu-id="8911f-108">Bu varlıklardan ilişkilendirme.</span><span class="sxs-lookup"><span data-stu-id="8911f-108">The association that it navigates.</span></span> <span data-ttu-id="8911f-109">(Gerekli)</span><span class="sxs-lookup"><span data-stu-id="8911f-109">(Required)</span></span>  
  
-   <span data-ttu-id="8911f-110">Bu varlıklardan ilişkilendirmenin sona erer.</span><span class="sxs-lookup"><span data-stu-id="8911f-110">The ends of the association that it navigates.</span></span> <span data-ttu-id="8911f-111">(Gerekli)</span><span class="sxs-lookup"><span data-stu-id="8911f-111">(Required)</span></span>  
  
 <span data-ttu-id="8911f-112">Gezinti özellikleri her iki varlık türlerini ilişkilendirme ucunda isteğe bağlı olduğunu unutmayın.</span><span class="sxs-lookup"><span data-stu-id="8911f-112">Note that navigation properties are optional on both entity types at the ends of an association.</span></span> <span data-ttu-id="8911f-113">İlişkilendirme sonunda bir varlık türünün gezinme özelliğinin tanımlarsanız, ilişkinin diğer ucundaki varlık türünün gezinme özelliğinin tanımlamak zorunda değil.</span><span class="sxs-lookup"><span data-stu-id="8911f-113">If you define a navigation property on one entity type at the end of an association, you do not have to define a navigation property on the entity type at the other end of the association.</span></span>  
  
 <span data-ttu-id="8911f-114">Veri türü bir gezinti özelliği tarafından belirlenir [çoğulluk](../../../../docs/framework/data/adonet/association-end-multiplicity.md) kendi uzak [ilişkilendirme end](../../../../docs/framework/data/adonet/association-end.md).</span><span class="sxs-lookup"><span data-stu-id="8911f-114">The data type of a navigation property is determined by the [multiplicity](../../../../docs/framework/data/adonet/association-end-multiplicity.md) of its remote [association end](../../../../docs/framework/data/adonet/association-end.md).</span></span> <span data-ttu-id="8911f-115">Örneğin, bir gezinti özelliği varsayalım `OrdersNavProp`, bulunuyor. bir `Customer` varlık türü ve arasında bir-çok ilişkisi gider `Customer` ve `Order`.</span><span class="sxs-lookup"><span data-stu-id="8911f-115">For example, suppose a navigation property, `OrdersNavProp`, exists on a `Customer` entity type and navigates a one-to-many association between `Customer` and `Order`.</span></span> <span data-ttu-id="8911f-116">Gezinti özelliği için Uzak ilişkilendirme end çokluğu birçok (\*) olduğundan, kendi veri türüne bir koleksiyonudur (biri `Order`).</span><span class="sxs-lookup"><span data-stu-id="8911f-116">Because the remote association end for the navigation property has multiplicity of many (\*), its data type is a collection (of `Order`).</span></span> <span data-ttu-id="8911f-117">Benzer şekilde, bir gezinti özelliği, `CustomerNavProp`, bulunuyor `Order` varlık türü, veri türü olacak `Customer`, uzak uç çokluğu bir (1) olduğundan.</span><span class="sxs-lookup"><span data-stu-id="8911f-117">Similarly, if a navigation property, `CustomerNavProp`, exists on the `Order` entity type, its data type would be `Customer`, because the multiplicity of the remote end is one (1).</span></span>

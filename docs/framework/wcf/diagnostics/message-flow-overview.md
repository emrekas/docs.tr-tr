### <a name="to-enable-message-flow-tracing-in-the-getting-started-tutorial"></a><span data-ttu-id="48a01-113">Başlarken öğreticide ileti akışı izlemeyi etkinleştirmek için</span><span class="sxs-lookup"><span data-stu-id="48a01-113">To enable message flow tracing in the Getting Started tutorial</span></span>

2. <span data-ttu-id="48a01-115">Çözümleme izleme etkinleştirmediyseniz genişletin **uygulama ve hizmet günlükleri**, **Microsoft**, **Windows**, **uygulama sunucu uygulamaları** .</span><span class="sxs-lookup"><span data-stu-id="48a01-115">If you haven’t enabled analytic tracing, expand **Applications and Services Logs**, **Microsoft**, **Windows**, **Application Server-Applications**.</span></span> <span data-ttu-id="48a01-116">Seçin **görünümü**, **Göster Analitik ve hata ayıklama günlüklerini**.</span><span class="sxs-lookup"><span data-stu-id="48a01-116">Select **View**, **Show Analytic and Debug Logs**.</span></span> <span data-ttu-id="48a01-117">Sağ **analitik** seçip **günlüğü etkinleştir**.</span><span class="sxs-lookup"><span data-stu-id="48a01-117">Right-click **Analytic** and select **Enable Log**.</span></span> <span data-ttu-id="48a01-118">Olay Görüntüleyicisi, böylece izlemeleri görüntülenebilir açık bırakın.</span><span class="sxs-lookup"><span data-stu-id="48a01-118">Leave Event Viewer open so that traces can be viewed.</span></span>

3. <span data-ttu-id="48a01-119">Oluşturulan örnek açın [başlangıç Öğreticisi](../../../../docs/framework/wcf/getting-started-tutorial.md) Visual Studio 2012.</span><span class="sxs-lookup"><span data-stu-id="48a01-119">Open the sample created in the [Getting Started Tutorial](../../../../docs/framework/wcf/getting-started-tutorial.md) in Visual Studio 2012.</span></span> <span data-ttu-id="48a01-120">Böylece hizmet oluşturulabilmesi için yönetici olarak Visual Studio 2012 çalıştırmanız gerektiğini unutmayın.</span><span class="sxs-lookup"><span data-stu-id="48a01-120">Note that you must run Visual Studio 2012 as an administrator so that the service can be created.</span></span> <span data-ttu-id="48a01-121">Yüklü WCF örnekleri varsa açabileceğiniz [Başlarken](../../../../docs/framework/wcf/samples/getting-started-sample.md), öğreticide oluşturulan projeyi içerir.</span><span class="sxs-lookup"><span data-stu-id="48a01-121">If you have the WCF samples installed, you can open the [Getting Started](../../../../docs/framework/wcf/samples/getting-started-sample.md), which contains the completed project created in the tutorial.</span></span>

4. <span data-ttu-id="48a01-122">Sağ **hizmet** seçin ve proje **Ekle**, **yeni öğe**.</span><span class="sxs-lookup"><span data-stu-id="48a01-122">Right-click the **Service** project and select **Add**, **New Item**.</span></span> <span data-ttu-id="48a01-123">Seçin **uygulama yapılandırma dosyası** tıklatıp **Tamam**.</span><span class="sxs-lookup"><span data-stu-id="48a01-123">Select **Application Configuration File** and click **OK**.</span></span>

5. <span data-ttu-id="48a01-124">Önceki adımda oluşturulan App.Config dosyasına aşağıdaki kodu ekleyin.</span><span class="sxs-lookup"><span data-stu-id="48a01-124">Add the following code to the App.Config file created in the previous step.</span></span>

      </diagnostics>
    </system.serviceModel>
    ```

6. <span data-ttu-id="48a01-125">CTRL + F5 tuşlarına basarak hata ayıklama olmadan sunucu uygulamasını çalıştırın.</span><span class="sxs-lookup"><span data-stu-id="48a01-125">Execute the server application without debugging by pressing CTRL+F5.</span></span> <span data-ttu-id="48a01-126">İstemci projesi sağ tıklayarak yürütme **istemci** proje ve seçerek **hata ayıklama**, **yeni örnek Başlat**.</span><span class="sxs-lookup"><span data-stu-id="48a01-126">Execute the client project by right-clicking the **Client** project and selecting **Debug**, **Start New Instance**.</span></span>

7. <span data-ttu-id="48a01-127">İstemciden sunucuya olayları izlemek için istemci projesindeki uygulama yapılandırma dosyasına aşağıdakileri ekleyin.</span><span class="sxs-lookup"><span data-stu-id="48a01-127">To trace the events from the client to the server, add the following to the application configuration file in the Client project.</span></span>

      <endToEndTracing propagateActivity="true" messageFlowTracing="true"/>
    </diagnostics>
    ```

8. <span data-ttu-id="48a01-128">İstemci program.cs'ye aşağıdaki Using deyimini ekleyin.</span><span class="sxs-lookup"><span data-stu-id="48a01-128">In Program.cs in the client, add the following Using statement.</span></span>

    ```csharp

    ```

9. <span data-ttu-id="48a01-129">İstemci projesindeki program.cs dosyasının ana yöntemde olay günlüğünde dağıtılmasını izleme GUID'i ayarlayın.</span><span class="sxs-lookup"><span data-stu-id="48a01-129">In the Main method in the program.cs file in the client project, set the Trace GUID to be propagated in the event log.</span></span>

    Trace.CorrelationManager.ActivityId = guid;
    ```

10. <span data-ttu-id="48a01-130">Yenileyin ve incelemek **analitik** günlük.</span><span class="sxs-lookup"><span data-stu-id="48a01-130">Refresh and examine the **Analytic**  log.</span></span>  <span data-ttu-id="48a01-131">Olay Kimliği 220 sahip bir olay olup olmadığına bakın.</span><span class="sxs-lookup"><span data-stu-id="48a01-131">Look for an event with Event ID 220.</span></span>  <span data-ttu-id="48a01-132">Olay seçin ve tıklayın **ayrıntıları** önizleme bölmesinde sekmesi.</span><span class="sxs-lookup"><span data-stu-id="48a01-132">Select the event, and click the **Details** tab in the preview pane.</span></span> <span data-ttu-id="48a01-133">Bu olay arama etkinliği için bağıntı Kimliğini içerir.</span><span class="sxs-lookup"><span data-stu-id="48a01-133">This event will contain the correlation ID for the calling activity.</span></span>

    > [!NOTE]
    >  <span data-ttu-id="48a01-134">Tüm olaylar aynı GUID etkinlik kimliği ile bir istekle ilişkili.</span><span class="sxs-lookup"><span data-stu-id="48a01-134">All events with the same GUID in the ActivityID are related to one request.</span></span> <span data-ttu-id="48a01-135">Bu, belirli bir istemci bir kuyruktan belirli bir hizmete ilişkilendirmek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="48a01-135">This can be used to correlate messages from a specific client to a specific service.</span></span> <span data-ttu-id="48a01-136">İstemci başka bir hizmet çağrılır, aynı istemci tarafından ActivityID tanımlanamadı.</span><span class="sxs-lookup"><span data-stu-id="48a01-136">If the client called another service, then the same client could be identified by ActivityID.</span></span>

11. <span data-ttu-id="48a01-137">Bazı durumlarda, etkinlik kimliği için yeni bir etkinlik kimliği özgün GUID'ini değiştirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="48a01-137">In some cases, the ActivityID can change from the original GUID to a new ActivityID.</span></span> <span data-ttu-id="48a01-138">Bu durumda, bir aktarım olayını yayınlanır.</span><span class="sxs-lookup"><span data-stu-id="48a01-138">In that case, a transfer event is emitted.</span></span> <span data-ttu-id="48a01-139">Bu olay kimliği 499 ve olay başlığı aşağıdaki verileri içerir.</span><span class="sxs-lookup"><span data-stu-id="48a01-139">This event ID is 499, and the event will contain the following data in the header.</span></span>

        <System>
            <Provider Name="Microsoft-Windows-Application Server-Applications" Guid="{c651f5f6-1c0d-492e-8ae1-b4efd7c9d503}" />
            <EventID>499</EventID>
            ...
            <Correlation ActivityID="{A066CCF1-8AB3-459B-B62F-F79F957A5036}" RelatedActivityID="{85FC0930-9C49-42DA-804B-A7368104BD1B}" />
            ...
       </System>
    </Event>
    ```

    > [!NOTE]
    >  <span data-ttu-id="48a01-140">Aktarım olayını, etkin RelatedActivityID ActivityID GUID olarak belirtilen olarak belirtilen GUID etkinlik değişikliği kaydeder.</span><span class="sxs-lookup"><span data-stu-id="48a01-140">The transfer event records the change of the active ActivityID from the GUID specified as the ActivityID to the GUID specified as RelatedActivityID.</span></span> <span data-ttu-id="48a01-141">Aktarım olayını yayılan sonra tüm olayları yeni bir GUID etkinlik kimliği içerir.</span><span class="sxs-lookup"><span data-stu-id="48a01-141">After the transfer event is emitted, all events will contain the new GUID as the ActivityID.</span></span>

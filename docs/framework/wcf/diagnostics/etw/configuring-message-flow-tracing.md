## <a name="enabling-tracing"></a><span data-ttu-id="22e77-108">İzlemeyi etkinleştirme</span><span class="sxs-lookup"><span data-stu-id="22e77-108">Enabling Tracing</span></span>  
 <span data-ttu-id="22e77-109">.NET Framework 4 ayarlayarak ileti akışı izlemeyi etkinleştirebilirsiniz `messageFlowTracing` yapılandırma öğesine `true`, aşağıdaki örnekte gösterildiği gibi.</span><span class="sxs-lookup"><span data-stu-id="22e77-109">You can enable message flow tracing by setting the .NET Framework 4 `messageFlowTracing` configuration element to `true`, as shown in the following example.</span></span>  
  
```xml  
<system.servicemodel>  
  <diagnostics>  
    <endToEndTracing propagateActivity="true" messageFlowTracing="true" />  
  </diagnostics>  
</system.servicemodel>  
```  
  
> [!NOTE]
>  <span data-ttu-id="22e77-110">Çünkü `endToEndTracing` yapılandırma öğesi, Web.config dosyasında bulunan, ETW aynı şekilde dinamik olarak yapılandırılamaz.</span><span class="sxs-lookup"><span data-stu-id="22e77-110">Because the `endToEndTracing` configuration element resides in a Web.config file, it cannot be dynamically configured in the same way as ETW.</span></span> <span data-ttu-id="22e77-111">İçin `endToEndTracing` etkili uygulama yapılandırma öğesi geri dönüştürülmüş olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="22e77-111">For the `endToEndTracing` configuration element to take effect, the application must be recycled.</span></span>


 <span data-ttu-id="22e77-121">Aşağıdaki kodu kullanarak üstbilgi program aracılığıyla ekleyebilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="22e77-121">You can programmatically add the header using the following code:</span></span>

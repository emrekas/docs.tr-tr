# <a name="streaming-message-transfer"></a><span data-ttu-id="6ebb4-102">İleti Aktarma Akışı</span><span class="sxs-lookup"><span data-stu-id="6ebb4-102">Streaming Message Transfer</span></span>
<span data-ttu-id="6ebb4-103">Windows Communication Foundation (WCF) taşımalar iletileri aktarmak için iki modu destekler:</span><span class="sxs-lookup"><span data-stu-id="6ebb4-103">Windows Communication Foundation (WCF) transports support two modes for transferring messages:</span></span>  
  
-   <span data-ttu-id="6ebb4-104">Arabelleğe alınan aktarımları aktarma işlemi tamamlanana kadar tüm ileti içinde bir bellek arabelleğini basılı tutun.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-104">Buffered transfers hold the entire message in a memory buffer until the transfer is complete.</span></span> <span data-ttu-id="6ebb4-105">Bir alıcı okumadan önce bir arabelleğe alınan ileti tamamen teslim edilmelidir.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-105">A buffered message must be completely delivered before a receiver can read it.</span></span>  
  
-   <span data-ttu-id="6ebb4-106">Akış aktarımları ileti akışı olarak kullanıma sunar.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-106">Streamed transfers expose the message as a stream.</span></span> <span data-ttu-id="6ebb4-107">Alıcı, tamamen teslim edilmeden önce iletiyi işlemeyi başlatır.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-107">The receiver starts processing the message before it is completely delivered.</span></span>  
  
-   <span data-ttu-id="6ebb4-108">Akış aktarımları, büyük bellek arabelleği gereksinimini ortadan kaldırarak hizmet ölçeklenebilirliği artırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-108">Streamed transfers can improve the scalability of a service by eliminating the requirement for large memory buffers.</span></span> <span data-ttu-id="6ebb4-109">Aktarım modunu değiştirme ölçeklenebilirliği iyileştirdi olmadığını aktarılan iletileri boyutuna bağlıdır.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-109">Whether changing the transfer mode improves scalability depends on the size of the messages being transferred.</span></span> <span data-ttu-id="6ebb4-110">Büyük ileti boyutları, akış aktarımları kullanarak favor.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-110">Large message sizes favor using streamed transfers.</span></span>

 <span data-ttu-id="6ebb4-128">WCF bu sürüm ayrıca adde doğru zaman uyumsuz gönderme tarafında akış yeteneği.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-128">This version of WCF also adde the capability of true asynchronous streaming on the send side.</span></span> <span data-ttu-id="6ebb4-129">Bu senaryolarda bazıları okuma yavaş olan birden çok istemciye iletileri burada akış hizmetinin ölçeklenebilirliği artırır; büyük olasılıkla Ağ Tıkanıklığı veya olan nedeniyle hiç okunmuyor.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-129">This improves scalability of the service in scenarios where it is streaming messages to multiple clients some of which are slow in reading; possibly due to network congestion or are not reading at all.</span></span> <span data-ttu-id="6ebb4-130">Bu senaryolarda WCF istemci başına hizmetine tek tek iş parçacığı artık engeller.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-130">In these scenarios WCF no longer blocks individual threads on the service per client.</span></span> <span data-ttu-id="6ebb4-131">Bu, hizmet birden çok daha fazla istemci böylece hizmetinin ölçeklenebilirliği artırma işleminin olmasını sağlar.</span><span class="sxs-lookup"><span data-stu-id="6ebb4-131">This ensures that the service is able to process many more clients thereby improving the scalability of the service.</span></span>

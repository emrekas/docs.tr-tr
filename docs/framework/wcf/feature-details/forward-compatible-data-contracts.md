<span data-ttu-id="d07ea-103">Bir özellik, Windows Communication Foundation (veri sözleşme sistem sözleşmelerin WCF) zaman içindeki bölünemez şekilde geliştirebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d07ea-103">A feature of the Windows Communication Foundation (WCF) data contract system is that contracts can evolve over time in nonbreaking ways.</span></span> <span data-ttu-id="d07ea-104">Diğer bir deyişle, bir istemci bir veri anlaşması daha eski bir sürümüyle aynı veri anlaşması bir sürüme sahip bir hizmet ile iletişim kurabilir veya bir istemci bir veri anlaşması'nın daha yeni bir sürümü ile aynı veri anlaşması daha eski bir sürümü ile iletişim kurabilir.</span><span class="sxs-lookup"><span data-stu-id="d07ea-104">That is, a client with an older version of a data contract can communicate with a service with a newer version of the same data contract, or a client with a newer version of a data contract can communicate with an older version of the same data contract.</span></span> <span data-ttu-id="d07ea-105">Daha fazla bilgi için [en iyi uygulamalar: Veri sözleşmesi sürümü oluşturma](../../../../docs/framework/wcf/best-practices-data-contract-versioning.md).</span><span class="sxs-lookup"><span data-stu-id="d07ea-105">For more information, see [Best Practices: Data Contract Versioning](../../../../docs/framework/wcf/best-practices-data-contract-versioning.md).</span></span>  
  
 <span data-ttu-id="d07ea-106">Mevcut bir veri anlaşması yeni sürümlerini oluşturulduğunda bir gerektiği şekilde sürüm oluşturma özelliklerinin çoğu uygulayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d07ea-106">You can apply most of the versioning features on an as-needed basis when new versions of an existing data contract are created.</span></span> <span data-ttu-id="d07ea-107">Ancak, bir sürüm oluşturma özelliği, *gidiş dönüşü*, ilk sürüm türünden düzgün çalışması için yerleşik gerekir.</span><span class="sxs-lookup"><span data-stu-id="d07ea-107">However, one versioning feature, *round-tripping*, must be built into the type from the first version in order to work properly.</span></span>  
  
## <a name="round-tripping"></a><span data-ttu-id="d07ea-108">Gidiş dönüşü</span><span class="sxs-lookup"><span data-stu-id="d07ea-108">Round-Tripping</span></span>  
 <span data-ttu-id="d07ea-109">Gidiş dönüşü, verileri yeni bir sürümü eski bir sürüm ve tekrar yeni bir veri sözleşmesi sürümü başarılı olduğunda gerçekleşir.</span><span class="sxs-lookup"><span data-stu-id="d07ea-109">Round-tripping occurs when data passes from a new version to an old version and back to the new version of a data contract.</span></span> <span data-ttu-id="d07ea-110">Gidiş dönüşü, veri kaybı olmamasına garanti eder.</span><span class="sxs-lookup"><span data-stu-id="d07ea-110">Round-tripping guarantees that no data is lost.</span></span> <span data-ttu-id="d07ea-111">Gidiş dönüşü etkinleştirme İleri uyumlu veri sözleşmesi sürümü oluşturma modeli tarafından desteklenen tüm gelecek değişiklikler ile yapar.</span><span class="sxs-lookup"><span data-stu-id="d07ea-111">Enabling round-tripping makes the type forward-compatible with any future changes supported by the data contract versioning model.</span></span>  
  
 <span data-ttu-id="d07ea-112">Belirli bir tür için gidiş dönüşü etkinleştirmek için tür uygulamalıdır <xref:System.Runtime.Serialization.IExtensibleDataObject> arabirimi.</span><span class="sxs-lookup"><span data-stu-id="d07ea-112">To enable round-tripping for a particular type, the type must implement the <xref:System.Runtime.Serialization.IExtensibleDataObject> interface.</span></span> <span data-ttu-id="d07ea-113">Arabirim bir özellik içeren <xref:System.Runtime.Serialization.IExtensibleDataObject.ExtensionData%2A> (döndüren <xref:System.Runtime.Serialization.ExtensionDataObject> türü).</span><span class="sxs-lookup"><span data-stu-id="d07ea-113">The interface contains one property, <xref:System.Runtime.Serialization.IExtensibleDataObject.ExtensionData%2A> (returning the <xref:System.Runtime.Serialization.ExtensionDataObject> type).</span></span> <span data-ttu-id="d07ea-114">Özelliği, geçerli sürümü için bilinmeyen veri anlaşması'nın gelecek sürümlerinden herhangi bir veri depolar.</span><span class="sxs-lookup"><span data-stu-id="d07ea-114">The property stores any data from future versions of the data contract that is unknown to the current version.</span></span>

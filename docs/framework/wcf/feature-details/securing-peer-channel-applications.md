 <span data-ttu-id="39e0a-107">Eş kanal taşıma, ikisi de ayarını WMI'ya gerektiren güvenliğini sağlamak için iki tür kimlik doğrulama bilgilerini destekler `ClientCredentialSettings.Peer` özelliği ilişkili `ChannelFactory`:</span><span class="sxs-lookup"><span data-stu-id="39e0a-107">Peer Channel supports two types of authentication credentials for securing transport, both of which require setting out the `ClientCredentialSettings.Peer` property on the associated `ChannelFactory`:</span></span>  
  
- <span data-ttu-id="39e0a-108">Parola.</span><span class="sxs-lookup"><span data-stu-id="39e0a-108">Password.</span></span> <span data-ttu-id="39e0a-109">İstemciler, bağlantıların kimliğini doğrulamak için gizli bir parola bilgisini kullanır.</span><span class="sxs-lookup"><span data-stu-id="39e0a-109">Clients use knowledge of a secret password to authenticate connections.</span></span> <span data-ttu-id="39e0a-110">Bu kimlik bilgisi türü kullanıldığında `ClientCredentialSettings.Peer.MeshPassword` geçerli bir parola taşımalıdır ve isteğe bağlı olarak bir `X509Certificate2` örneği.</span><span class="sxs-lookup"><span data-stu-id="39e0a-110">When this credential type is used, `ClientCredentialSettings.Peer.MeshPassword` must carry a valid password and optionally an `X509Certificate2` instance.</span></span>  
  
- <span data-ttu-id="39e0a-111">Sertifika.</span><span class="sxs-lookup"><span data-stu-id="39e0a-111">Certificate.</span></span> <span data-ttu-id="39e0a-112">Belirli uygulama kimlik doğrulaması kullanılır.</span><span class="sxs-lookup"><span data-stu-id="39e0a-112">Specific application authentication is used.</span></span> <span data-ttu-id="39e0a-113">Bu kimlik bilgisi türü kullanıldığında, somut bir uygulama kullanmalısınız <xref:System.IdentityModel.Selectors.X509CertificateValidator> içinde `ClientCredentialSettings.Peer.PeerAuthentication`.</span><span class="sxs-lookup"><span data-stu-id="39e0a-113">When this credential type is used, you must use a concrete implementation of <xref:System.IdentityModel.Selectors.X509CertificateValidator> in `ClientCredentialSettings.Peer.PeerAuthentication`.</span></span>  
  
## <a name="message-based-security"></a><span data-ttu-id="39e0a-114">İleti tabanlı güvenlik</span><span class="sxs-lookup"><span data-stu-id="39e0a-114">Message-Based Security</span></span>

## <a name="best-practices"></a><span data-ttu-id="39e0a-117">En İyi Yöntemler</span><span class="sxs-lookup"><span data-stu-id="39e0a-117">Best Practices</span></span>  
  
- <span data-ttu-id="39e0a-118">Bu bölümde, eş kanalı uygulamalarını güvenli hale getirmek için en iyi uygulamalar ele alınmaktadır.</span><span class="sxs-lookup"><span data-stu-id="39e0a-118">This section discusses the best practices for securing Peer Channel applications.</span></span>  
  
### <a name="enable-security-with-peer-channel-applications"></a><span data-ttu-id="39e0a-119">Eş kanalı uygulamalarını ile güvenliği etkinleştirme</span><span class="sxs-lookup"><span data-stu-id="39e0a-119">Enable Security with Peer Channel Applications</span></span>  
 <span data-ttu-id="39e0a-120">Eş kanal protokoller dağıtılmış doğası nedeniyle kafes üyelik, gizlilik ve güvenli olmayan bir ağ gizlilik zorlamak güçtür.</span><span class="sxs-lookup"><span data-stu-id="39e0a-120">Due to the distributed nature of the Peer Channel protocols, it is hard to enforce mesh membership, confidentiality, and privacy in an unsecured mesh.</span></span> <span data-ttu-id="39e0a-121">İstemcilerle çözümleyicisini arasındaki iletişimin güvenliğini sağlamak unutmamak önemlidir.</span><span class="sxs-lookup"><span data-stu-id="39e0a-121">It is also important to remember to secure communication between clients and the resolver service.</span></span> <span data-ttu-id="39e0a-122">Eş Adı Çözümleme Protokolü (PNRP) altında kimlik sahtekarlığı önlemek için güvenli adları ve diğer yaygın saldırılardan kullanın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-122">Under Peer Name Resolution Protocol (PNRP), use secure names to avoid spoofing and other common attacks.</span></span> <span data-ttu-id="39e0a-123">Bir özel Çözücü hizmet bağlantısı istemcilerin kullandığı her iki ileti ve aktarım tabanlı güvenlik dahil olmak üzere çözümleyici hizmetiyle bağlantı güvenliği etkinleştirerek güvenli hale getirin.</span><span class="sxs-lookup"><span data-stu-id="39e0a-123">Secure a custom resolver service by enabling security on the connection clients use to contact the resolver service, including both message- and transport-based security.</span></span>  
  
### <a name="use-the-strongest-possible-security-model"></a><span data-ttu-id="39e0a-124">En güçlü olası bir güvenlik modeli kullanır</span><span class="sxs-lookup"><span data-stu-id="39e0a-124">Use the Strongest Possible Security Model</span></span>  
 <span data-ttu-id="39e0a-125">Kafes her üyesi ayrı ayrı tanımlanması gerekir, örneğin, sertifika tabanlı kimlik doğrulama modeli kullanın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-125">For example, if each member of the mesh needs to be individually identified, use certificate-based authentication model.</span></span> <span data-ttu-id="39e0a-126">Bu mümkün değilse, bunları güvenli tutmak için parola tabanlı kimlik doğrulaması aşağıdaki geçerli önerileri kullanın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-126">If that is not possible, use password-based authentication following current recommendations to keep them secure.</span></span> <span data-ttu-id="39e0a-127">Bu parola yalnızca güvenilir kişilere güvenli bir ortam kullanarak parolaları iletmesini durdurabilirsiniz, parolaları sık değişen ve parolaları güçlü olmalarını sağlamaya paylaşım içerir (en az sekiz karakter uzunluğunda olmalı, her iki durumda da bir rakam, en az bir harf içerir ve özel karakter).</span><span class="sxs-lookup"><span data-stu-id="39e0a-127">This includes sharing passwords only with trusted parties, transmitting passwords using a secure medium, changing passwords frequently, and ensuring that passwords are strong (at least eight characters long, include at least one letter from both cases, a digit, and a special character).</span></span>  
  
### <a name="never-accept-self-signed-certificates"></a><span data-ttu-id="39e0a-128">Hiçbir zaman otomatik olarak imzalanan sertifikaları kabul et</span><span class="sxs-lookup"><span data-stu-id="39e0a-128">Never Accept Self-Signed Certificates</span></span>  
 <span data-ttu-id="39e0a-129">Hiçbir zaman bir sertifika konu adlarına göre kimlik bilgilerini kabul edin.</span><span class="sxs-lookup"><span data-stu-id="39e0a-129">Never accept a certificate credential based on subject names.</span></span> <span data-ttu-id="39e0a-130">Herhangi bir sertifika oluşturabilir ve herkesin doğrulamakta olduğunuz bir ad seçin unutmayın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-130">Note that anyone can create a certificate, and anyone can choose a name that you are validating.</span></span> <span data-ttu-id="39e0a-131">Kimlik sahtekarlığı olasılığını önlemek için veren yetkili kimlik bilgileri (güvenilir verene veya bir kök sertifika yetkilisi) üzerinde temel alarak sertifikaları doğrulayın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-131">To avoid the possibility of spoofing, validate certificates based on issuing authority credentials (either a trusted issuer or a root certification authority).</span></span>  
  
### <a name="use-message-authentication"></a><span data-ttu-id="39e0a-132">İleti kimlik doğrulaması kullan</span><span class="sxs-lookup"><span data-stu-id="39e0a-132">Use Message Authentication</span></span>  
 <span data-ttu-id="39e0a-133">İleti kimlik doğrulaması, bir ileti güvenilir bir kaynaktan geldiğini ve hiç ileti aktarım sırasında değişiklik yapmadığından doğrulamak için kullanın.</span><span class="sxs-lookup"><span data-stu-id="39e0a-133">Use message authentication to verify that a message originated from a trusted source and that no one has tampered with the message during transmission.</span></span> <span data-ttu-id="39e0a-134">İleti kimlik doğrulaması olmadan, sızmasını veya kafes iletilerinde değiştirmesine kötü amaçlı bir istemci için kolaydır.</span><span class="sxs-lookup"><span data-stu-id="39e0a-134">Without message authentication, it is easy for a malicious client to spoof or tamper with messages in the mesh.</span></span>

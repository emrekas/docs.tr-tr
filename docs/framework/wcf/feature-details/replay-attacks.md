# <a name="replay-attacks"></a><span data-ttu-id="0bae8-102">Yeniden Yürütme Saldırıları</span><span class="sxs-lookup"><span data-stu-id="0bae8-102">Replay Attacks</span></span>
<span data-ttu-id="0bae8-103">A *tekrarlama saldırı* bir saldırgan, iki taraflar arasında iletileri akışını kopyalar ve bir veya daha fazla taraflar akışa başlayarak yeniden oynatılır oluşur.</span><span class="sxs-lookup"><span data-stu-id="0bae8-103">A *replay attack* occurs when an attacker copies a stream of messages between two parties and replays the stream to one or more of the parties.</span></span> <span data-ttu-id="0bae8-104">Saldırı bilgisayarların akış azaltılabilir sürece, sonuçta hatalı sonuçları, bir öğenin yedekli siparişler gibi çeşitli yasal iletileri olarak işleyin.</span><span class="sxs-lookup"><span data-stu-id="0bae8-104">Unless mitigated, the computers subject to the attack process the stream as legitimate messages, resulting in a range of bad consequences, such as redundant orders of an item.</span></span>  
  
## <a name="bindings-may-be-subject-to-reflection-attacks"></a><span data-ttu-id="0bae8-105">Bağlamaları yansıma saldırılarıyla karşılaşabilir</span><span class="sxs-lookup"><span data-stu-id="0bae8-105">Bindings May Be Subject to Reflection Attacks</span></span>  
 <span data-ttu-id="0bae8-106">*Yansıma saldırıları* yanıt olarak alıcı geldiği gibi iletileri gönderen dön olumsuzlukları demektir.</span><span class="sxs-lookup"><span data-stu-id="0bae8-106">*Reflection attacks* are replays of messages back to a sender as if they came from the receiver as the reply.</span></span> <span data-ttu-id="0bae8-107">Standart *yeniden yürütme algılaması* Windows Communication Foundation (WCF) mekanizması otomatik olarak bu işlemez.</span><span class="sxs-lookup"><span data-stu-id="0bae8-107">The standard *replay detection* in the Windows Communication Foundation (WCF) mechanism does not automatically handle this.</span></span>  
  
 <span data-ttu-id="0bae8-108">Yansıma saldırıları WCF service model imzalı ileti kimliği, istek iletilerini ekler ve bir imzalı bekliyor çünkü varsayılan olarak azaltılabilir `relates-to` üstbilgisi yanıt iletileri.</span><span class="sxs-lookup"><span data-stu-id="0bae8-108">Reflection attacks are mitigated by default because the WCF service model adds a signed message ID to request messages and expects a signed `relates-to` header on response messages.</span></span> <span data-ttu-id="0bae8-109">Sonuç olarak, isteğine yanıt olarak yürütülemez.</span><span class="sxs-lookup"><span data-stu-id="0bae8-109">Consequently, the request message cannot be replayed as a response.</span></span> <span data-ttu-id="0bae8-110">Güvenli, güvenilir, ileti (RM) senaryolarında, çünkü yansıma saldırıları azaltıldığından:</span><span class="sxs-lookup"><span data-stu-id="0bae8-110">In secure reliable message (RM) scenarios, reflection attacks are mitigated because:</span></span>  
  
- <span data-ttu-id="0bae8-111">Oluşturma sırası yanıt iletisi şemaları ve Oluşturma sırası farklıdır.</span><span class="sxs-lookup"><span data-stu-id="0bae8-111">The create sequence and create sequence response message schemas are different.</span></span>  
  
- <span data-ttu-id="0bae8-112">İstemci bu türden iletilere anlayamıyor çünkü tek yönlü sıraları için istemcinin gönderdiği sırası iletileri yeniden yürütülemez.</span><span class="sxs-lookup"><span data-stu-id="0bae8-112">For simplex sequences, sequence messages the client sends cannot be replayed back to it because the client cannot understand such messages.</span></span>  
  
- <span data-ttu-id="0bae8-113">Çift yönlü dizileri için iki sıranın kimlikleri benzersiz olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="0bae8-113">For duplex sequences, the two sequence IDs must be unique.</span></span> <span data-ttu-id="0bae8-114">Bu nedenle, bir giden dizisi iletisi (tüm dizisi üstbilgileri gövdeleri, çok oturum açtığınızı) geri gelen sırası iletiyi yürütülemez.</span><span class="sxs-lookup"><span data-stu-id="0bae8-114">Thus, an outgoing sequence message cannot be replayed back as an incoming sequence message (all sequence headers and bodies are signed, too).</span></span>  
  
 <span data-ttu-id="0bae8-115">Yansıma saldırılarına yalnızca bağlamaları WS-Addressing olmadan olanlardır: simetrik anahtar tabanlı güvenliği kullanın-devre dışı WS Addressing ve özel bağlamalar.</span><span class="sxs-lookup"><span data-stu-id="0bae8-115">The only bindings that are susceptible to reflection attacks are those without WS-Addressing: custom bindings that have WS-Addressing disabled and use the symmetric key-based security.</span></span> <span data-ttu-id="0bae8-116"><xref:System.ServiceModel.BasicHttpBinding> Kullanılmıyor WS-Addressing varsayılan olarak yapar, ancak bunları simetrik anahtar tabanlı güvenlik, bu saldırısına açık olmasını sağlayan bir yolla kullanmaz.</span><span class="sxs-lookup"><span data-stu-id="0bae8-116">The <xref:System.ServiceModel.BasicHttpBinding> does not use WS-Addressing by default, but it does not use symmetric key-based security in a way that allows it to be vulnerable to this attack.</span></span>  
  
 <span data-ttu-id="0bae8-117">Risk azaltma özel bağlamalar için güvenlik bağlamı kurmak değil veya WS-Addressing üst bilgileri gerektirecek şekilde kullanmaktır.</span><span class="sxs-lookup"><span data-stu-id="0bae8-117">The mitigation for custom bindings is to not establish security context or to require WS-Addressing headers.</span></span>  
  
## <a name="web-farm-attacker-replays-request-to-multiple-nodes"></a><span data-ttu-id="0bae8-118">Web grubu: Saldırgan olumsuzlukları isteği birden çok düğüm</span><span class="sxs-lookup"><span data-stu-id="0bae8-118">Web Farm: Attacker Replays Request to Multiple Nodes</span></span>  
 <span data-ttu-id="0bae8-119">Bir istemci, bir Web grubunda uygulanan bir hizmeti kullanır.</span><span class="sxs-lookup"><span data-stu-id="0bae8-119">A client uses a service that is implemented on a Web farm.</span></span> <span data-ttu-id="0bae8-120">Bir saldırganın bir düğüm grubunda gruptaki başka bir düğüme gönderilen bir istek başlayarak yeniden oynatılır.</span><span class="sxs-lookup"><span data-stu-id="0bae8-120">An attacker replays a request that was sent to one node in the farm to another node in the farm.</span></span> <span data-ttu-id="0bae8-121">Hizmet yeniden başlatılırsa, ayrıca, yeniden yürütme önbellek, saldırganın isteği yeniden yürütme temizlenir.</span><span class="sxs-lookup"><span data-stu-id="0bae8-121">In addition, if a service is restarted, the replay cache is flushed, allowing an attacker to replay the request.</span></span> <span data-ttu-id="0bae8-122">(Önbellek kullanılan, daha önce görülen ileti imzası değerlerini içerir ve bu nedenle bu imzaların yalnızca bir kez kullanılabilir olumsuzlukları önler.</span><span class="sxs-lookup"><span data-stu-id="0bae8-122">(The cache contains used, previously seen message signature values and prevents replays so those signatures can be used only once.</span></span> <span data-ttu-id="0bae8-123">Yeniden yürütme önbellekler bir Web grubu arasında paylaşılmaz.)</span><span class="sxs-lookup"><span data-stu-id="0bae8-123">Replay caches are not shared across a Web farm.)</span></span>  
  
 <span data-ttu-id="0bae8-124">Risk azaltma işlemleri şunlardır:</span><span class="sxs-lookup"><span data-stu-id="0bae8-124">Mitigations include:</span></span>  
  
- <span data-ttu-id="0bae8-125">İleti modu güvenlik durum bilgisi olan güvenlik bağlamı belirteçleri (ile veya etkin güvenli konuşma olmadan) kullanın.</span><span class="sxs-lookup"><span data-stu-id="0bae8-125">Use message mode security with stateful security context tokens (with or without secure conversation enabled).</span></span> <span data-ttu-id="0bae8-126">Daha fazla bilgi için [nasıl yapılır: Bir güvenlik bağlamı oluşturmak için güvenli bir oturum belirteci](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span><span class="sxs-lookup"><span data-stu-id="0bae8-126">For more information, see [How to: Create a Security Context Token for a Secure Session](../../../../docs/framework/wcf/feature-details/how-to-create-a-security-context-token-for-a-secure-session.md).</span></span>  
  
- <span data-ttu-id="0bae8-127">Hizmetini aktarım düzeyi güvenlik kullanacak şekilde yapılandırın.</span><span class="sxs-lookup"><span data-stu-id="0bae8-127">Configure the service to use transport-level security.</span></span>

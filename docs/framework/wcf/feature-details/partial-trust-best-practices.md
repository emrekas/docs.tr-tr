## <a name="using-wcf-from-fully-trusted-platform-code-that-allows-calls-from-partially-trusted-callers"></a><span data-ttu-id="f38dc-126">WCF çağrılarından kısmen sağlayan tam olarak güvenilen koddan kullanarak güvenilen Arayanların</span><span class="sxs-lookup"><span data-stu-id="f38dc-126">Using WCF from Fully-Trusted Platform Code that Allows Calls from Partially Trusted Callers</span></span>

<span data-ttu-id="f38dc-127">WCF kısmi güven güvenliği modeli, herhangi bir WCF genel yöntem veya özellik çağıran kod erişim güvenliği (CAS) barındırma uygulama bağlamında çalıştığını varsayar.</span><span class="sxs-lookup"><span data-stu-id="f38dc-127">The WCF partial trust security model assumes that any caller of a WCF public method or property is running in the code access security (CAS) context of the hosting application.</span></span> <span data-ttu-id="f38dc-128">WCF ayrıca varsayar, yalnızca bir uygulama güvenlik bağlamı her biri için mevcut <xref:System.AppDomain>, ve bu bağlam, belirlenen <xref:System.AppDomain> oluşturma zamanı tarafından güvenilen bir konak (örneğin, bir çağrı tarafından <xref:System.AppDomain.CreateDomain%2A> veya ASP.NET Uygulama Yöneticisi tarafından).</span><span class="sxs-lookup"><span data-stu-id="f38dc-128">WCF also assumes that only one application security context exists for each <xref:System.AppDomain>, and that this context is established at <xref:System.AppDomain> creation time by a trusted host (for example, by a call to <xref:System.AppDomain.CreateDomain%2A> or by the ASP.NET Application Manager).</span></span>

<span data-ttu-id="f38dc-129">Bu güvenlik modeli Orta güven ASP.NET uygulama içinde çalışan kullanıcı kodu gibi ek CAS izinler assert olamaz, kullanıcı tarafından yazılan uygulamalar için geçerlidir.</span><span class="sxs-lookup"><span data-stu-id="f38dc-129">This security model applies to user-written applications that cannot assert additional CAS permissions, such as user code running in a Medium Trust ASP.NET application.</span></span> <span data-ttu-id="f38dc-130">Ancak, tam olarak güvenilen kod (örneğin, genel derleme önbelleğinde yüklü ve kısmen güvenilen koddan çağrıları kabul eden bir üçüncü taraf derleme) açık WCF kısmen güvenilen bir uygulama adına çağırırken dikkatli gerekir uygulama düzeyinde güvenlik açıklarını oluşturmaktan kaçının.</span><span class="sxs-lookup"><span data-stu-id="f38dc-130">However, fully-trusted platform code (for example, a third-party assembly that is installed in the global assembly cache and accepts calls from partially-trusted code) must take explicit care when calling into WCF on behalf of a partially-trusted application to avoid introducing application-level security vulnerabilities.</span></span>

<span data-ttu-id="f38dc-131">Tam güven kodu önlemek geçerli iş parçacığının CA izin kümesi değiştirme (çağırarak <xref:System.Security.PermissionSet.Assert%2A>, <xref:System.Security.PermissionSet.PermitOnly%2A>, veya <xref:System.Security.PermissionSet.Deny%2A>) kısmen güvenilen kod adına WCF API'larını çağırma önce.</span><span class="sxs-lookup"><span data-stu-id="f38dc-131">Full-trust code should avoid altering the CAS permission set of the current thread (by calling <xref:System.Security.PermissionSet.Assert%2A>, <xref:System.Security.PermissionSet.PermitOnly%2A>, or <xref:System.Security.PermissionSet.Deny%2A>) prior to calling WCF APIs on behalf of partially-trusted code.</span></span> <span data-ttu-id="f38dc-132">Sunduğundan, reddetme ya da aksi takdirde uygulama düzeyinde güvenlik bağlamı bağımsız bir iş parçacığına özgü izni bağlam oluşturma, beklenmeyen davranışlara neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="f38dc-132">Asserting, denying, or otherwise creating a thread-specific permission context that is independent of the application-level security context can result in unexpected behavior.</span></span> <span data-ttu-id="f38dc-133">Uygulamaya bağlı olarak, bu davranış, uygulama düzeyinde güvenlik açıklarına neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="f38dc-133">Depending on the application, this behavior may result in application-level security vulnerabilities.</span></span>

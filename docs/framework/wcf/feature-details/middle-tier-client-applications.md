# <a name="middle-tier-client-applications"></a><span data-ttu-id="cf2c8-102">Orta Katman İstemci Uygulamaları</span><span class="sxs-lookup"><span data-stu-id="cf2c8-102">Middle-Tier Client Applications</span></span>
<span data-ttu-id="cf2c8-103">Bu konu, Windows Communication Foundation (WCF) kullanan Orta katman istemci uygulamaları için belirli çeşitli sorunlar ele alınmıştır.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-103">This topic discusses various issues specific to middle-tier client applications that use Windows Communication Foundation (WCF).</span></span>  
  
## <a name="increasing-middle-tier-client-performance"></a><span data-ttu-id="cf2c8-104">Orta katman istemci performansını artırma</span><span class="sxs-lookup"><span data-stu-id="cf2c8-104">Increasing Middle-Tier Client Performance</span></span>  
 <span data-ttu-id="cf2c8-105">Kullanarak Web Hizmetleri gibi önceki iletişimi teknolojilerine kıyasla [!INCLUDE[vstecasp](../../../../includes/vstecasp-md.md)], bir WCF istemcisi örneği oluşturulmasını WCF zengin özellik kümesi nedeniyle daha karmaşık olabilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-105">Compared to previous communications technologies, such as Web services using [!INCLUDE[vstecasp](../../../../includes/vstecasp-md.md)], the creation of a WCF client instance can be more complex due to the rich feature set of WCF.</span></span> <span data-ttu-id="cf2c8-106">Örneğin, bir <xref:System.ServiceModel.ChannelFactory%601> nesnesinin açıldığı güvenli bir oturum hizmetiyle istemci örneği için başlangıç süresini artıran bir yordam oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-106">For example, when a <xref:System.ServiceModel.ChannelFactory%601> object is opened it can establish a secure session with the service, a procedure that increases the startup time for the client instance.</span></span> <span data-ttu-id="cf2c8-107">WCF istemcisini çeşitli çağrılar yapar ve ardından kapatır genellikle, bu ek özellik yetenekleri istemci uygulamaları önemli ölçüde etkilemez.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-107">Typically, these additional feature capabilities do not affect client applications greatly since the WCF client makes several calls, and then closes.</span></span>  
  
 <span data-ttu-id="cf2c8-108">Orta katman istemci uygulamaları, ancak birçok WCF istemci nesne hızlı bir şekilde oluşturabilir ve, sonuç olarak, artan başlatma gereksinimleri karşılaşırsınız.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-108">Middle-tier client applications, however, can create many WCF client objects quickly and, as a result, experience increased initialization requirements.</span></span> <span data-ttu-id="cf2c8-109">Hizmetleri çağırırken orta katman uygulamalarının performansını artırmak için iki ana yaklaşım vardır:</span><span class="sxs-lookup"><span data-stu-id="cf2c8-109">There are two main approaches to increasing the performance of middle-tier applications when calling services:</span></span>  
  
- <span data-ttu-id="cf2c8-110">WCF istemci nesnesini önbelleğe ve mümkün olduğunda sonraki çağrılar için indirilebilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-110">Cache the WCF client object and reuse it for subsequent calls where possible.</span></span>  
  
- <span data-ttu-id="cf2c8-111">Oluşturma bir <xref:System.ServiceModel.ChannelFactory%601> nesne ve o nesnenin kanal nesneler her çağrı için yeni bir WCF istemcisi oluşturmak için kullanın.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-111">Create a <xref:System.ServiceModel.ChannelFactory%601> object and then use that object to create new WCF client channel objects for each call.</span></span>  
  
 <span data-ttu-id="cf2c8-112">Bu yaklaşımların kullanırken dikkat etmeniz gereken sorunlar şunlardır:</span><span class="sxs-lookup"><span data-stu-id="cf2c8-112">Issues to consider when using these approaches include:</span></span>  
  
- <span data-ttu-id="cf2c8-113">Hizmet istemci özel durumu bir oturumu kullanarak koruma, hizmet durumu, orta katman istemci bağlı olmadığından birden çok istemci katmanı isteklerle orta katman WCF istemcisini yeniden kullanamazsınız.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-113">If the service is maintaining a client-specific state by using a session, then you cannot reuse the middle-tier WCF client with multiple-client tier requests because the service's state is tied to that of the middle-tier client.</span></span>  
  
- <span data-ttu-id="cf2c8-114">Hizmet istemci başına temelinde kimlik doğrulaması gerçekleştirmesi gerekiyorsa, çünkü orta katman WCF istemcisi (veya WCF istemci kanal nesnesi) yeniden kullanmak yerine orta katman her gelen istek için yeni bir istemci oluşturmalısınız orta katman istemci kimlik bilgileri WCF istemcisi sonra değiştirilemez (veya <xref:System.ServiceModel.ChannelFactory%601>) oluşturuldu.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-114">If the service must perform authentication on a per-client basis, you must create a new client for each incoming request on the middle tier instead of reusing the middle-tier WCF client (or WCF client channel object) because the client credentials of the middle tier cannot be modified after the WCF client (or <xref:System.ServiceModel.ChannelFactory%601>) has been created.</span></span>  
  
- <span data-ttu-id="cf2c8-115">Kanallar ve istemcilerin kanalları tarafından oluşturulan iş parçacığı açısından güvenli olsa da, bunlar için kablo, eşzamanlı olarak birden fazla ileti yazma desteklemiyor olabilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-115">While channels and clients created by the channels are thread-safe, they might not support writing more than one message to the wire concurrently.</span></span> <span data-ttu-id="cf2c8-116">Büyük iletileri gönderiyorsanız, özellikle akış, gönderme işlemi tamamlamak başka bir göndermek için bekleme engelleyebilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-116">If you are sending large messages, particularly if streaming, the send operation might block waiting for another send to complete.</span></span> <span data-ttu-id="cf2c8-117">Bu iki tür sorunlara neden olur: bir eşzamanlılık ve denetim akışını kanal yeniden hizmete döndürürse kilitlenme olasılığını eksikliği (diğer bir deyişle, paylaşılan istemci bir hizmet olan kod yolu sonuçları paylaşılan istemciye bir geri çağırma içinde çağırır).</span><span class="sxs-lookup"><span data-stu-id="cf2c8-117">This causes two sorts of problems: a lack of concurrency and the possibility of deadlock if the flow of control returns to the service reusing the channel (that is, the shared client calls a service whose code path results in a callback to the shared client).</span></span> <span data-ttu-id="cf2c8-118">Bu yeniden WCF istemci türü ne olursa olsun doğrudur.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-118">This is true regardless of the type of WCF client you reuse.</span></span>  
  
- <span data-ttu-id="cf2c8-119">Kanal paylaştığınız bağımsız olarak hatalı kanal işlemesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-119">You must handle faulted channels regardless of whether you share the channel.</span></span> <span data-ttu-id="cf2c8-120">Kanalları yeniden kullanılır, ancak hatalı kanal birden fazla bekleyen istek aşağı olması ya da gönderebilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-120">When channels are reused, however, a faulting channel can take down more than one pending request or send.</span></span>  
  
 <span data-ttu-id="cf2c8-121">Bir istemci birden çok istek için yeniden kullanmak için en iyi yöntemleri gösteren bir örnek için bkz: [bir ASP.NET istemcisinde veri bağlama](../../../../docs/framework/wcf/samples/data-binding-in-an-aspnet-client.md).</span><span class="sxs-lookup"><span data-stu-id="cf2c8-121">For an example that demonstrates best practices for reusing a client for multiple requests, see [Data Binding in an ASP.NET Client](../../../../docs/framework/wcf/samples/data-binding-in-an-aspnet-client.md).</span></span>  
  
 <span data-ttu-id="cf2c8-122">Ayrıca, seri hale getirilebilir kullanarak veri türleri kullanan bu istemciler için başlatma performansını artırabilir <xref:System.Xml.Serialization.XmlSerializer> oluşturmak ve yavaş başlatma performansı düşürebilir çalışma zamanında bu veri türleri için serileştirme kodu derleyin.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-122">In addition, you can increase the startup performance for those clients that use data types that are serializable using the <xref:System.Xml.Serialization.XmlSerializer> generate and compile serialization code for those data types at runtime, which can result in slow start-up performance.</span></span> <span data-ttu-id="cf2c8-123">[ServiceModel meta veri yardımcı Programracı (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) derlenmiş bütünleştirilmiş uygulama için gerekli serileştirme kod oluşturarak bu uygulamaları için başlatma performansını geliştirebilir.</span><span class="sxs-lookup"><span data-stu-id="cf2c8-123">The [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) can improve start-up performance for these applications by generating the necessary serialization code from the compiled assemblies for the application.</span></span> <span data-ttu-id="cf2c8-124">Daha fazla bilgi için [nasıl yapılır: Başlangıç zamanı, istemci XmlSerializer kullanarak WCF uygulamalarının geliştirilmesine](../../../../docs/framework/wcf/feature-details/startup-time-of-wcf-client-applications-using-the-xmlserializer.md).</span><span class="sxs-lookup"><span data-stu-id="cf2c8-124">For more information, see [How to: Improve the Startup Time of WCF Client Applications using the XmlSerializer](../../../../docs/framework/wcf/feature-details/startup-time-of-wcf-client-applications-using-the-xmlserializer.md).</span></span>

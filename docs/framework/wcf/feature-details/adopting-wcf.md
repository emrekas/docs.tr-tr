# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="8ace1-102">Windows Communication Foundation'ı Benimseme</span><span class="sxs-lookup"><span data-stu-id="8ace1-102">Adopting Windows Communication Foundation</span></span>

<span data-ttu-id="8ace1-103">ASP.NET kullanılarak geliştirilen mevcut uygulamaları korumak devam ederken, yeni geliştirme projeleri için Windows Communication Foundation (WCF) kullanmayı da tercih edebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="8ace1-104">WCF hiçbir senaryoda .NET Framework ile oluşturulan uygulamalar ile iletişimi etkinleştirme için en uygun seçeneği olacak şekilde tasarlandığından, çok çeşitli yazılım bir şekilde iletişim sorunlarını çözmek için standart bir araç ASP.NET görebilir uygulanamaz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="8ace1-105">Yeni WCF uygulamaları mevcut ASP.NET Web Hizmetleri olarak aynı makinede dağıtılabilir.</span><span class="sxs-lookup"><span data-stu-id="8ace1-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="8ace1-106">Mevcut ASP.NET Web Hizmetleri .NET Framework 2.0 sürümünden önceki bir sürümünü kullanıyorsanız, seçmeli olarak .NET Framework 2.0, barındırılması için yeni WCF uygulamaları olduğu IIS uygulama dağıtmak için ASP.NET IIS Kayıt aracını kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="8ace1-107">Bu aracı adresinde belgelenen [ASP.NET IIS Kayıt Aracı (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), ve IIS 6.0 yönetim konsoluna oluşturduğu bir kullanıcı arabirimi.</span><span class="sxs-lookup"><span data-stu-id="8ace1-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](https://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="8ace1-108">WCF, IIS mevcut ASP.NET Web hizmeti uygulama için ASP.NET uyumluluk modunda çalışacak şekilde yapılandırılmış WCF hizmetlerini ekleyerek mevcut bir ASP.NET Web Hizmetleri için yeni özellikler eklemek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="8ace1-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="8ace1-109">ASP.NET uyumluluk modunun nedeniyle yeni WCF hizmetleri için kod erişebilir ve aynı uygulama durumu bilgileri önceden mevcut olan ASP.NET kodunu kullanarak güncelleştirme <xref:System.Web.HttpContext> sınıfı.</span><span class="sxs-lookup"><span data-stu-id="8ace1-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="8ace1-110">Uygulamalar Ayrıca aynı sınıf kitaplıkları paylaşabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-110">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="8ace1-111">WCF istemcileri, ASP.NET Web hizmetlerini kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="8ace1-112">İle yapılandırılan WCF hizmetleri <xref:System.ServiceModel.BasicHttpBinding> ASP.NET Web hizmeti istemcileri tarafından kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="8ace1-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="8ace1-113">ASP.NET Web Hizmetleri ile WCF uygulamaları birlikte bulunabilir ve WCF bile var olan ASP.NET Web hizmetlerini özellikleri eklemek için kullanılabilir.</span><span class="sxs-lookup"><span data-stu-id="8ace1-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="8ace1-114">Tüm yolların, WCF ve ASP.NET Web hizmetlerini birlikte kullanılabilmesi için göz önünde bulundurulduğunda, WCF ve ASP.NET Web Hizmetleri tarafından sağlanan özellikler gerektiriyorsa, ASP.NET Web hizmetlerini WCF'ye taşıma isteyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="8ace1-115">Gerekli olduğu bazı durumlarda bile, bir teknoloji geçirme koddan diğerine nadiren doğru bir yaklaşımdır.</span><span class="sxs-lookup"><span data-stu-id="8ace1-115">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="8ace1-116">Yeni Teknoloji benimseme nedeni önceki teknolojisiyle karşılanamıyor yeni gereksinimlerini karşılamak için ve bu durumda, doğru bir şey yapmak için yeni Genişletilmiş gereksinimlerini karşılaması için yeni bir çözüm tasarlamak için.</span><span class="sxs-lookup"><span data-stu-id="8ace1-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="8ace1-117">Bu sistem tasarlandığından beri yeni tasarım avantajları bilgilerini ve mevcut sistem deneyiminizi kavuştu.</span><span class="sxs-lookup"><span data-stu-id="8ace1-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="8ace1-118">Yeni tasarım özelliklerinin tamamı yeni platformunda eski tasarım yeniden yerine yeni teknolojileri de kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="8ace1-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="8ace1-119">Prototip oluşturma anahtar yeni tasarım öğeleri sonra kod yeniden kullanımı için yeni bir tane var olan sistemden daha kolay olur.</span><span class="sxs-lookup"><span data-stu-id="8ace1-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>

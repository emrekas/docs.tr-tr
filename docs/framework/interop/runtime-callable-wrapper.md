# <a name="runtime-callable-wrapper"></a><span data-ttu-id="6d83f-102">Çalışma Zamanı Aranabilir Sarmalayıcısı</span><span class="sxs-lookup"><span data-stu-id="6d83f-102">Runtime Callable Wrapper</span></span>
<span data-ttu-id="6d83f-103">Ortak dil çalışma zamanı, COM nesneleri çalışma zamanı çağrılabilir sarmalayıcı (RCW) adlı bir ara sunucu aracılığıyla kullanıma sunar.</span><span class="sxs-lookup"><span data-stu-id="6d83f-103">The common language runtime exposes COM objects through a proxy called the runtime callable wrapper (RCW).</span></span> <span data-ttu-id="6d83f-104">RCW, .NET istemcileri için normal bir nesne görünmesine karşın, birincil bir .NET istemcisi ve COM nesnesi arasındaki çağrıların sıralamakta işlevidir.</span><span class="sxs-lookup"><span data-stu-id="6d83f-104">Although the RCW appears to be an ordinary object to .NET clients, its primary function is to marshal calls between a .NET client and a COM object.</span></span>  
  
 <span data-ttu-id="6d83f-105">Çalışma zamanı, nesne üzerinde mevcut başvuruları sayısından bağımsız olarak her bir COM nesnesi için tam olarak bir RCW oluşturur.</span><span class="sxs-lookup"><span data-stu-id="6d83f-105">The runtime creates exactly one RCW for each COM object, regardless of the number of references that exist on that object.</span></span> <span data-ttu-id="6d83f-106">Çalışma zamanı, her nesne için işlem başına tek bir RCW tutar.</span><span class="sxs-lookup"><span data-stu-id="6d83f-106">The runtime maintains a single RCW per process for each object.</span></span>  <span data-ttu-id="6d83f-107">Bir uygulama etki alanı veya grup bir RCW oluşturun ve ardından başka bir uygulama etki alanı ya da grup bir başvuru geçirin, ilk nesneye bir ara sunucu kullanılır.</span><span class="sxs-lookup"><span data-stu-id="6d83f-107">If you create an RCW in one application domain or apartment, and then pass a reference to another application domain or apartment, a proxy to the first object will be used.</span></span>  <span data-ttu-id="6d83f-108">Aşağıdaki çizimde gösterildiği gibi yönetilen istemcilerin herhangi bir sayıda INew ve INewer arabirimler sunun COM nesnelerine başvuru barındırabilir.</span><span class="sxs-lookup"><span data-stu-id="6d83f-108">As the following illustration shows, any number of managed clients can hold a reference to the COM objects that expose INew and INewer interfaces.</span></span>  

<span data-ttu-id="6d83f-109">Aşağıdaki görüntüde, çalışma zamanı çağrılabilir sarmalayıcı COM nesnelerine erişme işlemi gösterilmektedir:</span><span class="sxs-lookup"><span data-stu-id="6d83f-109">The following image shows the process for accessing COM objects through the runtime callable wrapper:</span></span>

 ![COM nesneleri throug RCW erişme işlemi.](./media/runtime-callable-wrapper/runtime-callable-wrapper.gif)  

 <span data-ttu-id="6d83f-111">Türetilmiş bir tür kitaplığından meta verileri kullanarak, çalışma zamanı hem çağrılan COM nesnesi hem de bu nesne için bir sarmalayıcı oluşturur.</span><span class="sxs-lookup"><span data-stu-id="6d83f-111">Using metadata derived from a type library, the runtime creates both the COM object being called and a wrapper for that object.</span></span> <span data-ttu-id="6d83f-112">Her RCW arabirim işaretçilerini sarmalar ve RCW artık gerekli olmadığında, başvuru COM nesnesi üzerinde serbest COM nesne üzerinde bir önbelleğini korur.</span><span class="sxs-lookup"><span data-stu-id="6d83f-112">Each RCW maintains a cache of interface pointers on the COM object it wraps and releases its reference on the COM object when the RCW is no longer needed.</span></span> <span data-ttu-id="6d83f-113">Çalışma zamanı üzerinde RCW atık toplama gerçekleştirir.</span><span class="sxs-lookup"><span data-stu-id="6d83f-113">The runtime performs garbage collection on the RCW.</span></span>  
  
 <span data-ttu-id="6d83f-114">Diğer etkinlikler arasında RCW Sarmalanan nesne adına yönetilen ve yönetilmeyen kod arasında veri sürekliliğe devreder.</span><span class="sxs-lookup"><span data-stu-id="6d83f-114">Among other activities, the RCW marshals data between managed and unmanaged code, on behalf of the wrapped object.</span></span> <span data-ttu-id="6d83f-115">Özellikle, RCW, istemci ve sunucu, aralarında aktarılan verilerin farklı temsilleri olabilir her yöntem bağımsız değişkenleri ve yöntem dönüş değerleri için sıralama sağlar.</span><span class="sxs-lookup"><span data-stu-id="6d83f-115">Specifically, the RCW provides marshaling for method arguments and method return values whenever the client and server have different representations of the data passed between them.</span></span>  
  
 <span data-ttu-id="6d83f-116">Standart sarmalayıcı yerleşik sıralama kuralları zorlar.</span><span class="sxs-lookup"><span data-stu-id="6d83f-116">The standard wrapper enforces built-in marshaling rules.</span></span> <span data-ttu-id="6d83f-117">Örneğin, bir .NET istemci yönetilmeyen bir nesneye bağımsız değişken bir parçası olarak bir dize türü geçtiğinde sarmalayıcı dize bir BSTR türüne dönüştürür.</span><span class="sxs-lookup"><span data-stu-id="6d83f-117">For example, when a .NET client passes a String type as part of an argument to an unmanaged object, the wrapper converts the string to a BSTR type.</span></span> <span data-ttu-id="6d83f-118">COM nesnesinin yönetilen arayanına BSTR döndürmelidir, çağıran bir dize alır.</span><span class="sxs-lookup"><span data-stu-id="6d83f-118">Should the COM object return a BSTR to its managed caller, the caller receives a String.</span></span> <span data-ttu-id="6d83f-119">İstemci ve sunucu gönderin ve onlara alışkın olduğu veri alın.</span><span class="sxs-lookup"><span data-stu-id="6d83f-119">Both the client and the server send and receive data that is familiar to them.</span></span> <span data-ttu-id="6d83f-120">Diğer türleri dönüştürme gerektirir.</span><span class="sxs-lookup"><span data-stu-id="6d83f-120">Other types require no conversion.</span></span> <span data-ttu-id="6d83f-121">Örneğin, standart bir sarmalayıcı türü dönüştürme olmadan yönetilen ve yönetilmeyen kod arasındaki her zaman bir 4 baytlık tamsayı geçer.</span><span class="sxs-lookup"><span data-stu-id="6d83f-121">For instance, a standard wrapper will always pass a 4-byte integer between managed and unmanaged code without converting the type.</span></span>

## <a name="what-was-the-app-doing"></a><span data-ttu-id="0dcd9-109">Uygulama neler oldu?</span><span class="sxs-lookup"><span data-stu-id="0dcd9-109">What was the app doing?</span></span>

 <span data-ttu-id="0dcd9-114">Bu örnekte, `LayoutVM` özelliği `AppViewModel.Current` olduğu **null**.</span><span class="sxs-lookup"><span data-stu-id="0dcd9-114">In this instance, the `LayoutVM` property on `AppViewModel.Current` was **null**.</span></span>  <span data-ttu-id="0dcd9-115">Meta verileri bazı olmaması, hafif bir davranışı fark neden ve yerine küme, beklenen uygulaması olarak başlatılmamış bir özellikte sonuçlandı.</span><span class="sxs-lookup"><span data-stu-id="0dcd9-115">Some absence of metadata caused a subtle behavior difference and resulted in a property being uninitialized instead of set, as the app expected.</span></span>  <span data-ttu-id="0dcd9-116">Kodda bir kesme noktası ayarlama burada `LayoutVM` başlatılmamış ışık durumunuza fırlatabilir.</span><span class="sxs-lookup"><span data-stu-id="0dcd9-116">Setting a breakpoint in the code where `LayoutVM` should have been initialized might throw light on the situation.</span></span>  <span data-ttu-id="0dcd9-117">Ancak, unutmayın `LayoutVM`ın türü `App.Core.ViewModels.Layout.LayoutApplicationVM`.</span><span class="sxs-lookup"><span data-stu-id="0dcd9-117">However, note that `LayoutVM`’s type is `App.Core.ViewModels.Layout.LayoutApplicationVM`.</span></span>  <span data-ttu-id="0dcd9-118">Şu ana kadar rd.xml dosyasında mevcut yalnızca meta veri yönergesi verilmiştir:</span><span class="sxs-lookup"><span data-stu-id="0dcd9-118">The only metadata directive present so far in the rd.xml file is:</span></span>

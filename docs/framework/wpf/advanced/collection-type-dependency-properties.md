<span data-ttu-id="b90e1-103">Bu konuda, rehberlik ve özellik türü bir koleksiyon türü olduğu bir bağımlılık özelliği uygulama öğrenmek için önerilen desenler sağlar.</span><span class="sxs-lookup"><span data-stu-id="b90e1-103">This topic provides guidance and suggested patterns for how to implement a dependency property where the type of the property is a collection type.</span></span>

## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="b90e1-104">Koleksiyon türü bağımlılık özelliği uygulama</span><span class="sxs-lookup"><span data-stu-id="b90e1-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="b90e1-105">Genel olarak, bir bağımlılık özelliği için tanımladığınız takip ettiğiniz uygulama düzeni olan bir [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] özelliği sarmalayıcı, burada bu özellik destekli bir <xref:System.Windows.DependencyProperty> tanımlayıcısı yerine bir alan veya diğer yapı.</span><span class="sxs-lookup"><span data-stu-id="b90e1-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="b90e1-106">Bir koleksiyon türü özelliği uyguladığınızda bu aynı deseni izleyin.</span><span class="sxs-lookup"><span data-stu-id="b90e1-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="b90e1-107">Ancak koleksiyonda bulunan tür kendisini olduğunda koleksiyon türü özelliği düzeni karmaşıklık getirir bir <xref:System.Windows.DependencyObject> veya <xref:System.Windows.Freezable> türetilmiş sınıf.</span><span class="sxs-lookup"><span data-stu-id="b90e1-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

 <span data-ttu-id="b90e1-109">Bağımlılık özelliği oluşturduğunuzda, ilk alan değeri olarak özellik varsayılan değeri belirtmeyin.</span><span class="sxs-lookup"><span data-stu-id="b90e1-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="b90e1-110">Bunun yerine, bağımlılık özelliği meta verisi aracılığıyla varsayılan değeri belirtin.</span><span class="sxs-lookup"><span data-stu-id="b90e1-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="b90e1-111">Bağımlılık özelliği meta verilerinde belirtilen varsayılan değerin, özellik bir başvuru türü ise, örnek başına varsayılan bir değer değil; Bunun yerine, türün tüm örnekleri için geçerli bir varsayılan değerdir.</span><span class="sxs-lookup"><span data-stu-id="b90e1-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; ## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="b90e1-129">Koleksiyon Özellikleri bağlama değeri değişiklikleri raporlama</span><span class="sxs-lookup"><span data-stu-id="b90e1-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="b90e1-130">Kendisini bir bağımlılık özelliği olan bir koleksiyon özelliği için onun alt değişiklikleri otomatik olarak bildirmez.</span><span class="sxs-lookup"><span data-stu-id="b90e1-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="b90e1-131">Derlemeye bağlamalar oluşturuyorsanız, bu nedenle bazı veri bağlama senaryoları geçersiz kılmalarını değişiklikleri bağlama engelleyebilir.</span><span class="sxs-lookup"><span data-stu-id="b90e1-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="b90e1-132">Ancak, koleksiyon türü kullanıyorsanız <xref:System.Windows.FreezableCollection%601> , koleksiyon türü olarak ardından alt özellik değişiklikleri koleksiyondaki içerilen öğelerin düzgün şekilde bildirilir ve bağlama beklendiği gibi çalışır.</span><span class="sxs-lookup"><span data-stu-id="b90e1-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>

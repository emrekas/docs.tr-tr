 <span data-ttu-id="22853-106">Performansı belirlemede [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] en önemli faktörlerden biri işleme bağlıdır — daha fazla piksel işlemek için, performans maliyeti artar.</span><span class="sxs-lookup"><span data-stu-id="22853-106">One of the most important factors in determining [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] performance is that it is render bound—the more pixels you have to render, the greater the performance cost.</span></span> <span data-ttu-id="22853-107">Ancak, üzerinde boşaltılabilecek [!INCLUDE[TLA#tla_gpu](../../../../includes/tlasharptla-gpu-md.md)]daha fazla işleme kazanılabilecek daha fazla performans avantajı vardır.</span><span class="sxs-lookup"><span data-stu-id="22853-107">However, the more rendering that can be offloaded to the [!INCLUDE[TLA#tla_gpu](../../../../includes/tlasharptla-gpu-md.md)], the more performance benefits you can gain.</span></span> <span data-ttu-id="22853-108">Uygulama [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] donanımı işleme işlem hattı, en az Microsoft DirectX 7,0 sürümünü destekleyen donanımda Microsoft DirectX özelliklerinden tam olarak yararlanır.</span><span class="sxs-lookup"><span data-stu-id="22853-108">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] application hardware rendering pipeline takes full advantage of Microsoft DirectX features on hardware that supports a minimum of Microsoft DirectX version 7.0.</span></span> <span data-ttu-id="22853-109">Microsoft DirectX sürüm 7,0 ve PixelShader 2.0 + özelliklerini destekleyen donanımlar tarafından daha fazla iyileştirmeler kazanılabilir.</span><span class="sxs-lookup"><span data-stu-id="22853-109">Further optimizations can be gained by hardware that supports Microsoft DirectX version 7.0 and PixelShader 2.0+ features.</span></span>  
  
## <a name="software-rendering-pipeline"></a><span data-ttu-id="22853-110">Yazılım Işleme işlem hattı</span><span class="sxs-lookup"><span data-stu-id="22853-110">Software Rendering Pipeline</span></span>  
 <span data-ttu-id="22853-111">[!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] Yazılım işleme işlem hattı tamamen CPU bağlanmıştır.</span><span class="sxs-lookup"><span data-stu-id="22853-111">The [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] software rendering pipeline is entirely CPU bound.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]<span data-ttu-id="22853-112">, iyileştirilmiş ve tam özellikli bir yazılım tarayıcısı uygulamak için CPU 'daki SSE ve SSE2 yönerge kümelerinden faydalanır.</span><span class="sxs-lookup"><span data-stu-id="22853-112">takes advantage of the SSE and SSE2 instruction sets in the CPU to implement an optimized, fully-featured software rasterizer.</span></span> <span data-ttu-id="22853-113">Yazılıma geri dönüş, uygulama işlevlerinin donanım işleme işlem hattı kullanılarak işlenemediğinde sorunsuz bir şekilde gerçekleştirilir.</span><span class="sxs-lookup"><span data-stu-id="22853-113">Fallback to software is seamless any time application functionality cannot be rendered using the hardware rendering pipeline.</span></span>  
  
 <span data-ttu-id="22853-114">Yazılım modunda işleme yaparken karşılaşacağınız en büyük performans sorunu, işleme yaptığınız piksel sayısı olarak tanımlanmış olan Fill Rate ile ilgilidir.</span><span class="sxs-lookup"><span data-stu-id="22853-114">The biggest performance issue you will encounter when rendering in software mode is related to fill rate, which is defined as the number of pixels that you are rendering.</span></span> <span data-ttu-id="22853-115">Yazılım işleme modundaki performansla ilgili endişeleriniz varsa, bir pikselin yeniden çizilme sayısını en aza indirmenize çalışın.</span><span class="sxs-lookup"><span data-stu-id="22853-115">If you are concerned about performance in software rendering mode, try to minimize the number of times a pixel is redrawn.</span></span> <span data-ttu-id="22853-116">Örneğin, mavi bir arka plana sahip bir uygulamanız varsa, daha sonra bunun üzerinde biraz saydam bir görüntü oluşturup uygulamadaki tüm pikselleri iki kez işleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="22853-116">For example, if you have an application with a blue background, which then renders a slightly transparent image over it, you will render all of the pixels in the application twice.</span></span> <span data-ttu-id="22853-117">Sonuç olarak, yalnızca mavi arka plana sahip olmak yerine, uygulamanın görüntüyle işlenmesi iki kat sürer.</span><span class="sxs-lookup"><span data-stu-id="22853-117">As a result, it will take twice as long to render the application with the image than if you had only the blue background.</span></span>  
  
### <a name="graphics-rendering-tiers"></a><span data-ttu-id="22853-118">Grafik İşleme Katmanları</span><span class="sxs-lookup"><span data-stu-id="22853-118">Graphics Rendering Tiers</span></span>

 <span data-ttu-id="22853-121">Bunu başarmak için, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] çalışma zamanında bir sistemin grafik yeteneklerini belirleme işlevselliği sağlar.</span><span class="sxs-lookup"><span data-stu-id="22853-121">To achieve this, [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] provides functionality to determine the graphics capability of a system at runtime.</span></span> <span data-ttu-id="22853-122">Grafik özelliği, video kartını üç işleme özelliği katmanlarından biri olarak kategorilere ayırarak belirlenir.</span><span class="sxs-lookup"><span data-stu-id="22853-122">Graphics capability is determined by categorizing the video card as one of three rendering capability tiers.</span></span> [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]<span data-ttu-id="22853-123">uygulamanın işleme yeteneği katmanını sorgulamasını sağlayan bir API sunar.</span><span class="sxs-lookup"><span data-stu-id="22853-123">exposes an API that allows an application to query the rendering capability tier.</span></span> <span data-ttu-id="22853-124">Daha sonra uygulamanız, donanım tarafından desteklenen işleme katmanına bağlı olarak, çalışma zamanında farklı kod yolları alabilir.</span><span class="sxs-lookup"><span data-stu-id="22853-124">Your application can then take different code paths at run time depending on the rendering tier supported by the hardware.</span></span>  
  
 <span data-ttu-id="22853-125">İşleme katmanı düzeylerini en çok etkileyen grafik donanımının özellikleri şunlardır:</span><span class="sxs-lookup"><span data-stu-id="22853-125">The features of the graphics hardware that most impact the rendering tier levels are:</span></span>  
  
- <span data-ttu-id="22853-126">**VIDEO RAM** 'i Grafik donanımındaki video belleği miktarı, birleştirme grafikleri için kullanılabilecek arabelleklerin boyutunu ve sayısını belirler.</span><span class="sxs-lookup"><span data-stu-id="22853-126">**Video RAM** The amount of video memory on the graphics hardware determines the size and number of buffers that can be used for compositing graphics.</span></span>  
  
- <span data-ttu-id="22853-127">**Piksel gölgelendiricisi** Piksel gölgelendiricisi, her piksel temelinde etkileri hesaplayan bir grafik işleme işlevidir.</span><span class="sxs-lookup"><span data-stu-id="22853-127">**Pixel Shader** A pixel shader is a graphics processing function that calculates effects on a per-pixel basis.</span></span> <span data-ttu-id="22853-128">Görüntülenen grafiklerin çözümüne bağlı olarak, her bir görüntüleme çerçevesi için işlenmesi gereken birkaç milyon piksel olabilir.</span><span class="sxs-lookup"><span data-stu-id="22853-128">Depending on the resolution of the displayed graphics, there could be several million pixels that need to be processed for each display frame.</span></span>

 <span data-ttu-id="22853-132">Piksel gölgelendirici, köşe gölgelendiricisi ve Multitexture özellikleri, içinde [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)]farklı işleme katmanlarını tanımlamak Için kullanılan belirli DirectX sürüm düzeylerini tanımlamak için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="22853-132">The pixel shader, vertex shader, and multitexture features are used to define specific DirectX version levels, which, in turn, are used to define the different rendering tiers in [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)].</span></span>

- <span data-ttu-id="22853-135">**Işleme katmanı 0** Grafik donanım hızlandırma yok.</span><span class="sxs-lookup"><span data-stu-id="22853-135">**Rendering Tier 0** No graphics hardware acceleration.</span></span> <span data-ttu-id="22853-136">DirectX sürüm düzeyi 7,0 sürümünden düşüktür.</span><span class="sxs-lookup"><span data-stu-id="22853-136">The DirectX version level is less than version 7.0.</span></span>  
  
- <span data-ttu-id="22853-137">**Işleme katmanı 1** Kısmi grafik donanım hızlandırma.</span><span class="sxs-lookup"><span data-stu-id="22853-137">**Rendering Tier 1** Partial graphics hardware acceleration.</span></span> <span data-ttu-id="22853-138">DirectX sürüm düzeyi 7,0 sürümüne eşit veya daha büyük ve sürüm 9,0 ' den **küçük** .</span><span class="sxs-lookup"><span data-stu-id="22853-138">The DirectX version level is greater than or equal to version 7.0, and **lesser** than version 9.0.</span></span>  
  
- <span data-ttu-id="22853-139">**Işleme katmanı 2** Çoğu grafik özelliği grafik donanım hızlandırmasını kullanır.</span><span class="sxs-lookup"><span data-stu-id="22853-139">**Rendering Tier 2** Most graphics features use graphics hardware acceleration.</span></span> <span data-ttu-id="22853-140">DirectX sürüm düzeyi 9,0 sürümüne eşit veya ondan daha büyük.</span><span class="sxs-lookup"><span data-stu-id="22853-140">The DirectX version level is greater than or equal to version 9.0.</span></span>

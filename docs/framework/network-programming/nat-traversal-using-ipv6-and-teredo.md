## <a name="overview"></a><span data-ttu-id="d8972-109">Genel Bakış</span><span class="sxs-lookup"><span data-stu-id="d8972-109">Overview</span></span>  
 <span data-ttu-id="d8972-110">Internet Protokolü sürüm 4 (IPv4), bir IPv4 adresi 32 bit uzunluğunda tanımlanır.</span><span class="sxs-lookup"><span data-stu-id="d8972-110">The Internet Protocol version 4 (IPv4) defined an IPv4 address as 32 bits long.</span></span> <span data-ttu-id="d8972-111">Sonuç olarak, IPv4 yaklaşık 4 milyarı aşan benzersiz IP adreslerini destekler (2 ^ 32).</span><span class="sxs-lookup"><span data-stu-id="d8972-111">As a result, IPv4 supports approximately 4 billion unique IP addresses (2^32).</span></span> <span data-ttu-id="d8972-112">Bilgisayarları ve ağ cihazları 1990'ların genişletilmiş Internet üzerindeki sayısı arttıkça, IPv4 adres alanı sınırları belirgin hale geldi.</span><span class="sxs-lookup"><span data-stu-id="d8972-112">As the number of computers and network devices on the Internet expanded in the 1990s, the limits of the IPv4 address space became apparent.</span></span>  
  
 <span data-ttu-id="d8972-113">Çok sayıda özel IP temsil etmek bir tek benzersiz genel IP adreslerine izin verecek şekilde NAT'ı dağıtmak için yapılmış IPv4 ömrünü genişletmek için kullanılan birkaç tekniklerden birini (özel Intranet) adresler.</span><span class="sxs-lookup"><span data-stu-id="d8972-113">One of several techniques used to extend the lifetime of IPv4 has been to deploy NAT to allow a single unique public IP address to represent a large number of private IP addresses (private Intranet).</span></span> <span data-ttu-id="d8972-114">NAT cihazının arkasında özel IP adresleri, tek bir ortak IPv4 adresi paylaşın.</span><span class="sxs-lookup"><span data-stu-id="d8972-114">The private IP addresses behind the NAT device share the single public IPv4 address.</span></span> <span data-ttu-id="d8972-115">NAT cihazının (bir ucuz kablosuz erişim noktası ve yönlendirici, örneğin) bir adanmış donanım cihazı veya NAT sağlamak için bir hizmeti çalıştıran bir bilgisayar olabilir.</span><span class="sxs-lookup"><span data-stu-id="d8972-115">The NAT device may be a dedicated hardware device (an inexpensive Wireless Access Point and router, for example) or a computer running a service to provide NAT.</span></span> <span data-ttu-id="d8972-116">Bir cihaz veya hizmete bu genel IP adresi için genel Internet ile özel Intranet arasında IP ağ paketlerinin çevirir.</span><span class="sxs-lookup"><span data-stu-id="d8972-116">A device or service for this public IP address translates IP network packets between the public Internet and the private Intranet.</span></span>  
  
 <span data-ttu-id="d8972-117">Bu düzeni de Internet üzerindeki diğer IP adreslerine (genellikle sunucuları) isteklerini göndermek özel Intranet üzerinde çalışan istemci uygulamalar için çalışır.</span><span class="sxs-lookup"><span data-stu-id="d8972-117">This scheme works well for client applications running on the private Intranet that send requests to other IP addresses (usually servers) on the Internet.</span></span> <span data-ttu-id="d8972-118">Yanıtın gönderileceği bildiği bir yanıt döndürüldüğünde NAT cihazı veya sunucu, istemci istekleri bir eşleme bunu tutabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="d8972-118">The NAT device or server can keep a mapping of client requests so when a response is returned it knows where to send the response.</span></span> <span data-ttu-id="d8972-119">Ancak, sorunları istediğiniz hizmetleri sağlamak, paketler için dinleme ve yanıt, özel intranet NAT cihazının arkasında çalışan uygulamalar için bu düzeni doğurur.</span><span class="sxs-lookup"><span data-stu-id="d8972-119">But this scheme poses problems for applications running in the private Intranet behind the NAT device that want to provide services, listen for packets, and respond.</span></span> <span data-ttu-id="d8972-120">Bu özellikle eşler arası uygulamalar için geçerlidir.</span><span class="sxs-lookup"><span data-stu-id="d8972-120">This is particularly the case for peer-to-peer applications.</span></span>  
  
 <span data-ttu-id="d8972-121">IPv6 protokolü, bir IPv4 adresi 128 bit uzunluğunda tanımlanır.</span><span class="sxs-lookup"><span data-stu-id="d8972-121">The IPv6 protocol defined an IPv4 address as 128 bits long.</span></span> <span data-ttu-id="d8972-122">Sonuç olarak, bir çok büyük bir IP adres alanı 3.2 x 10 IPv6 destekleyen ^ 38 benzersiz adresler (2 ^ 128).</span><span class="sxs-lookup"><span data-stu-id="d8972-122">As a result, IPv6 supports very a large IP address space of 3.2 x 10^38 unique addresses (2^128).</span></span> <span data-ttu-id="d8972-123">Bu boyuttaki bir adres alanı ile her cihazın benzersiz adresi verilmesi için Internet'e bağlı mümkündür.</span><span class="sxs-lookup"><span data-stu-id="d8972-123">With an address space of this size, it is possible for every device connected to the Internet to be given a unique address.</span></span> <span data-ttu-id="d8972-124">Ancak, sorunları vardır.</span><span class="sxs-lookup"><span data-stu-id="d8972-124">But there are problems.</span></span> <span data-ttu-id="d8972-125">Dünya çoğunu, yine de yalnızca IPv4 kullanıyor.</span><span class="sxs-lookup"><span data-stu-id="d8972-125">Much of the world is still using only IPv4.</span></span> <span data-ttu-id="d8972-126">Özellikle, birçok küçük şirketler, kuruluşlar ve hanenin tarafından kullanılan kablosuz erişim noktaları ve mevcut yönlendiricilerin IPv6 desteklemez.</span><span class="sxs-lookup"><span data-stu-id="d8972-126">In particular, many of the existing routers and wireless access points used by small companies, organizations, and households do not support IPv6.</span></span> <span data-ttu-id="d8972-127">Ayrıca bu müşterilere bazı Internet hizmet sağlayıcıları desteklemeyen ya da IPv6 desteğini yapılandırılmadı.</span><span class="sxs-lookup"><span data-stu-id="d8972-127">Also some Internet service providers that serve these customers either do not support or have not configured support for IPv6.</span></span>  
  
 <span data-ttu-id="d8972-128">Birden fazla IPv6 geçiş teknolojileri IPv4 tüneli IPv6 adresleri için geliştirilmiştir.</span><span class="sxs-lookup"><span data-stu-id="d8972-128">Several IPv6 transition technologies have been developed to tunnel IPv6 addresses in an IPv4 packet.</span></span> <span data-ttu-id="d8972-129">6to4, ISATAP, bu teknolojiler şunları içerir ve IPv6 ana IP4 ağları diğer IPv6 ağları ulaşmak için geçmesi gereken, tek noktaya yayın IPv6 trafiği için adres ataması ve ana otomatik tünel sağlamak Teredo tünel oluşturur.</span><span class="sxs-lookup"><span data-stu-id="d8972-129">These technologies include 6to4, ISATAP, and Teredo tunnels that provide address assignment and host-to-host automatic tunneling for unicast IPv6 traffic when IPv6 hosts must traverse IP4 networks to reach other IPv6 networks.</span></span> <span data-ttu-id="d8972-130">IPv6 paketlerini IPv4 paketleri tünel gönderilir.</span><span class="sxs-lookup"><span data-stu-id="d8972-130">IPv6 packets are sent tunneled as IPv4 packets.</span></span> <span data-ttu-id="d8972-131">IPv6 adresleri için NAT geçişi bir NAT cihazı üzerinden izin veren birkaç tünel teknikleri kullanılır.</span><span class="sxs-lookup"><span data-stu-id="d8972-131">Several tunneling techniques are being used that allow NAT traversal for IPv6 addresses through a NAT device.</span></span>  
  
 <span data-ttu-id="d8972-132">Teredo IPv4 ağları için IPv6 bağlantısı getiren IPv6 geçiş teknolojileri biridir.</span><span class="sxs-lookup"><span data-stu-id="d8972-132">Teredo is one of the IPv6 transition technologies which brings IPv6 connectivity to IPv4 networks.</span></span> <span data-ttu-id="d8972-133">Teredo RFC Internet Engineering Task Force (IETF) tarafından yayınlanan 4380 belgelenmiştir.</span><span class="sxs-lookup"><span data-stu-id="d8972-133">Teredo is documented in RFC 4380 published by the Internet Engineering Task Force (IETF).</span></span> <span data-ttu-id="d8972-134">Windows XP SP2 ve daha sonra genel IPv6 adresi de Aralık 2001:0 sağlayan sanal bir Teredo bağdaştırıcı için destek sağlayabilirsiniz:: / 32.</span><span class="sxs-lookup"><span data-stu-id="d8972-134">Windows XP SP2 and later provide support for a virtual Teredo adapter which can provide a public IPv6 address in the range 2001:0::/32.</span></span> <span data-ttu-id="d8972-135">Şu IPv6 adresi, Internet'ten gelen bağlantıları için dinlemek için kullanılan ve dinleme hizmete bağlanmak istediğiniz IPv6 etkin istemciler için sağlanabilir.</span><span class="sxs-lookup"><span data-stu-id="d8972-135">This IPv6 address can be used to listen for incoming connections from the Internet and can be provided to IPv6 enabled clients that wish to connect to the listening service.</span></span> <span data-ttu-id="d8972-136">Bu uygulama yalnızca Teredo IPv6 adresini kullanarak bağlanabilir olduğundan, bir bilgisayara bir NAT cihazının arkasında adres konusunda endişelenmenize gerek uygulamanın serbest bırakır.</span><span class="sxs-lookup"><span data-stu-id="d8972-136">This frees an application from worrying about how to address a computer behind a NAT device, since the application can just connect to it using its IPv6 Teredo address.</span></span>

## <a name="pnrp-ids"></a><span data-ttu-id="1f959-118">PNRP kimlikleri</span><span class="sxs-lookup"><span data-stu-id="1f959-118">PNRP IDs</span></span>

 <span data-ttu-id="1f959-120">PNRP kimlikleri aşağıdakilerden oluşur:</span><span class="sxs-lookup"><span data-stu-id="1f959-120">PNRP IDs are composed of the following:</span></span>  
  
- <span data-ttu-id="1f959-121">Yüksek düzeyli 128 bit eşler arası (P2P) kimliği olarak da bilinen bir karmasını uç noktasına atanan bir eş adı var.</span><span class="sxs-lookup"><span data-stu-id="1f959-121">The high-order 128 bits, known as the peer-to-peer (P2P) ID, are a hash of a peer name assigned to the endpoint.</span></span> <span data-ttu-id="1f959-122">Eş adı şu biçimdedir: *Authority.Classifier*.</span><span class="sxs-lookup"><span data-stu-id="1f959-122">The peer name has the following format: *Authority.Classifier*.</span></span> <span data-ttu-id="1f959-123">Güvenli adları için *yetkilisi* on altılık karakterler eş ad ortak anahtarı Güvenli Karma Algoritması 1 (SHA1) karmasıdır.</span><span class="sxs-lookup"><span data-stu-id="1f959-123">For secured names, *Authority* is the Secure Hash Algorithm 1 (SHA1) hash of the public key of the peer name in hexadecimal characters.</span></span> <span data-ttu-id="1f959-124">Güvenli olmayan adları için *yetkilisi* tek karakteri "0".</span><span class="sxs-lookup"><span data-stu-id="1f959-124">For unsecured names, the *Authority* is the single character "0".</span></span> <span data-ttu-id="1f959-125">*Sınıflandırıcı* uygulamayı tanımlayan bir dizedir.</span><span class="sxs-lookup"><span data-stu-id="1f959-125">*Classifier* is a string that identifies the application.</span></span> <span data-ttu-id="1f959-126">Hiçbir eş adı sınıflandırıcı 149 karakterden uzun dahil olmak üzere `null` Sonlandırıcı.</span><span class="sxs-lookup"><span data-stu-id="1f959-126">No peer name classifier can be greater than 149 characters long, including the `null` terminator.</span></span>  
  
- <span data-ttu-id="1f959-127">Düşük düzey 128 bit tanımlayan farklı örnekleri aynı P2P kimliğinin aynı bulutta oluşturulan bir sayıdır hizmet konumu için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="1f959-127">The low-order 128 bits are used for the Service Location, which is a generated number that identifies different instances of the same P2P ID in the same cloud.</span></span>  
  
 <span data-ttu-id="1f959-128">Tek bir bilgisayardan kaydedilmesi birden çok PNRP kimlikleri bu P2P kimliği ve hizmet konumu birleşimini sağlar.</span><span class="sxs-lookup"><span data-stu-id="1f959-128">This combination of P2P ID and Service Location allows multiple PNRP IDs to be registered from a single computer.</span></span>

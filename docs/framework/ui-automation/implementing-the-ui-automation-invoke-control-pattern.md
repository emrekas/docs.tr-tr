## <a name="implementation-guidelines-and-conventions"></a><span data-ttu-id="82ce8-110">Uygulama yönergeleri ve kuralları</span><span class="sxs-lookup"><span data-stu-id="82ce8-110">Implementation Guidelines and Conventions</span></span>  
 <span data-ttu-id="82ce8-111">Çağır denetim düzeni uygularken aşağıdaki yönergeler ve kuralları dikkat edin:</span><span class="sxs-lookup"><span data-stu-id="82ce8-111">When implementing the Invoke control pattern, note the following guidelines and conventions:</span></span>  
  
- <span data-ttu-id="82ce8-112">Denetimleri uygulayın <xref:System.Windows.Automation.Provider.IInvokeProvider> aynı davranışı başka bir denetim düzeni sağlayıcısı aracılığıyla açık değilse.</span><span class="sxs-lookup"><span data-stu-id="82ce8-112">Controls implement <xref:System.Windows.Automation.Provider.IInvokeProvider> if the same behavior is not exposed through another control pattern provider.</span></span> <span data-ttu-id="82ce8-113">Örneğin, varsa <xref:System.Windows.Automation.InvokePattern.Invoke%2A> denetim üzerinde yöntemi, aynı eylemi gerçekleştirir <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> veya <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> yöntemi, denetimin uygulamaz <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span><span class="sxs-lookup"><span data-stu-id="82ce8-113">For example, if the <xref:System.Windows.Automation.InvokePattern.Invoke%2A> method on a control performs the same action as the <xref:System.Windows.Automation.ExpandCollapsePattern.Expand%2A> or <xref:System.Windows.Automation.ExpandCollapsePattern.Collapse%2A> method, the control should not implement <xref:System.Windows.Automation.Provider.IInvokeProvider>.</span></span>  
  
- <span data-ttu-id="82ce8-114">Bir denetim çağırma genellikle tıklayarak veya çift tıklayın veya ENTER, önceden tanımlı klavye kısayolu ya da diğer bazı tuş birleşimi tuşuna basarak gerçekleştirilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-114">Invoking a control is generally performed by clicking or double-clicking or pressing ENTER, a predefined keyboard shortcut, or some alternate combination of keystrokes.</span></span>  
  
- <span data-ttu-id="82ce8-115"><xref:System.Windows.Automation.InvokePatternIdentifiers.InvokedEvent> (bir yanıt kendi ilişkili eylemi taşıyan bir denetime) olarak etkinleştirilmiş denetim üzerinde oluşturulur.</span><span class="sxs-lookup"><span data-stu-id="82ce8-115"><xref:System.Windows.Automation.InvokePatternIdentifiers.InvokedEvent> is raised on a control that has been activated (as a response to a control carrying out its associated action).</span></span> <span data-ttu-id="82ce8-116">Mümkünse, Olay denetimi eylemi tamamlandı ve engellemeden döndürülen sonra oluşmalıdır.</span><span class="sxs-lookup"><span data-stu-id="82ce8-116">If possible, the event should be raised after the control has completed the action and returned without blocking.</span></span> <span data-ttu-id="82ce8-117">Aşağıdaki senaryolarda Invoke isteği karşılamada önce çağrılan olayı tetikleyen:</span><span class="sxs-lookup"><span data-stu-id="82ce8-117">The Invoked event should be raised before servicing the Invoke request in the following scenarios:</span></span>  
  
    - <span data-ttu-id="82ce8-118">Bu mümkün ve pratik eylem tamamlanana kadar beklenecek değildir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-118">It is not possible or practical to wait until the action is complete.</span></span>  
  
    - <span data-ttu-id="82ce8-119">Eylem kullanıcı etkileşimini gerektirir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-119">The action requires user interaction.</span></span>  
  
    - <span data-ttu-id="82ce8-120">Eylem zaman alır ve önemli bir süre için engellemek çağıran istemci neden olur.</span><span class="sxs-lookup"><span data-stu-id="82ce8-120">The action is time-consuming and will cause the calling client to block for a significant amount of time.</span></span>  
  
- <span data-ttu-id="82ce8-121">Aracılığıyla denetim çağırma önemli yan etkileri varsa, bu yan etkileri açılmamalıdır <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> özelliği.</span><span class="sxs-lookup"><span data-stu-id="82ce8-121">If invoking the control has significant side-effects, those side-effects should be exposed through the <xref:System.Windows.Automation.AutomationElement.AutomationElementInformation.HelpText%2A> property.</span></span> <span data-ttu-id="82ce8-122">Örneğin, olsa bile <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> seçimi ile ilişkilendirilmemiş <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> başka bir denetim seçili hale neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-122">For example, even though <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> is not associated with selection, <xref:System.Windows.Automation.Provider.IInvokeProvider.Invoke%2A> may cause another control to become selected.</span></span>  
  
- <span data-ttu-id="82ce8-123">Vurgu (veya fare bekletme) etkileri genellikle değil oluşturan çağrılan olay.</span><span class="sxs-lookup"><span data-stu-id="82ce8-123">Hover (or mouse-over) effects generally do not constitute an Invoked event.</span></span> <span data-ttu-id="82ce8-124">Ancak, üzerine gelindiğinde kullanılacak durumuna bağlı bir eylem (aksine, görsel efekt neden) gerçekleştirmek denetimlerin desteklemelidir <xref:System.Windows.Automation.InvokePattern> denetim düzeni.</span><span class="sxs-lookup"><span data-stu-id="82ce8-124">However, controls that perform an action (as opposed to cause a visual effect) based on the hover state should support the <xref:System.Windows.Automation.InvokePattern> control pattern.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="82ce8-125">Bu uygulama yalnızca bir fare ile ilgili yan etkisi sonucunda denetim çağrılabilir erişilebilirlik sorunu kabul edilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-125">This implementation is considered an accessibility issue if the control can be invoked only as a result of a mouse-related side effect.</span></span>  
  
- <span data-ttu-id="82ce8-126">Bir denetim çağırma öğeyi seçmekten farklıdır.</span><span class="sxs-lookup"><span data-stu-id="82ce8-126">Invoking a control is different from selecting an item.</span></span> <span data-ttu-id="82ce8-127">Ancak, denetimin bağlı olarak, onu çağırır bir yan etkisi olarak seçili hale neden olabilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-127">However, depending on the control, invoking it may cause the item to become selected as a side-effect.</span></span> <span data-ttu-id="82ce8-128">Örneğin, çağırma bir [!INCLUDE[TLA#tla_word](../../../includes/tlasharptla-word-md.md)] Belgelerim klasöründeki belge liste öğesi hem öğeyi seçer ve belge açılır.</span><span class="sxs-lookup"><span data-stu-id="82ce8-128">For example, invoking a [!INCLUDE[TLA#tla_word](../../../includes/tlasharptla-word-md.md)] document list item in the My Documents folder both selects the item and opens the document.</span></span>  
  
- <span data-ttu-id="82ce8-129">Bir öğe kaldırılmasını [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] açtıktan hemen sonra çağrılan ağaç.</span><span class="sxs-lookup"><span data-stu-id="82ce8-129">An element can disappear from the [!INCLUDE[TLA2#tla_uiautomation](../../../includes/tla2sharptla-uiautomation-md.md)] tree immediately upon being invoked.</span></span> <span data-ttu-id="82ce8-130">Olay geri çağırma tarafından sağlanan öğeden bilgi isteniyor sonucunda başarısız olabilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-130">Requesting information from the element provided by the event callback may fail as a result.</span></span> <span data-ttu-id="82ce8-131">Önbelleğe alınan bilgileri önceden getirme önerilen çözüm olabilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-131">Pre-fetching cached information is the recommended workaround.</span></span>  
  
- <span data-ttu-id="82ce8-132">Birden çok denetim düzenleri denetimlerini uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="82ce8-132">Controls can implement multiple control patterns.</span></span> <span data-ttu-id="82ce8-133">Örneğin, dolgu rengi denetimde [!INCLUDE[TLA#tla_xl](../../../includes/tlasharptla-xl-md.md)] araç hem de uygular <xref:System.Windows.Automation.InvokePattern> ve <xref:System.Windows.Automation.ExpandCollapsePattern> denetim desenleri.</span><span class="sxs-lookup"><span data-stu-id="82ce8-133">For example, the Fill Color control on the [!INCLUDE[TLA#tla_xl](../../../includes/tlasharptla-xl-md.md)] toolbar implements both the <xref:System.Windows.Automation.InvokePattern> and the <xref:System.Windows.Automation.ExpandCollapsePattern> control patterns.</span></span> <span data-ttu-id="82ce8-134"><xref:System.Windows.Automation.ExpandCollapsePattern> Menü kullanıma sunar ve <xref:System.Windows.Automation.InvokePattern> etkin seçimin seçilen renk ile doldurur.</span><span class="sxs-lookup"><span data-stu-id="82ce8-134"><xref:System.Windows.Automation.ExpandCollapsePattern> exposes the menu and the <xref:System.Windows.Automation.InvokePattern> fills the active selection with the chosen color.</span></span>

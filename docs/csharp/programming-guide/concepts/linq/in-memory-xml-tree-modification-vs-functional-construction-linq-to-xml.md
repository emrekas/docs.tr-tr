 <span data-ttu-id="45ef0-109">Çoğu durumda veri deposu işlemek için harcadığım süreyi kesir dönüşümsel kodu yazabilirsiniz ve bu kodu daha sağlam ve bakımı kolay.</span><span class="sxs-lookup"><span data-stu-id="45ef0-109">In many cases you can write the transformational code in a fraction of the time that it would take to manipulate the data store, and that code is more robust and easier to maintain.</span></span> <span data-ttu-id="45ef0-110">Daha fazla işlemci gücü, dönüşümsel yaklaşım sürebilir olsa bile bu gibi durumlarda, bu verileri değiştirmek için bir daha etkili yoludur.</span><span class="sxs-lookup"><span data-stu-id="45ef0-110">In these cases, even though the transformational approach can take more processing power, it is a more effective way to modify data.</span></span> <span data-ttu-id="45ef0-111">Bir geliştirici işlevsel yaklaşım ile ilgili bilgi sahibi ise, çoğu durumda Sonuç kodu anlamak daha kolay olur.</span><span class="sxs-lookup"><span data-stu-id="45ef0-111">If a developer is familiar with the functional approach, the resulting code in many cases is easier to understand.</span></span> <span data-ttu-id="45ef0-112">Her ağacın parçası değiştiren bir kodun bulmak daha kolaydır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-112">It is easy to find the code that modifies each part of the tree.</span></span>  
  
 <span data-ttu-id="45ef0-113">İşlevsel yaklaşım kullanılarak yazılmış kod henüz bu yaklaşımı anlamıyor bir geliştirici için tanıdık gelmiyor ancak burada bir XML ağacı yerinde değişiklik birçok DOM programcıları için daha tanıdık bir yaklaşımdır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-113">The approach where you modify an XML tree in-place is more familiar to many DOM programmers, whereas code written using the functional approach might look unfamiliar to a developer who doesn't yet understand that approach.</span></span> <span data-ttu-id="45ef0-114">Yalnızca büyük bir XML ağacı için küçük bir değişiklik yapmak varsa, burada, çoğu durumda bir yerde bir ağaç değişiklik yaklaşım daha az CPU zaman alabilir.</span><span class="sxs-lookup"><span data-stu-id="45ef0-114">If you have to only make a small modification to a large XML tree, the approach where you modify a tree in place in many cases will take less CPU time.</span></span>

## <a name="transforming-attributes-into-elements"></a><span data-ttu-id="45ef0-116">Öğelerine dönüştürme öznitelikleri</span><span class="sxs-lookup"><span data-stu-id="45ef0-116">Transforming Attributes into Elements</span></span>  
 <span data-ttu-id="45ef0-117">Bu örnekte, böylece öznitelikleri öğelerine haline gelir. aşağıdaki basit XML belgesi değiştirmek istediğiniz varsayalım.</span><span class="sxs-lookup"><span data-stu-id="45ef0-117">For this example, suppose you want to modify the following simple XML document so that the attributes become elements.</span></span> <span data-ttu-id="45ef0-118">Bu konuda, ilk yerinde değişikliği geleneksel yaklaşım sunulmaktadır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-118">This topic first presents the traditional in-place modification approach.</span></span> <span data-ttu-id="45ef0-119">Ardından, bir işlev oluşturma yaklaşımı da gösterir.</span><span class="sxs-lookup"><span data-stu-id="45ef0-119">It then shows the functional construction approach.</span></span>  
  
```xml  
<?xml version="1.0" encoding="utf-8" ?>  
<Root Data1="123" Data2="456">  
  <Child1>Content</Child1>  
</Root>  
```  
  
### <a name="modifying-the-xml-tree"></a><span data-ttu-id="45ef0-120">XML ağacı değiştirme</span><span class="sxs-lookup"><span data-stu-id="45ef0-120">Modifying the XML Tree</span></span>  
 <span data-ttu-id="45ef0-121">Öznitelikleri öğeleri oluşturmak için yordam kod yazma ve öznitelikleri gibi silin:</span><span class="sxs-lookup"><span data-stu-id="45ef0-121">You can write some procedural code to create elements from the attributes, and then delete the attributes, as follows:</span></span>  
  
```csharp  
XElement root = XElement.Load("Data.xml");  
foreach (XAttribute att in root.Attributes()) {  
    root.Add(new XElement(att.Name, (string)att));  
}  
root.Attributes().Remove();  
Console.WriteLine(root);  
```  
  
 <span data-ttu-id="45ef0-122">Bu kod aşağıdaki çıktıyı üretir:</span><span class="sxs-lookup"><span data-stu-id="45ef0-122">This code produces the following output:</span></span>  
  
```xml  
<Root>  
  <Child1>Content</Child1>  
  <Data1>123</Data1>  
  <Data2>456</Data2>  
</Root>  
```  
  
### <a name="functional-construction-approach"></a><span data-ttu-id="45ef0-123">İşlevsel oluşturma yaklaşımı</span><span class="sxs-lookup"><span data-stu-id="45ef0-123">Functional Construction Approach</span></span>  
 <span data-ttu-id="45ef0-124">Bunun aksine, işlevsel bir yaklaşım çekme ve öğeler ve öznitelikler kaynak ağacından seçme ve bunları yeni ağacına eklendikçe uygun şekilde dönüştürme yeni bir ağaç oluşturmak için kod oluşur.</span><span class="sxs-lookup"><span data-stu-id="45ef0-124">By contrast, a functional approach consists of code to form a new tree, picking and choosing elements and attributes from the source tree, and transforming them as appropriate as they are added to the new tree.</span></span> <span data-ttu-id="45ef0-125">İşlevsel yaklaşım aşağıdaki gibi görünür:</span><span class="sxs-lookup"><span data-stu-id="45ef0-125">The functional approach looks like the following:</span></span>  
  
```csharp  
XElement root = XElement.Load("Data.xml");  
XElement newTree = new XElement("Root",  
    root.Element("Child1"),  
    from att in root.Attributes()  
    select new XElement(att.Name, (string)att)  
);  
Console.WriteLine(newTree);  
```  
  
 <span data-ttu-id="45ef0-126">Bu örnekte, ilk örnekle aynı XML çıkarır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-126">This example outputs the same XML as the first example.</span></span> <span data-ttu-id="45ef0-127">Ancak, elde edilen işlevsel yaklaşım yeni XML yapısını gerçekten görebilirsiniz dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="45ef0-127">However, notice that you can actually see the resulting structure of the new XML in the functional approach.</span></span> <span data-ttu-id="45ef0-128">Oluşturulmasını gördüğünüz `Root` öğesinde, kodu çeker `Child1` kaynak ağacının ve kaynak ağacının öznitelikleri öğelere yeni ağacında dönüştüren kod öğesi.</span><span class="sxs-lookup"><span data-stu-id="45ef0-128">You can see the creation of the `Root` element, the code that pulls the `Child1` element from the source tree, and the code that transforms the attributes from the source tree to elements in the new tree.</span></span>  
  
 <span data-ttu-id="45ef0-129">İşlevsel örneği bu durumda ilk örnek kısa herhangi değil ve gerçekten daha kolay değildir.</span><span class="sxs-lookup"><span data-stu-id="45ef0-129">The functional example in this case is not any shorter than the first example, and it is not really any simpler.</span></span> <span data-ttu-id="45ef0-130">Ancak, bir XML ağacına yapmak için birçok değişiklik varsa, işlev olmayan bir yaklaşım oldukça karmaşık ve biraz geniş açılı olur.</span><span class="sxs-lookup"><span data-stu-id="45ef0-130">However, if you have many changes to make to an XML tree, the non functional approach will become quite complex and somewhat obtuse.</span></span> <span data-ttu-id="45ef0-131">Buna karşılık, işlevsel yaklaşım kullanırken, yalnızca form istenen XML, sorguları ve ifadeler, istenen içeriği çekmek uygun olarak eklemeye devam.</span><span class="sxs-lookup"><span data-stu-id="45ef0-131">In contrast, when using the functional approach, you still just form the desired XML, embedding queries and expressions as appropriate, to pull in the desired content.</span></span> <span data-ttu-id="45ef0-132">İşlevsel yaklaşım oluşturmanın kod üretir.</span><span class="sxs-lookup"><span data-stu-id="45ef0-132">The functional approach yields code that is easier to maintain.</span></span>  
  
 <span data-ttu-id="45ef0-133">Bu durumda işlevsel yaklaşım büyük olasılıkla oldukça ağaç işleme yaklaşım yanı sıra gerçekleştirecek değil dikkat edin.</span><span class="sxs-lookup"><span data-stu-id="45ef0-133">Notice that in this case the functional approach probably would not perform quite as well as the tree manipulation approach.</span></span> <span data-ttu-id="45ef0-134">Ana sorunu işlevsel yaklaşım daha kısa süreli nesneleri oluşturmasıdır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-134">The main issue is that the functional approach creates more short lived objects.</span></span> <span data-ttu-id="45ef0-135">Ancak işlevsel yaklaşım kullanarak büyük Programcı verimliliği için izin veriyorsa etkili bir düşüş olur.</span><span class="sxs-lookup"><span data-stu-id="45ef0-135">However, the tradeoff is an effective one if using the functional approach allows for greater programmer productivity.</span></span>  
  
 <span data-ttu-id="45ef0-136">Bu çok basit bir örneği, ancak felsefesi iki yaklaşım arasındaki farkı göstermek için kullanılır.</span><span class="sxs-lookup"><span data-stu-id="45ef0-136">This is a very simple example, but it serves to show the difference in philosophy between the two approaches.</span></span> <span data-ttu-id="45ef0-137">İşlevsel yaklaşım daha büyük XML belgelerini dönüştürmek için büyük üretkenlik artışı sağlar.</span><span class="sxs-lookup"><span data-stu-id="45ef0-137">The functional approach yields greater productivity gains for transforming larger XML documents.</span></span>

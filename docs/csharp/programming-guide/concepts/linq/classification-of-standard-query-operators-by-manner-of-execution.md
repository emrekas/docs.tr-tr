<span data-ttu-id="e1374-103">Standart sorgu işleci yöntemleri nesneleri uygulamalarına LINQ iki ana yoldan biriyle yürütün: hemen veya ertelenmiş.</span><span class="sxs-lookup"><span data-stu-id="e1374-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="e1374-104">Ertelenmiş yürütme kullanan sorgu işleçler ek olarak iki kategoriye ayrılabilir: akış ve akış dışı.</span><span class="sxs-lookup"><span data-stu-id="e1374-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="e1374-105">Nasıl farklı sorgu işleçlerinin yürütme biliyorsanız, belirli bir sorgudan Al sonuçları anlamanıza yardımcı olabilir.</span><span class="sxs-lookup"><span data-stu-id="e1374-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="e1374-106">Bu veri kaynağı değiştiğinde veya başka bir sorgu üzerinde bir sorgu oluşturuyorsanız özellikle doğrudur.</span><span class="sxs-lookup"><span data-stu-id="e1374-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="e1374-107">Bu konuda, standart sorgu işleçlerinin yürütme kendi şekilde göre sınıflandırır.</span><span class="sxs-lookup"><span data-stu-id="e1374-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="e1374-108">Yolla yürütme</span><span class="sxs-lookup"><span data-stu-id="e1374-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="e1374-109">Hemen</span><span class="sxs-lookup"><span data-stu-id="e1374-109">Immediate</span></span>  
 <span data-ttu-id="e1374-110">Hemen yürütme, veri kaynağını okuyun ve işlem kodda bir noktada sorgu burada bildirilir gerçekleştirilir anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="e1374-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="e1374-111">Numaralandırılabilir olmayan, tek bir sonuç tüm standart sorgu işleçleri hemen yürütün.</span><span class="sxs-lookup"><span data-stu-id="e1374-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="e1374-112">Ertelenmiş</span><span class="sxs-lookup"><span data-stu-id="e1374-112">Deferred</span></span>  
 <span data-ttu-id="e1374-113">Ertelenmiş yürütme işlemi kodda bir noktada sorgu burada bildirilir yapılmaz, anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="e1374-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="e1374-114">Yalnızca bir sorgu değişkeni, örneğin kullanarak numaralandırılana işlemi gerçekleştirilir bir `foreach` deyimi.</span><span class="sxs-lookup"><span data-stu-id="e1374-114">The operation is performed only when the query variable is enumerated, for example by using a `foreach` statement.</span></span> <span data-ttu-id="e1374-115">Bu sorgu zaman tanımlanan yerine sorgu yürütüldüğünde sorgu yürütülürken sonuçlarını veri kaynağı içeriğine bağlı anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="e1374-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="e1374-116">Sorgu değişkeni birden çok kez numaralandırılana, sonuçları her seferinde farklı olabilir.</span><span class="sxs-lookup"><span data-stu-id="e1374-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="e1374-117">Dönüş türü olan neredeyse tüm standart sorgu işleçleri <xref:System.Collections.Generic.IEnumerable%601> veya <xref:System.Linq.IOrderedEnumerable%601> ertelenmiş bir şekilde yürütün.</span><span class="sxs-lookup"><span data-stu-id="e1374-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="e1374-118">Ertelenmiş yürütme kullanan bir sorgu işleçleri, ayrıca akış veya akış dışı olarak sınıflandırılabilir.</span><span class="sxs-lookup"><span data-stu-id="e1374-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="e1374-119">Akış</span><span class="sxs-lookup"><span data-stu-id="e1374-119">Streaming</span></span>  
 <span data-ttu-id="e1374-120">Akış işleçleri bunlar öğeleri yield önce tüm kaynak verileri okumak yok.</span><span class="sxs-lookup"><span data-stu-id="e1374-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="e1374-121">Okunan ve uygunsa öğe verir, yürütme sırasında bir akış işleci her kaynak öğesi kendi işlemi gerçekleştirir.</span><span class="sxs-lookup"><span data-stu-id="e1374-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="e1374-122">Bir akış işleci, bir sonuç öğesi üretilen kadar kaynak öğeleri okumak devam eder.</span><span class="sxs-lookup"><span data-stu-id="e1374-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="e1374-123">Başka bir deyişle, bir sonuç öğesi oluşturmak için birden fazla kaynak öğesi okuma.</span><span class="sxs-lookup"><span data-stu-id="e1374-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="e1374-124">Akış dışı</span><span class="sxs-lookup"><span data-stu-id="e1374-124">Non-Streaming</span></span>  
 <span data-ttu-id="e1374-125">Akış dışı işleçler, bir sonuç öğesi yol açabilir önce tüm kaynak verilerini okuyun gerekir.</span><span class="sxs-lookup"><span data-stu-id="e1374-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="e1374-126">Sıralama veya gruplama gibi operations, bu kategoriye girer.</span><span class="sxs-lookup"><span data-stu-id="e1374-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="e1374-127">Yürütme zamanında akış dışı işleçler sorgu tüm kaynak verileri okumasını, bir veri yapısına yerleştirin, işlemi gerçekleştirmek ve verim elde edilen öğeleri.</span><span class="sxs-lookup"><span data-stu-id="e1374-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>

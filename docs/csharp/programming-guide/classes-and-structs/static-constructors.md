## <a name="remarks"></a><span data-ttu-id="abeca-105">Açıklamalar</span><span class="sxs-lookup"><span data-stu-id="abeca-105">Remarks</span></span>
<span data-ttu-id="abeca-106">Statik Oluşturucular, aşağıdaki özelliklere sahiptir:</span><span class="sxs-lookup"><span data-stu-id="abeca-106">Static constructors have the following properties:</span></span>  
  
- <span data-ttu-id="abeca-107">Statik Oluşturucu erişim değiştiricileri alın veya parametreleri desteklemez.</span><span class="sxs-lookup"><span data-stu-id="abeca-107">A static constructor does not take access modifiers or have parameters.</span></span>  

- <span data-ttu-id="abeca-108">Bir sınıf veya yapı, yalnızca bir statik Oluşturucu olabilir.</span><span class="sxs-lookup"><span data-stu-id="abeca-108">A class or struct can only have one static constructor.</span></span>

- <span data-ttu-id="abeca-109">Statik oluşturucular devralınmış veya aşırı yüklenmiş.</span><span class="sxs-lookup"><span data-stu-id="abeca-109">Static constructors cannot be inherited or overloaded.</span></span>

- <span data-ttu-id="abeca-110">Statik Oluşturucu yalnızca ortak dil çalışma (CLR) tarafından çağrılacak tutulmamıştır ve doğrudan çağrılamaz.</span><span class="sxs-lookup"><span data-stu-id="abeca-110">A static constructor cannot be called directly and is only meant to be called by the common language runtime (CLR).</span></span> <span data-ttu-id="abeca-111">Otomatik olarak çağrılır.</span><span class="sxs-lookup"><span data-stu-id="abeca-111">It is invoked automatically.</span></span>

- <span data-ttu-id="abeca-112">Kullanıcının statik Oluşturucu programa ne zaman çalıştırılır üzerinde denetimi yoktur.</span><span class="sxs-lookup"><span data-stu-id="abeca-112">The user has no control on when the static constructor is executed in the program.</span></span>
  
- <span data-ttu-id="abeca-113">Statik Oluşturucu otomatik olarak başlatmak için çağırılır [sınıfı](../../../csharp/language-reference/keywords/class.md) önce ilk örneği oluşturulduğunda veya herhangi bir statik üye başvurulur.</span><span class="sxs-lookup"><span data-stu-id="abeca-113">A static constructor is called automatically to initialize the [class](../../../csharp/language-reference/keywords/class.md) before the first instance is created or any static members are referenced.</span></span> <span data-ttu-id="abeca-114">Statik Oluşturucu bir örnek oluşturucusu önce çalıştırılır.</span><span class="sxs-lookup"><span data-stu-id="abeca-114">A static constructor will run before an instance constructor.</span></span> <span data-ttu-id="abeca-115">Bir türün statik Oluşturucusu bir olay ya da temsilci atanmış statik bir yöntemi çağrıldığında ve değil atandığı zaman çağrıldığını unutmayın.</span><span class="sxs-lookup"><span data-stu-id="abeca-115">Note that a type's static constructor is called when a static method assigned to an event or a delegate is invoked and not when it is assigned.</span></span> <span data-ttu-id="abeca-116">Statik alan değişken başlatıcıları statik oluşturucunun sınıfında mevcut olması durumunda, statik oluşturucunun yürütülmeden hemen önce sınıf bildiriminde göründükleri metinsel sırayla yürütülür.</span><span class="sxs-lookup"><span data-stu-id="abeca-116">If static field variable initializers are present in the class of the static constructor, they will be executed in the textual order in which they appear in the class declaration immediately prior to the execution of the static constructor.</span></span>

- <span data-ttu-id="abeca-117">Statik alanları başlatmak için bir statik Oluşturucu sağlamazsanız, tüm statik alanları varsayılan değerlerine bağlantısında listelendiği gibi başlatılır [varsayılan değerler tablosu](../../../csharp/language-reference/keywords/default-values-table.md).</span><span class="sxs-lookup"><span data-stu-id="abeca-117">If you don't provide a static constructor to initialize static fields, all static fields are initialized to their default value as listed in the [Default Values Table](../../../csharp/language-reference/keywords/default-values-table.md).</span></span> 
  
- <span data-ttu-id="abeca-118">Statik Oluşturucu bir özel durum oluşturursa, çalışma zamanı, ikinci kez çağırmayacaktır ve türü, programınızın çalıştığı uygulama etki alanı ömrü boyunca başlatılmamış kalır.</span><span class="sxs-lookup"><span data-stu-id="abeca-118">If a static constructor throws an exception, the runtime will not invoke it a second time, and the type will remain uninitialized for the lifetime of the application domain in which your program is running.</span></span> <span data-ttu-id="abeca-119">En yaygın olarak, bir <xref:System.TypeInitializationException> statik Oluşturucu bir tür örneği işlenemediğinde özel durum ya da statik oluşturucu içinde bir işlenmeyen özel durum ortaya çıkma için.</span><span class="sxs-lookup"><span data-stu-id="abeca-119">Most commonly, a <xref:System.TypeInitializationException> exception is thrown when a static constructor is unable to instantiate a type or for an unhandled exception occuring within a static constructor.</span></span> <span data-ttu-id="abeca-120">Kaynak kodunda açıkça tanımlanmayan örtük statik oluşturucular için sorun giderme Ara dil (IL) kod İnceleme gerektirebilir.</span><span class="sxs-lookup"><span data-stu-id="abeca-120">For implicit static constructors that are not explicitly defined in source code, troubleshooting may require inspection of the intermediate language (IL) code.</span></span>

- <span data-ttu-id="abeca-121">Statik Oluşturucu varlığını eklenmesini engeller <xref:System.Reflection.TypeAttributes.BeforeFieldInit> tür özniteliği.</span><span class="sxs-lookup"><span data-stu-id="abeca-121">The presence of a static constructor prevents the addition of the the <xref:System.Reflection.TypeAttributes.BeforeFieldInit> type attribute.</span></span> <span data-ttu-id="abeca-122">Bu, çalışma zamanı iyileştirmesi sınırlar.</span><span class="sxs-lookup"><span data-stu-id="abeca-122">This limits runtime optimization.</span></span>

- <span data-ttu-id="abeca-123">Bir alan olarak bildirilen `static readonly` yalnızca bildiriminden veya bir statik oluşturucunun parçası olarak atanabilir.</span><span class="sxs-lookup"><span data-stu-id="abeca-123">A field declared as `static readonly` may only be assigned as part of its declaration or in a static constructor.</span></span> <span data-ttu-id="abeca-124">Açık bir statik Oluşturucu gerekli olmadığı durumlarda, daha iyi çalışma zamanı iyileştirmesi için statik bir oluşturucu üzerinden değil, bildirim sırasında statik alanları başlatın.</span><span class="sxs-lookup"><span data-stu-id="abeca-124">When an explicit static constructor is not required, initialize static fields at declaration, rather than through a static constructor for better runtime optimization.</span></span>

> [!Note]
> <span data-ttu-id="abeca-125">Doğrudan erişilemez ancak başlatma özel durum sorunlarını giderme ile yardımcı olmak için açık bir statik Oluşturucu varlığını belgelenmelidir.</span><span class="sxs-lookup"><span data-stu-id="abeca-125">Though not directly accessible, the presence of an explicit static constructor should be documented to assist with troubleshooting initialization exceptions.</span></span>

### <a name="usage"></a><span data-ttu-id="abeca-126">Kullanım</span><span class="sxs-lookup"><span data-stu-id="abeca-126">Usage</span></span>

- <span data-ttu-id="abeca-127">Tipik bir kullanımı statik Oluşturucular, sınıf, bir günlük dosyası kullanarak ve oluşturucu girişleri bu dosyaya yazmak için kullanılan andır.</span><span class="sxs-lookup"><span data-stu-id="abeca-127">A typical use of static constructors is when the class is using a log file and the constructor is used to write entries to this file.</span></span>

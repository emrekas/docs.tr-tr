## <a name="rules-for-command-line-syntax-for-the-c-compiler"></a><span data-ttu-id="b69de-123">Komut satırı sözdizimi C# derleyicisi için kurallar</span><span class="sxs-lookup"><span data-stu-id="b69de-123">Rules for command-line syntax for the C# compiler</span></span>

<span data-ttu-id="b69de-124">İşletim sistemi komut satırında belirtilen bağımsız değişkenler yorumlarken C# derleyici aşağıdaki kuralları kullanır:</span><span class="sxs-lookup"><span data-stu-id="b69de-124">The C# compiler uses the following rules when it interprets arguments given on the operating system command line:</span></span>

- <span data-ttu-id="b69de-125">Bağımsız değişkenler bir boşluk veya sekme olduğu boşluk tarafından ayrılmış.</span><span class="sxs-lookup"><span data-stu-id="b69de-125">Arguments are delimited by white space, which is either a space or a tab.</span></span>

- <span data-ttu-id="b69de-126">İnceltme işareti (^) bir kaçış karakteri veya sınırlayıcı olarak tanınmıyor.</span><span class="sxs-lookup"><span data-stu-id="b69de-126">The caret character (^) is not recognized as an escape character or delimiter.</span></span> <span data-ttu-id="b69de-127">İşletim sistemindeki komut satırı ayrıştırıcı tarafından karakter için geçirilmeden önce işlenir `argv` programı dizisi.</span><span class="sxs-lookup"><span data-stu-id="b69de-127">The character is handled by the command-line parser in the operating system before it's passed to the `argv` array in the program.</span></span>

- <span data-ttu-id="b69de-128">Çift tırnak işaretleri ("dize") arasına bir dize içinde bulunan boşluk bağımsız olarak tek bir bağımsız değişken olarak yorumlanır.</span><span class="sxs-lookup"><span data-stu-id="b69de-128">A string enclosed in double quotation marks ("string") is interpreted as a single argument, regardless of white space that is contained within.</span></span> <span data-ttu-id="b69de-129">Tırnak işaretli dize bağımsız değişkeni eklenebilir.</span><span class="sxs-lookup"><span data-stu-id="b69de-129">A quoted string can be embedded in an argument.</span></span>

- <span data-ttu-id="b69de-130">Çift tırnak işareti öncesinde bir ters eğik çizgi (\\") bir değişmez değer çift tırnak işareti karakteri (") yorumlanır.</span><span class="sxs-lookup"><span data-stu-id="b69de-130">A double quotation mark preceded by a backslash (\\") is interpreted as a literal double quotation mark character (").</span></span>

- <span data-ttu-id="b69de-131">Ters eğik çizgi, başka bir deyişle, bunlar hemen çift tırnak işareti koyun sürece yorumlanır.</span><span class="sxs-lookup"><span data-stu-id="b69de-131">Backslashes are interpreted literally, unless they immediately precede a double quotation mark.</span></span>

- <span data-ttu-id="b69de-132">Çift tırnak işareti ters eğik çizgi sayıda izlediyseniz, bir ters eğik çizgi yerleştirecek `argv` dizisi için her çift ters eğik çizgi ve çift tırnak işareti, dize ayırıcı olarak yorumlanır.</span><span class="sxs-lookup"><span data-stu-id="b69de-132">If an even number of backslashes is followed by a double quotation mark, one backslash is put in the `argv` array for every pair of backslashes, and the double quotation mark is interpreted as a string delimiter.</span></span>

- <span data-ttu-id="b69de-133">Çift tırnak işareti ters eğik çizgi tek sayıda izlediyseniz, bir ters eğik çizgi yerleştirecek `argv` dizisi için her çift ters eğik çizgi ve çift tırnak işareti "kaçış" kalan ters eğik çizgi.</span><span class="sxs-lookup"><span data-stu-id="b69de-133">If an odd number of backslashes is followed by a double quotation mark, one backslash is put in the `argv` array for every pair of backslashes, and the double quotation mark is "escaped" by the remaining backslash.</span></span> <span data-ttu-id="b69de-134">Bu sabit değeri çift tırnak içinde eklenecek işareti (") neden `argv`.</span><span class="sxs-lookup"><span data-stu-id="b69de-134">This causes a literal double quotation mark (") to be added in `argv`.</span></span>

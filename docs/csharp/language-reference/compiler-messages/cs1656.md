# <a name="compiler-error-cs1656"></a><span data-ttu-id="32f6d-102">Derleyici Hatası CS1656</span><span class="sxs-lookup"><span data-stu-id="32f6d-102">Compiler Error CS1656</span></span>
<span data-ttu-id="32f6d-103">Bir 'salt okunur değişken türü' olduğu için 'değişkenine' nesnesine atanamaz</span><span class="sxs-lookup"><span data-stu-id="32f6d-103">Cannot assign to 'variable' because it is a 'read-only variable type'</span></span>  
  
 <span data-ttu-id="32f6d-104">Atama değişkeni salt okunur bir bağlamda oluştuğunda bu hata oluşur.</span><span class="sxs-lookup"><span data-stu-id="32f6d-104">This error occurs when an assignment to variable occurs in a read-only context.</span></span> <span data-ttu-id="32f6d-105">Salt okunur içerikler dahil [foreach](../../../csharp/language-reference/keywords/foreach-in.md) yineleme değişkenleri [kullanarak](../../../csharp/language-reference/keywords/using-statement.md) değişkenleri ve [sabit](../../../csharp/language-reference/keywords/fixed-statement.md) değişkenleri.</span><span class="sxs-lookup"><span data-stu-id="32f6d-105">Read-only contexts include [foreach](../../../csharp/language-reference/keywords/foreach-in.md) iteration variables, [using](../../../csharp/language-reference/keywords/using-statement.md) variables, and [fixed](../../../csharp/language-reference/keywords/fixed-statement.md) variables.</span></span> <span data-ttu-id="32f6d-106">Bu hatayı gidermek için atamaları deyimi bir değişkene önlemek `using` blokları, `foreach` deyimleri ve `fixed` deyimleri.</span><span class="sxs-lookup"><span data-stu-id="32f6d-106">To resolve this error, avoid assignments to a statement variable in `using` blocks, `foreach` statements, and `fixed` statements.</span></span>  
  
## <a name="example"></a><span data-ttu-id="32f6d-107">Örnek</span><span class="sxs-lookup"><span data-stu-id="32f6d-107">Example</span></span>  
 <span data-ttu-id="32f6d-108">Aşağıdaki örnek, tam bir koleksiyon içinde öğelerin yerine getirmeyi denediğinden Hatası CS1656 oluşturur bir `foreach` döngü.</span><span class="sxs-lookup"><span data-stu-id="32f6d-108">The following example generates error CS1656 because it tries to replace complete elements of a collection inside a `foreach` loop.</span></span> <span data-ttu-id="32f6d-109">Geçici bir yoludur değiştirmek için `foreach` döngüsü bir [için](../../../csharp/language-reference/keywords/for.md) döngü.</span><span class="sxs-lookup"><span data-stu-id="32f6d-109">One way to work around the error is to change the `foreach` loop to a [for](../../../csharp/language-reference/keywords/for.md) loop.</span></span> <span data-ttu-id="32f6d-110">Varolan öğeyi üyelerini değiştirmek için burada gösterilmeyen başka bir yolu olan; Bu sınıf, ancak değil yapı birimleri ile.</span><span class="sxs-lookup"><span data-stu-id="32f6d-110">Another way, not shown here, is to modify the members of the existing element; this is possible with classes, but not with structs.</span></span>  
  
```csharp  
using System;  
using System.Collections;  
using System.Collections.Generic;  
using System.Text;  
  
namespace CS1656_2  
{  
  
    class Book  
    {  
        public string Title;  
        public string Author;  
        public double Price;  
        public Book(string t, string a, double p)  
        {  
            Title=t;  
            Author=a;  
            Price=p;  
  
        }  
    }  
  
    class Program  
    {  
        private List<Book> list;  
        static void Main(string[] args)  
        {  
            Program prog = new Program();  
            prog.list = new List<Book>();  
            prog.list.Add(new Book ("The C# Programming Language",  
                                    "Hejlsberg, Wiltamuth, Golde",  
                                     29.95));  
            prog.list.Add(new Book ("The C++ Programming Language",  
                                    "Stroustrup",  
                                     29.95));  
            prog.list.Add(new Book ("The C Programming Language",  
                                    "Kernighan, Ritchie",  
                                    29.95));  
            foreach(Book b in prog.list)  
            {  
                // Cannot modify an entire element in a foreach loop   
                // even with reference types.  
                // Use a for or while loop instead  
                if (b.Title == "The C Programming Language")  
                    // Cannot assign to 'b' because it is a 'foreach   
                    // iteration variable'  
                    b = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95); //CS1656  
            }  
  
            //With a for loop you can modify elements  
            //for(int x = 0; x < prog.list.Count; x++)  
            //{  
            //    if(prog.list[x].Title== "The C Programming Language")  
            //        prog.list[x] = new Book("Programming Windows, 5th Ed.", "Petzold", 29.95);  
            //}  
            //foreach(Book b in prog.list)  
            //    Console.WriteLine(b.Title);  
  
        }  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="32f6d-111">Örnek</span><span class="sxs-lookup"><span data-stu-id="32f6d-111">Example</span></span>  
 <span data-ttu-id="32f6d-112">Aşağıdaki örnek yanı sıra diğer bağlamlarda CS1656'ın nasıl oluşturulabileceğini gösterir bir `foreach` döngü:</span><span class="sxs-lookup"><span data-stu-id="32f6d-112">The following sample demonstrates how CS1656 can be generated in other contexts besides a `foreach` loop:</span></span>  
  
```csharp  
// CS1656.cs  
// compile with: /unsafe  
using System;  
  
class C : IDisposable  
{  
    public void Dispose() { }  
}  
  
class CMain  
{  
    unsafe public static void Main()  
    {  
        using (C c = new C())  
        {  
            // Cannot assign to 'c' because it is a 'using variable'  
            c = new C(); // CS1656  
        }  
  
        int[] ary = new int[] { 1, 2, 3, 4 };  
        fixed (int* p = ary)  
        {  
            // Cannot assign to 'p' because it is a 'fixed variable'  
            p = null; // CS1656  
        }  
    }  
}  
```

# <a name="compiler-error-cs0269"></a><span data-ttu-id="841e8-102">Derleyici Hatası CS0269</span><span class="sxs-lookup"><span data-stu-id="841e8-102">Compiler Error CS0269</span></span>
<span data-ttu-id="841e8-103">Atanmamış'out 'parametresi' parametresinin kullanımı</span><span class="sxs-lookup"><span data-stu-id="841e8-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="841e8-104">Derleyici, kullanılmadan önce out parametresi bir değer atandı doğrulanamadı; atandığında, değer tanımsız olabilir.</span><span class="sxs-lookup"><span data-stu-id="841e8-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="841e8-105">Bir değer atayın mutlaka `out` değeri erişmeden önce çağrılan yöntem parametreleri.</span><span class="sxs-lookup"><span data-stu-id="841e8-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="841e8-106">Gerekirse geçirilen değişkenin değerini kullanın, bir `ref` parametresi yerine.</span><span class="sxs-lookup"><span data-stu-id="841e8-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="841e8-107">Daha fazla bilgi için [parametreleri geçirme](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="841e8-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="841e8-108">Örnek</span><span class="sxs-lookup"><span data-stu-id="841e8-108">Example</span></span>  
 <span data-ttu-id="841e8-109">Aşağıdaki örnek, CS0269 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="841e8-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="841e8-110">Örnek</span><span class="sxs-lookup"><span data-stu-id="841e8-110">Example</span></span>  
 <span data-ttu-id="841e8-111">Bu da oluşabilir bir değişkenin başlatılması try bloğunda ortaya çıkarsa, derleyici doğrulanamıyor olan başarıyla çalıştırır:</span><span class="sxs-lookup"><span data-stu-id="841e8-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```

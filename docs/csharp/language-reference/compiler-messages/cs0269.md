# <a name="compiler-error-cs0269"></a>Derleyici Hatası CS0269
Atanmamış parametre ' parametre ' kullanımı  
  
 Derleyici, out parametresine kullanılmadan önce bir değer atandığını doğrulayamadı; atandığında değeri tanımsız olabilir. Değere erişmeden önce çağrılan yöntemdeki `out` parametrelere bir değer atadığınızdan emin olun. Geçirilen değişkenin değerini kullanmanız gerekiyorsa, bunun yerine bir `ref` parametre kullanın. Daha fazla bilgi için bkz. [parametreleri geçirme](../../programming-guide/classes-and-structs/passing-parameters.md).  
  
## <a name="example"></a>Örnek  
 Aşağıdaki örnek CS0269 oluşturur:  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a>Örnek  
 Bu durum, bir değişkenin başlatılması bir try bloğunda gerçekleşirse, derleyicinin doğrulayamaması durumunda da meydana gelir:  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```

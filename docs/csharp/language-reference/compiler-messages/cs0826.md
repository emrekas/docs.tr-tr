# <a name="compiler-error-cs0826"></a><span data-ttu-id="825fe-102">Derleyici Hatası CS0826</span><span class="sxs-lookup"><span data-stu-id="825fe-102">Compiler Error CS0826</span></span>
<span data-ttu-id="825fe-103">En iyi türü örtük olarak belirlenmiş dizi için bulunamadı.</span><span class="sxs-lookup"><span data-stu-id="825fe-103">No best type found for implicitly typed array.</span></span>  
  
 <span data-ttu-id="825fe-104">Dizi öğelerinin tümü aynı türde veya örtük olarak dönüştürülebilir derleyici tarafından kullanılan tür çıkarımı kurallarına göre aynı türde olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="825fe-104">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="825fe-105">En iyi türü dizi ifadede türlerinden biri olması gerekir.</span><span class="sxs-lookup"><span data-stu-id="825fe-105">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="825fe-106">Öğeleri dönüştürülmeyecek için yeni bir tür gibi `object`.</span><span class="sxs-lookup"><span data-stu-id="825fe-106">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="825fe-107">Türü örtük olarak belirlenmiş dizi için derleyicinin dizi tanım Çıkarsama gerekir atanmış öğelerin türü temel türü.</span><span class="sxs-lookup"><span data-stu-id="825fe-107">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="825fe-108">Bu hatayı düzeltmek için</span><span class="sxs-lookup"><span data-stu-id="825fe-108">To correct this error</span></span>  
  
- <span data-ttu-id="825fe-109">Açık bir tür dizisi verin.</span><span class="sxs-lookup"><span data-stu-id="825fe-109">Give the array an explicit type.</span></span>  
  
- <span data-ttu-id="825fe-110">Tüm dizi öğeleri aynı türde verin.</span><span class="sxs-lookup"><span data-stu-id="825fe-110">Give all array elements the same type.</span></span>  
  
- <span data-ttu-id="825fe-111">Açık yayınları soruna neden bu öğeler üzerinde sağlar.</span><span class="sxs-lookup"><span data-stu-id="825fe-111">Provide explicit casts on those elements that might be causing the problem.</span></span>  
  
## <a name="example"></a><span data-ttu-id="825fe-112">Örnek</span><span class="sxs-lookup"><span data-stu-id="825fe-112">Example</span></span>  
 <span data-ttu-id="825fe-113">Aşağıdaki kod, CS0826 oluşturur, çünkü öğeleri olan bir dizi türü değil aynı ve derleyicinin tür çıkarımı mantığı tek bir en iyi tür bulunamadı:</span><span class="sxs-lookup"><span data-stu-id="825fe-113">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```

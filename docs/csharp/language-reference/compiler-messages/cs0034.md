# <a name="compiler-error-cs0034"></a><span data-ttu-id="ec6f4-102">Derleyici Hatası CS0034</span><span class="sxs-lookup"><span data-stu-id="ec6f4-102">Compiler Error CS0034</span></span>
<span data-ttu-id="ec6f4-103">İşleci 'operator' türü 'type1' ve 'type2' işlenenler üzerinde belirsizdir</span><span class="sxs-lookup"><span data-stu-id="ec6f4-103">Operator 'operator' is ambiguous on operands of type 'type1' and 'type2'</span></span>  
  
 <span data-ttu-id="ec6f4-104">Bir işleç iki nesneler üzerinde kullanıldı ve birden fazla dönüştürme derleyici bulunamadı.</span><span class="sxs-lookup"><span data-stu-id="ec6f4-104">An operator was used on two objects and the compiler found more than one conversion.</span></span> <span data-ttu-id="ec6f4-105">Dönüştürmeleri benzersiz olması gerektiği için ya da bir tür dönüştürme yapmak için veya bir dönüştürme açık hale getirmek için sahip.</span><span class="sxs-lookup"><span data-stu-id="ec6f4-105">Because conversions have to be unique, you either have to make a cast or to make one of the conversions explicit.</span></span> <span data-ttu-id="ec6f4-106">Daha fazla bilgi için [kullanıcı tanımlı dönüştürme işleçleri](../../../csharp/language-reference/operators/user-defined-conversion-operators.md).</span><span class="sxs-lookup"><span data-stu-id="ec6f4-106">For more information, see [User-defined conversion operators](../../../csharp/language-reference/operators/user-defined-conversion-operators.md).</span></span>  
  
 <span data-ttu-id="ec6f4-107">Aşağıdaki örnek, CS0034 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="ec6f4-107">The following sample generates CS0034:</span></span>  
  
```csharp  
// CS0034.cs  
public class A  
{  
   // allows for the conversion of A object to int  
   public static implicit operator int (A s)  
   {  
      return 0;  
   }  
  
   public static implicit operator string (A i)  
   {  
      return null;  
   }  
}  
  
public class B  
{  
   public static implicit operator int (B s)  
   // one way to resolve this CS0034 is to make one conversion explicit  
   // public static explicit operator int (B s)  
   {  
      return 0;  
   }  
  
   public static implicit operator string (B i)  
   {  
      return null;  
   }  
  
   public static implicit operator B (string i)  
   {  
      return null;  
   }  
  
   public static implicit operator B (int i)  
   {  
      return null;  
   }  
}  
  
public class C  
{  
   public static void Main ()  
   {  
      A a = new A();  
      B b = new B();  
      b = b + a;   // CS0034  
      // another way to resolve this CS0034 is to make a cast  
      // b = b + (int)a;  
   }  
}  
```  
  
 <span data-ttu-id="ec6f4-108">İçinde C# bir derleyici hatası yaptığı örtük kullanıcı tanımlı dönüştürmeler her ikisine de sahip bir sınıf tanımlamak olası 1.1 `int` ve `bool`, bit düzeyinde kullanılacak ve `AND` işleci (`&`) bu tür nesneler üzerinde.</span><span class="sxs-lookup"><span data-stu-id="ec6f4-108">In C# 1.1 a compiler bug made it possible to define a class that has implicit user-defined conversions to both `int` and `bool`, and to use a bitwise `AND` operator (`&`) on objects of that type.</span></span> <span data-ttu-id="ec6f4-109">İçinde C# 2.0, bu hata düzeltildi derleyici ile uyumlu hale getirme C# belirtimi ve bu nedenle aşağıdaki kodu CS0034 neden şimdi:</span><span class="sxs-lookup"><span data-stu-id="ec6f4-109">In C# 2.0, this bug was fixed to bring the compiler into compliance with the C# specification, and therefore the following code will now cause CS0034:</span></span>  
  
```csharp  
namespace CS0034  
{  
    class TestClass2  
    {  
        public void Test()  
        {  
            TestClass o1 = new TestClass();  
            TestClass o2 = new TestClass();  
            TestClass o3 = o1 & o2; //CS0034  
        }  
    }  
  
    class TestClass  
    {  
        public static implicit operator int(TestClass o)  
        {  
            return 1;  
        }  
  
        public static implicit operator TestClass(int v)  
        {  
            return new TestClass();  
        }  
  
        public static implicit operator bool(TestClass o)  
        {  
            return true;  
        }  
    }  
  
}  
```

## <a name="example"></a><span data-ttu-id="ca13d-110">Örnek</span><span class="sxs-lookup"><span data-stu-id="ca13d-110">Example</span></span>  
 <span data-ttu-id="ca13d-111">Aşağıdaki örnek, CS0686 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="ca13d-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="ca13d-112">Örnek</span><span class="sxs-lookup"><span data-stu-id="ca13d-112">Example</span></span>  
 <span data-ttu-id="ca13d-113">Bu hata, olayları bildirme zaman da meydana gelebilir.</span><span class="sxs-lookup"><span data-stu-id="ca13d-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="ca13d-114">Olay yapı otomatik olarak oluşturduğu `add_event` ve `remove_event` aşağıdaki örnekte olduğu gibi bir arabirimde aynı ada sahip yöntemleri ile çakışabilir yöntemleri:</span><span class="sxs-lookup"><span data-stu-id="ca13d-114">The event construct automatically generates the `add_event` and `remove_event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```

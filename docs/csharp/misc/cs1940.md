 <span data-ttu-id="5ad04-105">Bu hata, bir sorgu yönteminin birden çok uygulaması tanımlandığında ve derleyici, sorgu için en iyi olanı ayırt edemez.</span><span class="sxs-lookup"><span data-stu-id="5ad04-105">This error is generated when multiple implementations of a query method are defined and the compiler cannot disambiguate which one is best to use for the query.</span></span> <span data-ttu-id="5ad04-106">Aşağıdaki örnekte her iki sürümü `Select` de aynı imzaya sahiptir çünkü her ikisi de bir giriş parametresi `int` olarak kabul eder ve `int` dönüş değeri olarak kabul eder.</span><span class="sxs-lookup"><span data-stu-id="5ad04-106">In the following example, both versions of `Select` have the same signature, because they both accept one `int` as an input parameter and have `int` as a return value.</span></span>

## <a name="example"></a><span data-ttu-id="5ad04-109">Örnek</span><span class="sxs-lookup"><span data-stu-id="5ad04-109">Example</span></span>  
 <span data-ttu-id="5ad04-110">Aşağıdaki kod CS1940 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="5ad04-110">The following code generates CS1940:</span></span>  
  
```csharp  
// cs1940.cs  
using System; //must include explicitly for types defined in 3.5  
class Test  
{  
    public delegate int Dele(int x);  
    int num = 0;  
    public int Select(Func<int, int> d)  
    {  
        return d(this.num);  
    }  
    public int Select(Dele d) // CS1940  
    {  
        return d(this.num) + 1;  
    }  
    public static void Main()  
    {  
        var q = from x in new Test()  
        select x;  
    }  
}  
```

# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="daceb-102">Derleyici Uyarısı (Düzey 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="daceb-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="daceb-103">'Field' özel alanı atanır ancak değeri hiç kullanılmadı</span><span class="sxs-lookup"><span data-stu-id="daceb-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="daceb-104">Bu uyarı, derleyici bir değişkeni hiçbir zaman başvurulan doğrulayabilirsiniz çeşitli senaryolarda oluşabilir:</span><span class="sxs-lookup"><span data-stu-id="daceb-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
- <span data-ttu-id="daceb-105">Özel bir alan sabit bir değer atanır ancak hiçbir zaman sonradan okuyun.</span><span class="sxs-lookup"><span data-stu-id="daceb-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="daceb-106">Gereksiz atamayı performans efekt.</span><span class="sxs-lookup"><span data-stu-id="daceb-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="daceb-107">Alan kaldırmayı düşünün.</span><span class="sxs-lookup"><span data-stu-id="daceb-107">Consider removing the field.</span></span>  
  
- <span data-ttu-id="daceb-108">Özel veya iç statik alan başlatıcısında yalnızca sabit bir değer atanır.</span><span class="sxs-lookup"><span data-stu-id="daceb-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="daceb-109">Alan hesaplanmasını değiştirmeyi düşünün.</span><span class="sxs-lookup"><span data-stu-id="daceb-109">Consider changing the field to a const.</span></span>  
  
- <span data-ttu-id="daceb-110">Özel veya iç alan sabit değerler atanır ve #ifdef yönergesi tarafından bırakılan bloklar yalnızca kullanılır.</span><span class="sxs-lookup"><span data-stu-id="daceb-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="daceb-111">#İfdef bloğu içinde alanın yerleştirmeyi göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="daceb-111">Consider putting the field inside the #ifdef block.</span></span>  
  
- <span data-ttu-id="daceb-112">Özel veya iç alan birden fazla konumda sabit değerler atanır ancak aksi erişilebilir.</span><span class="sxs-lookup"><span data-stu-id="daceb-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="daceb-113">Alan gerekmiyorsa kaldırmadan göz önünde bulundurun.</span><span class="sxs-lookup"><span data-stu-id="daceb-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="daceb-114">Aksi takdirde, uygun şekilde kullanın.</span><span class="sxs-lookup"><span data-stu-id="daceb-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="daceb-115">Diğer durumlarda veya burada önerilen çözüm kabul edilebilir değil #pragma 0414 kullanın.</span><span class="sxs-lookup"><span data-stu-id="daceb-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="daceb-116">Aşağıdaki örnek CS0414 oluşturulacak yollarından biri gösterilmektedir:</span><span class="sxs-lookup"><span data-stu-id="daceb-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414

class C  
{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="daceb-117">**Not** , değişken `i` olarak bildirilen `protected or public`, derleyici bir türetilmiş sınıf kullanıyor olabilir veya başka bir istemci kod sınıfının örneği ve değişken başvurusu olup olmadığını bilemezsiniz herhangi bir hata oluşturulur</span><span class="sxs-lookup"><span data-stu-id="daceb-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>

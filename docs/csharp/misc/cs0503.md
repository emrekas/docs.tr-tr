 <span data-ttu-id="eace3-104">Bir üye yöntemi her ikisi de olarak işaretlemek için gereksizdir [soyut](../../csharp/language-reference/keywords/abstract.md) ve [sanal](../../csharp/language-reference/keywords/virtual.md) çünkü **soyut** gelir **sanal**.</span><span class="sxs-lookup"><span data-stu-id="eace3-104">It is redundant to mark a member method as both [abstract](../../csharp/language-reference/keywords/abstract.md) and [virtual](../../csharp/language-reference/keywords/virtual.md) because **abstract** implies **virtual**.</span></span>  
  
 <span data-ttu-id="eace3-105">Aşağıdaki örnek, CS0503 oluşturur:</span><span class="sxs-lookup"><span data-stu-id="eace3-105">The following sample generates CS0503:</span></span>

      abstract virtual public void f();   // CS0503

-   <span data-ttu-id="80efd-114">Her ikisi de üyeleri erişim düzeyleri individualized.</span><span class="sxs-lookup"><span data-stu-id="80efd-114">Members of both can have individualized access levels.</span></span> <span data-ttu-id="80efd-115">Örneğin, bir üye bildirilebilir `Public` ve başka bir `Private`.</span><span class="sxs-lookup"><span data-stu-id="80efd-115">For example, one member can be declared `Public` and another `Private`.</span></span>  
  
-   <span data-ttu-id="80efd-116">Her ikisi de arabirim uygulayabilir.</span><span class="sxs-lookup"><span data-stu-id="80efd-116">Both can implement interfaces.</span></span>  
  
-   <span data-ttu-id="80efd-117">Her ikisi de Oluşturucular, parametrelerle veya parametresiz paylaşılan sahip.</span><span class="sxs-lookup"><span data-stu-id="80efd-117">Both can have shared constructors, with or without parameters.</span></span>

## <a name="differences"></a><span data-ttu-id="80efd-120">Farkları</span><span class="sxs-lookup"><span data-stu-id="80efd-120">Differences</span></span>  
 <span data-ttu-id="80efd-121">Yapılar ve sınıflar aşağıdaki Bununla farklılık gösterir:</span><span class="sxs-lookup"><span data-stu-id="80efd-121">Structures and classes differ in the following particulars:</span></span>  
  
-   <span data-ttu-id="80efd-122">Yapıları, *değer türleri*; sınıflardır *başvuru türleri*.</span><span class="sxs-lookup"><span data-stu-id="80efd-122">Structures are *value types*; classes are *reference types*.</span></span> <span data-ttu-id="80efd-123">Bir yapı türünün değişkenini yapısı verileri içeren yerine verileri bir sınıf türü olarak bir başvuru içeren yapar.</span><span class="sxs-lookup"><span data-stu-id="80efd-123">A variable of a structure type contains the structure's data, rather than containing a reference to the data as a class type does.</span></span>  
  
-   <span data-ttu-id="80efd-124">Yığın ayırma yapıları kullanın. yığın ayırma sınıflarını kullanın.</span><span class="sxs-lookup"><span data-stu-id="80efd-124">Structures use stack allocation; classes use heap allocation.</span></span>  
  
-   <span data-ttu-id="80efd-125">Tüm yapı öğeleri `Public` değişkenleri tarafından varsayılan; sınıf ve sabitler `Private` olsa da diğer sınıf üyeleri varsayılan olarak `Public` varsayılan olarak.</span><span class="sxs-lookup"><span data-stu-id="80efd-125">All structure elements are `Public` by default; class variables and constants are `Private` by default, while other class members are `Public` by default.</span></span> <span data-ttu-id="80efd-126">Sınıf üyeleri için bu davranış Visual Basic 6.0 sistem varsayılanlarını ile uyumluluk sağlar.</span><span class="sxs-lookup"><span data-stu-id="80efd-126">This behavior for class members provides compatibility with the Visual Basic 6.0 system of defaults.</span></span>  
  
-   <span data-ttu-id="80efd-127">Bir yapı paylaşılmayan en az bir değişken veya paylaşılmayan, noncustom olmalıdır; olay öğesi bir sınıf tamamen boş olabilir.</span><span class="sxs-lookup"><span data-stu-id="80efd-127">A structure must have at least one nonshared variable or nonshared, noncustom event element; a class can be completely empty.</span></span>  
  
-   <span data-ttu-id="80efd-128">Yapı öğeleri olarak bildirilemez `Protected`; sınıf üyeleri olabilir.</span><span class="sxs-lookup"><span data-stu-id="80efd-128">Structure elements cannot be declared as `Protected`; class members can.</span></span>  
  
-   <span data-ttu-id="80efd-129">Yalnızca yapı yordamı olaylarını işleyebileceğinizi bir [paylaşılan](../../../../visual-basic/language-reference/modifiers/shared.md) `Sub` yordamı ve yalnızca, [AddHandler deyimi](../../../../visual-basic/language-reference/statements/addhandler-statement.md); herhangi bir sınıf yordam ya da kullanarakolaylarıişleyebilir[ İşleme](../../../../visual-basic/language-reference/statements/handles-clause.md) anahtar sözcüğü veya `AddHandler` deyimi.</span><span class="sxs-lookup"><span data-stu-id="80efd-129">A structure procedure can handle events only if it is a [Shared](../../../../visual-basic/language-reference/modifiers/shared.md)`Sub` procedure, and only by means of the [AddHandler Statement](../../../../visual-basic/language-reference/statements/addhandler-statement.md); any class procedure can handle events, using either the [Handles](../../../../visual-basic/language-reference/statements/handles-clause.md) keyword or the `AddHandler` statement.</span></span> <span data-ttu-id="80efd-130">Daha fazla bilgi için [olayları](../../../../visual-basic/programming-guide/language-features/events/index.md).</span><span class="sxs-lookup"><span data-stu-id="80efd-130">For more information, see [Events](../../../../visual-basic/programming-guide/language-features/events/index.md).</span></span>  
  
-   <span data-ttu-id="80efd-131">Yapı değişken bildirimleri başlatıcılar veya ilk boyutları diziler için belirtilemez; sınıf değişken bildirimlerini kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="80efd-131">Structure variable declarations cannot specify initializers or initial sizes for arrays; class variable declarations can.</span></span>  
  
-   <span data-ttu-id="80efd-132">Yapıları örtük olarak devralmak <xref:System.ValueType?displayProperty=nameWithType> sınıfı ve her türden; devralınamaz sınıflar devralınan herhangi bir sınıf veya sınıflar dışında <xref:System.ValueType?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="80efd-132">Structures implicitly inherit from the <xref:System.ValueType?displayProperty=nameWithType> class and cannot inherit from any other type; classes can inherit from any class or classes other than <xref:System.ValueType?displayProperty=nameWithType>.</span></span>  
  
-   <span data-ttu-id="80efd-133">Yapıları devralınabilir değil; sınıflardır.</span><span class="sxs-lookup"><span data-stu-id="80efd-133">Structures are not inheritable; classes are.</span></span>  
  
-   <span data-ttu-id="80efd-134">Yapıları hiçbir zaman sonlandırılır, ortak dil çalışma zamanı (CLR) hiçbir zaman çağırır <xref:System.Object.Finalize%2A> herhangi bir yapının; metodunda sınıfları, çağıran atıktoplayıcı tarafından (GC) sonlandırılır <xref:System.Object.Finalize%2A> başvuru active vardır algıladığında bir sınıf Kalan.</span><span class="sxs-lookup"><span data-stu-id="80efd-134">Structures are never terminated, so the common language runtime (CLR) never calls the <xref:System.Object.Finalize%2A> method on any structure; classes are terminated by the garbage collector (GC), which calls <xref:System.Object.Finalize%2A> on a class when it detects there are no active references remaining.</span></span>  
  
-   <span data-ttu-id="80efd-135">Bir yapının bir oluşturucu gerektirmez; bir sınıf yapar.</span><span class="sxs-lookup"><span data-stu-id="80efd-135">A structure does not require a constructor; a class does.</span></span>  
  
-   <span data-ttu-id="80efd-136">Yapıları olabilir yalnızca parametreleri; izlerseniz paylaşılmayan oluşturucular sınıfları bunları parametrelerle veya parametresiz olabilir.</span><span class="sxs-lookup"><span data-stu-id="80efd-136">Structures can have nonshared constructors only if they take parameters; classes can have them with or without parameters.</span></span>  
  
 <span data-ttu-id="80efd-137">Her yapı daha genel bir parametresiz oluşturucu örtük daha vardır.</span><span class="sxs-lookup"><span data-stu-id="80efd-137">Every structure has an implicit public constructor without parameters.</span></span> <span data-ttu-id="80efd-138">Bu oluşturucu, yapının tüm veri öğeleri varsayılan değerlerine başlatır.</span><span class="sxs-lookup"><span data-stu-id="80efd-138">This constructor initializes all the structure's data elements to their default values.</span></span> <span data-ttu-id="80efd-139">Bu davranış tanımlanamaz.</span><span class="sxs-lookup"><span data-stu-id="80efd-139">You cannot redefine this behavior.</span></span>  
  
## <a name="instances-and-variables"></a><span data-ttu-id="80efd-140">Örnekler ve değişkenleri</span><span class="sxs-lookup"><span data-stu-id="80efd-140">Instances and Variables</span></span>  
 <span data-ttu-id="80efd-141">Yapılar değer türleri olduğundan, her bir yapı değişken kalıcı olarak tek tek yapısı örneğine bağlı.</span><span class="sxs-lookup"><span data-stu-id="80efd-141">Because structures are value types, each structure variable is permanently bound to an individual structure instance.</span></span> <span data-ttu-id="80efd-142">Ancak başvuru türleri sınıflardır ve bir nesne değişkeninin için çeşitli sınıf örneklerinin farklı zamanlarda başvuruda bulunabilir.</span><span class="sxs-lookup"><span data-stu-id="80efd-142">But classes are reference types, and an object variable can refer to various class instances at different times.</span></span> <span data-ttu-id="80efd-143">Bu ayrım kullanımınızı yapılar ve sınıflar aşağıdaki şekillerde etkiler:</span><span class="sxs-lookup"><span data-stu-id="80efd-143">This distinction affects your usage of structures and classes in the following ways:</span></span>  
  
-   <span data-ttu-id="80efd-144">**Başlatma.**</span><span class="sxs-lookup"><span data-stu-id="80efd-144">**Initialization.**</span></span> <span data-ttu-id="80efd-145">Yapısı değişkeni örtük olarak bir başlatma yapısı parametresiz oluşturucu kullanılarak öğeleri içerir.</span><span class="sxs-lookup"><span data-stu-id="80efd-145">A structure variable implicitly includes an initialization of the elements using the structure's parameterless constructor.</span></span> <span data-ttu-id="80efd-146">Bu nedenle, `Dim s As struct1` eşdeğerdir `Dim s As struct1 = New struct1()`.</span><span class="sxs-lookup"><span data-stu-id="80efd-146">Therefore, `Dim s As struct1` is equivalent to `Dim s As struct1 = New struct1()`.</span></span>  
  
-   <span data-ttu-id="80efd-147">**Değişkenleri atanıyor.**</span><span class="sxs-lookup"><span data-stu-id="80efd-147">**Assigning Variables.**</span></span> <span data-ttu-id="80efd-148">Başka bir yapı değişkenine atayın veya yapısı örneği için bir yordam bağımsız değişkeninin geçirmek, geçerli öğelerin değerlerinin tüm değişken yeni yapısına kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="80efd-148">When you assign one structure variable to another, or pass a structure instance to a procedure argument, the current values of all the variable elements are copied to the new structure.</span></span> <span data-ttu-id="80efd-149">Başka bir nesne değişkenine atayın veya bir nesne değişkeni geçirmek için bir yordam, yalnızca başvuru işaretçi kopyalanır.</span><span class="sxs-lookup"><span data-stu-id="80efd-149">When you assign one object variable to another, or pass an object variable to a procedure, only the reference pointer is copied.</span></span>  
  
-   <span data-ttu-id="80efd-150">**Hiçbir şey atanıyor.**</span><span class="sxs-lookup"><span data-stu-id="80efd-150">**Assigning Nothing.**</span></span> <span data-ttu-id="80efd-151">Değer atayabilirsiniz [hiçbir şey](../../../../visual-basic/language-reference/nothing.md) değişkeni, ancak örnek bir yapıya devam değişkeniyle ilişkilendirilecek.</span><span class="sxs-lookup"><span data-stu-id="80efd-151">You can assign the value [Nothing](../../../../visual-basic/language-reference/nothing.md) to a structure variable, but the instance continues to be associated with the variable.</span></span> <span data-ttu-id="80efd-152">Yine, yöntemlerinin çağrılması ve değişken öğeleri atama tarafından başlatılır ancak veri öğelerine erişebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="80efd-152">You can still call its methods and access its data elements, although variable elements are reinitialized by the assignment.</span></span>  
  
     <span data-ttu-id="80efd-153">Buna karşılık, bir nesne değişkeninin ayarlarsanız, `Nothing`, herhangi bir sınıfı örnekten ilişkisini kaldırın ve başka bir örneği atamak kadar hiç üye değişkeni erişemezsiniz.</span><span class="sxs-lookup"><span data-stu-id="80efd-153">In contrast, if you set an object variable to `Nothing`, you dissociate it from any class instance, and you cannot access any members through the variable until you assign another instance to it.</span></span>  
  
-   <span data-ttu-id="80efd-154">**Birden çok örneği.**</span><span class="sxs-lookup"><span data-stu-id="80efd-154">**Multiple Instances.**</span></span> <span data-ttu-id="80efd-155">Bir nesne değişkeninin farklı sınıf örneklerinin farklı zamanlarda atanmış olabilir ve birden fazla nesne değişkenleri aynı anda aynı sınıf örneğine bakabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="80efd-155">An object variable can have different class instances assigned to it at different times, and several object variables can refer to the same class instance at the same time.</span></span> <span data-ttu-id="80efd-156">Sınıf üyelerinin değerlerini için yaptığınız değişiklikler, aynı örneğine işaret eden başka bir değişken üzerinden erişildiğinde bu üyeler etkiler.</span><span class="sxs-lookup"><span data-stu-id="80efd-156">Changes you make to the values of class members affect those members when accessed through another variable pointing to the same instance.</span></span>  
  
     <span data-ttu-id="80efd-157">Yapı öğeleri, ancak kendi örneği içinde yalıtılır.</span><span class="sxs-lookup"><span data-stu-id="80efd-157">Structure elements, however, are isolated within their own instance.</span></span> <span data-ttu-id="80efd-158">Değişiklikleri değerlerine değil yansıtılır herhangi diğer yapı değişkenleri, aynı diğer durumlarda bile `Structure` bildirimi.</span><span class="sxs-lookup"><span data-stu-id="80efd-158">Changes to their values are not reflected in any other structure variables, even in other instances of the same `Structure` declaration.</span></span>  
  
-   <span data-ttu-id="80efd-159">**Eşitlik.**</span><span class="sxs-lookup"><span data-stu-id="80efd-159">**Equality.**</span></span> <span data-ttu-id="80efd-160">İki yapıları eşitlik testi ile bir öğeye göre test gerçekleştirilmesi gerekir.</span><span class="sxs-lookup"><span data-stu-id="80efd-160">Equality testing of two structures must be performed with an element-by-element test.</span></span> <span data-ttu-id="80efd-161">İki nesne değişkenini kullanarak karşılaştırılabilir <xref:System.Object.Equals%2A> yöntemi.</span><span class="sxs-lookup"><span data-stu-id="80efd-161">Two object variables can be compared using the <xref:System.Object.Equals%2A> method.</span></span> <span data-ttu-id="80efd-162"><xref:System.Object.Equals%2A> iki değişken aynı örneğine kılınmayacağını gösterir.</span><span class="sxs-lookup"><span data-stu-id="80efd-162"><xref:System.Object.Equals%2A> indicates whether the two variables point to the same instance.</span></span>

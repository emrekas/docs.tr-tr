<span data-ttu-id="1283b-103">A `TypeOf`... `Is` ifade, tutamıyor bir veri türü için bir nesne başvuru değişkenini test eder.</span><span class="sxs-lookup"><span data-stu-id="1283b-103">A `TypeOf`...`Is` expression tests an object reference variable to a data type that it cannot hold.</span></span>  
  
 <span data-ttu-id="1283b-104">Bazı durumlarda, derleyicinin belirlemek bir `TypeOf`... `Is` test yalnızca başarısız olabilir, örneğin iki sınıfları devralma ilişkisi yok ise.</span><span class="sxs-lookup"><span data-stu-id="1283b-104">In some cases the compiler can determine that a `TypeOf`...`Is` test can only fail, for example if there is no inheritance relationship between two classes.</span></span>  
  
 <span data-ttu-id="1283b-105">Aşağıdaki kod, bu hata oluşturabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="1283b-105">The following code can generate this error.</span></span>

 `End If`  
  
 <span data-ttu-id="1283b-106">Çünkü <xref:System.Windows.Forms.Form> ve <xref:System.Array> tamamen ilişkisiz türleri, derleyici can belirlemek, `TypeOf`... `Is` ifade döndürür `False` için herhangi bir değerini `refVar`.</span><span class="sxs-lookup"><span data-stu-id="1283b-106">Because <xref:System.Windows.Forms.Form> and <xref:System.Array> are totally unrelated types, the compiler can determine that the `TypeOf`...`Is` expression returns `False` for any value of `refVar`.</span></span>

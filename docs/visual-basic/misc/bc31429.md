<span data-ttu-id="d1708-103">Bir ifade, sınıf, yapı, modül veya aynı ada sahip birden fazla üye içeren arabirimi içinde tanımlanan bir programlama öğesi erişir.</span><span class="sxs-lookup"><span data-stu-id="d1708-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="d1708-104">Bu hatanın en olası nedeni *büyük küçük harf duyarlılığı*.</span><span class="sxs-lookup"><span data-stu-id="d1708-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="d1708-105">Visual Basic adlarını büyük küçük harf duyarsız, bunları farklı farklı konumlarda kodunuzda harfleri büyük harfe dönüştürebilirsiniz anlamına gelir.</span><span class="sxs-lookup"><span data-stu-id="d1708-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="d1708-106">Örneğin, adında bir değişken tanımlayın `XYZ` ve daha sonra olarak erişmek `xyz`, derleyici denk olarak iki ad göz önünde bulundurur.</span><span class="sxs-lookup"><span data-stu-id="d1708-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="d1708-107">Ancak, diğer diller gibi [ C# ](../../csharp/index.md) ve [Visual C++](/cpp/index), büyük küçük harfe duyarlıdır.</span><span class="sxs-lookup"><span data-stu-id="d1708-107">However, other languages, such as [C#](../../csharp/index.md) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="d1708-108">Böyle bir dilde `XYZ` ve `xyz` adla aynı olması için dikkate alınmaz.</span><span class="sxs-lookup"><span data-stu-id="d1708-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="d1708-109">Bu nedenle, böyle bir dilde yazılmış bir sınıf adlı bir değişken tanımlayabilirsiniz `XYZ` ve adlı bir özellik `xyz`.</span><span class="sxs-lookup"><span data-stu-id="d1708-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="d1708-110">Ortak dil çalışma zamanı (CLR) büyük/küçük harfe duyarlılık derlemelerde korur.</span><span class="sxs-lookup"><span data-stu-id="d1708-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="d1708-111">Ancak, bir Visual Basic uygulama adlarına sahip bir derleme erişir, `XYZ` ve `xyz`, aynı adı olarak görünürler.</span><span class="sxs-lookup"><span data-stu-id="d1708-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>

<span data-ttu-id="bb5cf-103">Bir iş parçacığı ve uygulama etki alanı için benzersiz olan yönetilen iş parçacığı yerel depolaması (TLS) verileri depolamak için kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-103">You can use managed thread local storage (TLS) to store data that is unique to a thread and application domain.</span></span> <span data-ttu-id="bb5cf-104">.NET Framework yönetilen TLS kullanmak için iki yol sunar: iş parçacığı göreli statik alanları ve veri yuvaları.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-104">The .NET Framework provides two ways to use managed TLS: thread-relative static fields and data slots.</span></span>

 <span data-ttu-id="bb5cf-112">İçinde [!INCLUDE[net_v40_long](../../../includes/net-v40-long-md.md)], kullanabileceğiniz <xref:System.Threading.ThreadLocal%601?displayProperty=nameWithType> nesne ilk kez kullanıldığında, gevşek başlatılır iş parçacığı-yerel nesneleri oluşturmak için sınıf.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-112">In the [!INCLUDE[net_v40_long](../../../includes/net-v40-long-md.md)], you can use the <xref:System.Threading.ThreadLocal%601?displayProperty=nameWithType> class to create thread-local objects that are initialized lazily when the object is first consumed.</span></span> <span data-ttu-id="bb5cf-113">Daha fazla bilgi için [yavaş başlatma](../../../docs/framework/performance/lazy-initialization.md).</span><span class="sxs-lookup"><span data-stu-id="bb5cf-113">For more information, see [Lazy Initialization](../../../docs/framework/performance/lazy-initialization.md).</span></span>  
  
## <a name="uniqueness-of-data-in-managed-tls"></a><span data-ttu-id="bb5cf-114">Yönetilen TLS veri benzersizliği</span><span class="sxs-lookup"><span data-stu-id="bb5cf-114">Uniqueness of Data in Managed TLS</span></span>  
 <span data-ttu-id="bb5cf-115">İş parçacığı göreli statik alanları veya veri yuvaları kullanmanıza bakılmaksızın, iş parçacığı ve uygulama etki alanı birleşimi için yönetilen TLS verilerinde benzersizdir.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-115">Whether you use thread-relative static fields or data slots, data in managed TLS is unique to the combination of thread and application domain.</span></span>  
  
-   <span data-ttu-id="bb5cf-116">Her iki iş parçacığının aynı alanı veya yuvası kullandığınızda da bir uygulama etki alanı içinde başka bir iş parçacığından verileri tek bir iş parçacığı değiştiremezsiniz.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-116">Within an application domain, one thread cannot modify data from another thread, even when both threads use the same field or slot.</span></span>  
  
-   <span data-ttu-id="bb5cf-117">Bir iş parçacığı aynı alanı veya yuvası birden çok uygulama etki alanından eriştiğinde, ayrı bir değer her uygulama etki alanında tutulur.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-117">When a thread accesses the same field or slot from multiple application domains, a separate value is maintained in each application domain.</span></span>  
  
 <span data-ttu-id="bb5cf-118">Örneğin, bir iş parçacığı ayarlar iş parçacığı göreli statik alanının değeri başka bir uygulama etki alanına girdiğinden ve alanın değerini alır., ikinci uygulama etki alanında alınan değeri ilk uygulama etki alanı değeri farklılık gösterir.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-118">For example, if a thread sets the value of a thread-relative static field, enters another application domain, and then retrieves the value of the field, the value retrieved in the second application domain differs from the value in the first application domain.</span></span> <span data-ttu-id="bb5cf-119">İkinci uygulama etki alanında alan için yeni bir değer ayarlamak alanın değeri ilk uygulama etki alanındaki etkilemez.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-119">Setting a new value for the field in the second application domain does not affect the field's value in the first application domain.</span></span>  
  
 <span data-ttu-id="bb5cf-120">Benzer şekilde, bir iş parçacığı iki farklı uygulama etki alanları aynı adlandırılmış veri yuvaya aldığında, verileri ilk uygulama etki alanında ikinci uygulama etki alanındaki verilerin bağımsız kalır.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-120">Similarly, when a thread gets the same named data slot in two different application domains, the data in the first application domain remains independent of the data in the second application domain.</span></span>  
  
## <a name="thread-relative-static-fields"></a><span data-ttu-id="bb5cf-121">İş parçacığı göreli statik alanları</span><span class="sxs-lookup"><span data-stu-id="bb5cf-121">Thread-Relative Static Fields</span></span>  
 <span data-ttu-id="bb5cf-122">Bir veri parçasını her zaman bir iş parçacığı ve uygulama etki alanı birleşimi benzersiz olduğunu biliyorsanız, uygulama <xref:System.ThreadStaticAttribute> statik alanına öznitelik.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-122">If you know that a piece of data is always unique to a thread and application-domain combination, apply the <xref:System.ThreadStaticAttribute> attribute to the static field.</span></span> <span data-ttu-id="bb5cf-123">Herhangi bir statik alan kullandığınız gibi alanını kullanın.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-123">Use the field as you would use any other static field.</span></span> <span data-ttu-id="bb5cf-124">Bu alandaki veri, bunu kullanan her bir iş parçacığı için benzersizdir.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-124">The data in the field is unique to each thread that uses it.</span></span>  
  
 <span data-ttu-id="bb5cf-125">İş parçacığı göreli statik alanları veri yuvaları daha iyi performans sağlar ve derleme zamanı tür denetimi faydası vardır.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-125">Thread-relative static fields provide better performance than data slots and have the benefit of compile-time type checking.</span></span>  
  
 <span data-ttu-id="bb5cf-126">Herhangi bir sınıf oluşturucu kodu alan erişen ilk bağlamında ilk iş parçacığında çalışacağını unutmayın.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-126">Be aware that any class constructor code will run on the first thread in the first context that accesses the field.</span></span> <span data-ttu-id="bb5cf-127">İçin tüm diğer iş parçacıkları veya bağlamları aynı uygulama etki alanında, alanları başlatılacak `null` (`Nothing` Visual Basic'te) başvuru türleridir veya varsayılan değer türleri olmaları durumunda değerleri.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-127">In all other threads or contexts in the same application domain, the fields will be initialized to `null` (`Nothing` in Visual Basic) if they are reference types, or to their default values if they are value types.</span></span> <span data-ttu-id="bb5cf-128">Bu nedenle, iş parçacığı göreli statik alanları başlatmak için sınıf oluşturucuları güvenmemelisiniz.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-128">Therefore, you should not rely on class constructors to initialize thread-relative static fields.</span></span> <span data-ttu-id="bb5cf-129">Bunun yerine, iş parçacığı göreli statik alanları başlatma önlemek ve bunlar için başlatılır varsayar `null` (`Nothing`) veya varsayılan değerlerine.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-129">Instead, avoid initializing thread-relative static fields and assume that they are initialized to `null` (`Nothing`) or to their default values.</span></span>  
  
## <a name="data-slots"></a><span data-ttu-id="bb5cf-130">Veri yuvaları</span><span class="sxs-lookup"><span data-stu-id="bb5cf-130">Data Slots</span></span>

-   <span data-ttu-id="bb5cf-134">Bir adlandırılmış veri yuvası oluşturmak için kullanın <xref:System.Threading.Thread.AllocateNamedDataSlot%2A?displayProperty=nameWithType> veya <xref:System.Threading.Thread.GetNamedDataSlot%2A?displayProperty=nameWithType> yöntemi.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-134">To create a named data slot, use the <xref:System.Threading.Thread.AllocateNamedDataSlot%2A?displayProperty=nameWithType> or <xref:System.Threading.Thread.GetNamedDataSlot%2A?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="bb5cf-135">Mevcut bir yuva adlı bir başvuru almak için adının geçirmek <xref:System.Threading.Thread.GetNamedDataSlot%2A> yöntemi.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-135">To get a reference to an existing named slot, pass its name to the <xref:System.Threading.Thread.GetNamedDataSlot%2A> method.</span></span>  
  
-   <span data-ttu-id="bb5cf-136">Adlandırılmamış yuvası oluşturmak için kullanın <xref:System.Threading.Thread.AllocateDataSlot%2A?displayProperty=nameWithType> yöntemi.</span><span class="sxs-lookup"><span data-stu-id="bb5cf-136">To create an unnamed data slot, use the <xref:System.Threading.Thread.AllocateDataSlot%2A?displayProperty=nameWithType> method.</span></span>

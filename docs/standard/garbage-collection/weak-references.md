# <a name="weak-references"></a><span data-ttu-id="dabe6-102">Zayıf Başvurular</span><span class="sxs-lookup"><span data-stu-id="dabe6-102">Weak References</span></span>

 <span data-ttu-id="dabe6-109">Zayıf başvurular, çok miktarda bellek kullanır, ancak çöp toplama tarafından talep edilen, kolayca yeniden oluşturulabilir nesneler için kullanışlıdır.</span><span class="sxs-lookup"><span data-stu-id="dabe6-109">Weak references are useful for objects that use a lot of memory, but can be recreated easily if they are reclaimed by garbage collection.</span></span>  
  
 <span data-ttu-id="dabe6-110">Bir Windows Forms uygulaması ağaç görünümünde, kullanıcıya bir karmaşık hiyerarşik, tercih ettiğiniz seçenekleri görüntüler varsayalım.</span><span class="sxs-lookup"><span data-stu-id="dabe6-110">Suppose a tree view in a Windows Forms application displays a complex hierarchical choice of options to the user.</span></span> <span data-ttu-id="dabe6-111">Temel alınan veriler büyükse, ağaç bellekte tutulması, kullanıcı uygulamada başka bir şey ile söz konusu olduğunda verimsizdir.</span><span class="sxs-lookup"><span data-stu-id="dabe6-111">If the underlying data is large, keeping the tree in memory is inefficient when the user is involved with something else in the application.</span></span>  
  
 <span data-ttu-id="dabe6-112">Kullanıcı başka bir uygulamanın parçası için hemen geçtiğinde kullanabileceğiniz <xref:System.WeakReference> zayıf bir başvuru ağacı oluşturmak ve tüm güçlü atıflar yok etmek için sınıf.</span><span class="sxs-lookup"><span data-stu-id="dabe6-112">When the user switches away to another part of the application, you can use the <xref:System.WeakReference> class to create a weak reference to the tree and destroy all strong references.</span></span> <span data-ttu-id="dabe6-113">Kullanıcı ağaca geri dön geçirildiğinde, uygulamanın güçlü bir başvuru ağacı almayı dener ve, başarılı olursa, ağaç yeniden oluşturuluyor önler.</span><span class="sxs-lookup"><span data-stu-id="dabe6-113">When the user switches back to the tree, the application attempts to obtain a strong reference to the tree and, if successful, avoids reconstructing the tree.</span></span>  
  
 <span data-ttu-id="dabe6-114">Zayıf bir başvuru içeren bir nesne oluşturmak için oluşturduğunuz bir <xref:System.WeakReference> izlenmesi nesnesinin örneğini kullanarak.</span><span class="sxs-lookup"><span data-stu-id="dabe6-114">To establish a weak reference with an object, you create a <xref:System.WeakReference> using the instance of the object to be tracked.</span></span> <span data-ttu-id="dabe6-115">Ardından <xref:System.WeakReference.Target%2A> bu nesne ve özgün nesneye başvurusu kümesi özelliğini `null`.</span><span class="sxs-lookup"><span data-stu-id="dabe6-115">You then set the <xref:System.WeakReference.Target%2A> property to that object and set the original reference to the object to `null`.</span></span> <span data-ttu-id="dabe6-116">Kod örneği için bkz: <xref:System.WeakReference> Sınıf Kitaplığı'nda.</span><span class="sxs-lookup"><span data-stu-id="dabe6-116">For a code example, see <xref:System.WeakReference> in the class library.</span></span>  
  
## <a name="short-and-long-weak-references"></a><span data-ttu-id="dabe6-117">Kısa ve uzun zayıf başvurular</span><span class="sxs-lookup"><span data-stu-id="dabe6-117">Short and Long Weak References</span></span>  
 <span data-ttu-id="dabe6-118">Zayıf bir başvuru kısa veya uzun zayıf bir başvuru oluşturabilirsiniz:</span><span class="sxs-lookup"><span data-stu-id="dabe6-118">You can create a short weak reference or a long weak reference:</span></span>  
  
- <span data-ttu-id="dabe6-119">Kısa</span><span class="sxs-lookup"><span data-stu-id="dabe6-119">Short</span></span>  
  
     <span data-ttu-id="dabe6-120">Kısa bir zayıf başvuru hedefinin olur `null` zaman nesne iadesi çöp toplama tarafından.</span><span class="sxs-lookup"><span data-stu-id="dabe6-120">The target of a short weak reference becomes `null` when the object is reclaimed by garbage collection.</span></span> <span data-ttu-id="dabe6-121">Zayıf başvuru kendisi yönetilen bir nesnedir ve çöp toplama gibi herhangi bir yönetilen nesne tabidir.</span><span class="sxs-lookup"><span data-stu-id="dabe6-121">The weak reference is itself a managed object, and is subject to garbage collection just like any other managed object.</span></span>  <span data-ttu-id="dabe6-122">Parametresiz oluşturucusu kısa zayıf bir başvuru olduğundan <xref:System.WeakReference>.</span><span class="sxs-lookup"><span data-stu-id="dabe6-122">A short weak reference is the parameterless constructor for <xref:System.WeakReference>.</span></span>  
  
- <span data-ttu-id="dabe6-123">Uzun</span><span class="sxs-lookup"><span data-stu-id="dabe6-123">Long</span></span>  
  
     <span data-ttu-id="dabe6-124">Nesnenin sonra uzun zayıf bir başvuru korunur <xref:System.Object.Finalize%2A> yöntemi çağrılır.</span><span class="sxs-lookup"><span data-stu-id="dabe6-124">A long weak reference is retained after the object's <xref:System.Object.Finalize%2A> method has been called.</span></span> <span data-ttu-id="dabe6-125">Bu nesnenin oluşturulması sağlar, ancak nesnenin durumu beklenmedik kalır.</span><span class="sxs-lookup"><span data-stu-id="dabe6-125">This allows the object to be recreated, but the state of the object remains unpredictable.</span></span> <span data-ttu-id="dabe6-126">Uzun bir başvuru kullanılacağını belirtin `true` içinde <xref:System.WeakReference> Oluşturucusu.</span><span class="sxs-lookup"><span data-stu-id="dabe6-126">To use a long reference, specify `true` in the <xref:System.WeakReference> constructor.</span></span>  
  
     <span data-ttu-id="dabe6-127">Nesnenin türü yoksa bir <xref:System.Object.Finalize%2A> yöntemi, kısa zayıf başvuru işlevselliğini uygular ve hedef toplanana kadar sonra dilediğiniz zaman Sonlandırıcı oluşabilen çalıştırılan yalnızca zayıf başvuru geçerli değil.</span><span class="sxs-lookup"><span data-stu-id="dabe6-127">If the object's type does not have a <xref:System.Object.Finalize%2A> method, the short weak reference functionality applies and the weak reference is valid only until the target is collected, which can occur anytime after the finalizer is run.</span></span>

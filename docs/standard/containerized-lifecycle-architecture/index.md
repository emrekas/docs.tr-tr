# <a name="introduction-to-containers-and-docker"></a><span data-ttu-id="da281-103">Kapsayıcılar ve Docker'a giriş</span><span class="sxs-lookup"><span data-stu-id="da281-103">Introduction to containers and Docker</span></span>

<span data-ttu-id="da281-104">*Kapsayıcı içinde bir uygulama veya hizmeti ve bağımlılıklarını yapılandırmasıyla (dağıtım bildirim dosyaları soyutlanır) bir arada bir kapsayıcı görüntüsü paketlenmiş yazılım geliştirme için bir yaklaşımdır. Ardından bir birim olarak kapsayıcılı uygulamayı test etme ve konak işletim sistemi (OS) bir kapsayıcı görüntüsü örneği olarak dağıtabilirsiniz.*</span><span class="sxs-lookup"><span data-stu-id="da281-104">*Containerization is an approach to software development in which an application or service, its dependencies, and its configuration (abstracted as deployment manifest files) are packaged together as a container image. You then can test the containerized application as a unit and deploy it as a container image instance to the host operating system (OS).*</span></span>

<span data-ttu-id="da281-105">Yalnızca ürün sevk, train veya kargo içinde bağımsız olarak kamyon taşınmasını nakliye izin vermek gibi yazılım kapsayıcıları farklı kodu ve bağımlılıkları içerebilir, yazılım dağıtımının standart bir birim olarak davranır.</span><span class="sxs-lookup"><span data-stu-id="da281-105">Just as shipping containers allow goods to be transported by ship, train, or truck regardless of the cargo inside, software containers act as a standard unit of software deployment that can contain different code and dependencies.</span></span> <span data-ttu-id="da281-106">Bu şekilde yazılım kapsayıcılı hale getirmek, geliştiriciler ve BT uzmanları onları ortamlarında çok az kayıpla veya hiç değişiklik ile dağıtmak etkinleştirir.</span><span class="sxs-lookup"><span data-stu-id="da281-106">Containerizing software this way enables developers and IT professionals to deploy them across environments with little or no modification.</span></span>

<span data-ttu-id="da281-107">Kapsayıcılar ayrıca paylaşılan bir işletim sisteminde uygulamaları birbirinden yalıtın.</span><span class="sxs-lookup"><span data-stu-id="da281-107">Containers also isolate applications from each other on a shared OS.</span></span> <span data-ttu-id="da281-108">Kapsayıcılı uygulamaların, işletim sisteminde (Linux veya Windows) sırayla çalıştırılan bir kapsayıcı konağı üzerinde çalıştırın.</span><span class="sxs-lookup"><span data-stu-id="da281-108">Containerized applications run on top of a container host that in turn runs on the OS (Linux or Windows).</span></span> <span data-ttu-id="da281-109">Kapsayıcılar, bu nedenle bir çok daha küçük kaplama alanı sanal makine (VM) görüntülerini daha da içerir.</span><span class="sxs-lookup"><span data-stu-id="da281-109">Containers therefore have a much smaller footprint than virtual machine (VM) images.</span></span>

<span data-ttu-id="da281-110">Her kapsayıcı, tüm web uygulaması veya hizmeti, Şekil 1-1'de gösterildiği gibi çalıştırabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="da281-110">Each container can run a whole web application or a service, as shown in Figure 1-1.</span></span> <span data-ttu-id="da281-111">Bu örnekte, Docker ana bir kapsayıcı konağı ve App1 ve App2, Svc1 ve Svc2 kapsayıcılı uygulamalar veya hizmetler şunlardır.</span><span class="sxs-lookup"><span data-stu-id="da281-111">In this example, Docker host is a container host, and App1, App2, Svc1, and Svc2 are containerized applications or services.</span></span>

<span data-ttu-id="da281-115">Kapsayıcı türetilip bir diğer avantajı ölçeklenebilirlik özelliğidir.</span><span class="sxs-lookup"><span data-stu-id="da281-115">Another benefit you can derive from containerization is scalability.</span></span> <span data-ttu-id="da281-116">Hızlı bir şekilde yeni kapsayıcılar için kısa vadeli görevleri oluşturarak ölçeği genişletebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="da281-116">You can scale out quickly by creating new containers for short-term tasks.</span></span> <span data-ttu-id="da281-117">Bir uygulama açısından bakıldığında, örnekleme (bir kapsayıcı oluşturma) görüntü hizmeti veya web uygulaması gibi bir işlem örnekleme için benzerdir.</span><span class="sxs-lookup"><span data-stu-id="da281-117">From an application point of view, instantiating an image (creating a container) is similar to instantiating a process like a service or web app.</span></span> <span data-ttu-id="da281-118">Birden çok ana bilgisayar sunucuları arasında aynı görüntüsünün birden çok örneği çalıştırdığınızda güvenilirliği, ancak genellikle her bir kapsayıcı (farklı bir ana sunucu çalıştırmak için görüntü örnek) veya VM farklı hata etki alanlarında istersiniz.</span><span class="sxs-lookup"><span data-stu-id="da281-118">For reliability, however, when you run multiple instances of the same image across multiple host servers, you typically want each container (image instance) to run in a different host server or VM in different fault domains.</span></span>

<span data-ttu-id="da281-119">Kısacası, kapsayıcılar arasında tüm uygulama yaşam döngüsü iş akışı yalıtımı, taşınabilirliği, çevikliği, ölçeklenebilirlik ve denetim avantajlarını sunar.</span><span class="sxs-lookup"><span data-stu-id="da281-119">In short, containers offer the benefits of isolation, portability, agility, scalability, and control across the entire application lifecycle workflow.</span></span> <span data-ttu-id="da281-120">En önemli avantajı, geliştirme ve Ops sağlanan ortam yalıtım olur.</span><span class="sxs-lookup"><span data-stu-id="da281-120">The most important benefit is the environment isolation provided between Dev and Ops.</span></span>

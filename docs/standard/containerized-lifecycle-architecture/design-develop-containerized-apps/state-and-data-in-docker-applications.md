<span data-ttu-id="db873-108">Aşağıdaki çözümlerden Docker uygulamalarında kalıcı verileri yönetmek için kullanılır:</span><span class="sxs-lookup"><span data-stu-id="db873-108">The following solutions are used to manage persistent data in Docker applications:</span></span>

<span data-ttu-id="db873-109">Docker ana bilgisayarından olarak [Docker birimleri](https://docs.docker.com/engine/admin/volumes/):</span><span class="sxs-lookup"><span data-stu-id="db873-109">From the Docker host, as [Docker Volumes](https://docs.docker.com/engine/admin/volumes/):</span></span>

- <span data-ttu-id="db873-111">**Takar bağlama** erişim Docker işlemden kontrol edilemez ve bir kapsayıcı önemli işletim sistemi klasörleri erişebilecek şekilde bir güvenlik riski oluşturabilir, ana bilgisayar dosya sistemi herhangi bir klasörde eşleyebilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="db873-111">**Bind mounts** can map to any folder in the host filesystem, so access can't be controlled from a Docker process and can pose a security risk as a container could access sensitive OS folders.</span></span>

- <span data-ttu-id="db873-112">**tmpfs bağlar** yalnızca ana bilgisayarın belleğinde bulunur ve dosya sistemi için hiçbir zaman yazılır sanal klasörler gibi şunlardır.</span><span class="sxs-lookup"><span data-stu-id="db873-112">**tmpfs mounts** are like virtual folders that only exist in the host's memory and are never written to the filesystem.</span></span>

<span data-ttu-id="db873-113">Uzaktaki depolama biriminden:</span><span class="sxs-lookup"><span data-stu-id="db873-113">From remote storage:</span></span>

- <span data-ttu-id="db873-114">[Azure depolama](https://azure.microsoft.com/documentation/services/storage/) kapsayıcılar için iyi bir uzun vadeli Kalıcılık çözüm sağlayan coğrafi dağıtılabilir depolama sağlar.</span><span class="sxs-lookup"><span data-stu-id="db873-114">[Azure Storage](https://azure.microsoft.com/documentation/services/storage/) provides geo-distributable storage, providing a good long-term persistence solution for containers.</span></span>

- <span data-ttu-id="db873-115">Gibi uzak ilişkisel veritabanları [Azure SQL veritabanı](https://azure.microsoft.com/services/sql-database/), gibi NoSQL veritabanları [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), veya önbellek gibi hizmetler [Redis](https://redis.io/).</span><span class="sxs-lookup"><span data-stu-id="db873-115">Remote relational databases like [Azure SQL Database](https://azure.microsoft.com/services/sql-database/), NoSQL databases like [Azure Cosmos DB](https://docs.microsoft.com/azure/cosmos-db/introduction), or cache services like [Redis](https://redis.io/).</span></span>

<span data-ttu-id="db873-116">Docker kapsayıcısı'ndan:</span><span class="sxs-lookup"><span data-stu-id="db873-116">From the Docker container:</span></span>

- <span data-ttu-id="db873-122">Ancak, Docker birimler artık docker'da yerel verileri işlemek için tercih edilen yoludur.</span><span class="sxs-lookup"><span data-stu-id="db873-122">However, Docker Volumes is now the preferred way to handle local data in Docker.</span></span> <span data-ttu-id="db873-123">Kapsayıcılarında depolama hakkında daha fazla bilgiye ihtiyacınız varsa, kontrol [Docker depolama sürücülerini](https://docs.docker.com/engine/userguide/storagedriver/) ve [görüntüleri, kapsayıcılar ve depolama sürücülerini hakkında](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/).</span><span class="sxs-lookup"><span data-stu-id="db873-123">If you need more information about storage in containers, check on [Docker storage drivers](https://docs.docker.com/engine/userguide/storagedriver/) and [About images, containers, and storage drivers](https://docs.docker.com/engine/userguide/storagedriver/imagesandcontainers/).</span></span>

<span data-ttu-id="db873-124">Bu seçenekler hakkında ek ayrıntılar sağlar.</span><span class="sxs-lookup"><span data-stu-id="db873-124">The following provides additional detail about these options.</span></span>

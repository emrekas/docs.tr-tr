# <a name="managing-namespaces-in-an-xml-document"></a><span data-ttu-id="a1890-102">XML belgesinde ad alanlarını yönetme</span><span class="sxs-lookup"><span data-stu-id="a1890-102">Managing Namespaces in an XML Document</span></span>
<span data-ttu-id="a1890-103">XML ad alanları XML belgesinde öğe ve öznitelik adları, özel ve önceden tanımlanmış bir URI'leri ile ilişkilendirin.</span><span class="sxs-lookup"><span data-stu-id="a1890-103">XML namespaces associate element and attribute names in an XML document with custom and predefined URIs.</span></span> <span data-ttu-id="a1890-104">Bu ilişkileri oluşturmak için URI ad alanı ön eklerini tanımlayın ve bu ön ekler öğe ve öznitelik adları XML verisindeki nitelemek için kullanın.</span><span class="sxs-lookup"><span data-stu-id="a1890-104">To create these associations, you define prefixes for namespace URIs, and use those prefixes to qualify element and attribute names in XML data.</span></span> <span data-ttu-id="a1890-105">Ad alanları, öğe ve öznitelik adı çakışmalarını önlemek ve öğeleri ve öznitelikleri aynı ada sahip işlenmesini ve farklı şekilde doğrulanmış etkinleştirin.</span><span class="sxs-lookup"><span data-stu-id="a1890-105">Namespaces prevent element and attribute name collisions, and enable elements and attributes of the same name to be handled and validated differently.</span></span>

## <a name="declaring-namespaces"></a><span data-ttu-id="a1890-106">Ad alanlarını bildirme</span><span class="sxs-lookup"><span data-stu-id="a1890-106">Declaring namespaces</span></span>  
 <span data-ttu-id="a1890-107">Bir öğe üzerinde bir ad alanı bildirmek için kullanmanız `xmlns:` özniteliği:</span><span class="sxs-lookup"><span data-stu-id="a1890-107">To declare a namespace on an element, you use the `xmlns:` attribute:</span></span>  
  
 `xmlns:<name>=<"uri">`  
  
 <span data-ttu-id="a1890-108">Burada `<name>` ad alanı öneki ve `<"uri">` ad alanını tanımlayan URI.</span><span class="sxs-lookup"><span data-stu-id="a1890-108">where `<name>` is the namespace prefix and `<"uri">` is the URI that identifies the namespace.</span></span> <span data-ttu-id="a1890-109">Önek bildirdikten sonra öğeleri ve özniteliklerinin bir XML belgesi sınıflandırmak ve ad alanı URI ile ilişkilendirmek için kullanabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="a1890-109">After you declare the prefix, you can use it to qualify elements and attributes in an XML document and associate them with the namespace URI.</span></span> <span data-ttu-id="a1890-110">Ad alanı öneki belge kullanılmakta olduğundan uzunluğu kısa olmalıdır.</span><span class="sxs-lookup"><span data-stu-id="a1890-110">Because the namespace prefix is used throughout a document, it should be short in length.</span></span>  
  
 <span data-ttu-id="a1890-111">Bu örnek iki tanımlar `BOOK` öğeleri.</span><span class="sxs-lookup"><span data-stu-id="a1890-111">This example defines two `BOOK` elements.</span></span> <span data-ttu-id="a1890-112">Ön eke göre ilk öğeyi nitelenmiş `mybook`, ve ikinci öğe ön eke göre nitelenmiş `bb`.</span><span class="sxs-lookup"><span data-stu-id="a1890-112">The first element is qualified by the prefix, `mybook`, and the second element is qualified by the prefix, `bb`.</span></span> <span data-ttu-id="a1890-113">Her ön eki farklı ad alanı URI ile ilişkilendirilir:</span><span class="sxs-lookup"><span data-stu-id="a1890-113">Each prefix is associated with a different namespace URI:</span></span>  
  
```xml  
<mybook:BOOK xmlns:mybook="http://www.contoso.com/books.dtd">  
<bb:BOOK xmlns:bb="urn:blueyonderairlines">  
```  
  
 <span data-ttu-id="a1890-114">Bir öğenin belirli bir ad alanının bir parçası olduğunu belirtmek için ad alanı öneki ekleyin.</span><span class="sxs-lookup"><span data-stu-id="a1890-114">To signify that an element is a part of a particular namespace, add the namespace prefix to it.</span></span> <span data-ttu-id="a1890-115">Örneğin, bir `Author` öğenin ait olduğu `mybook` ad alanı, bildirilen olarak `<mybook:Author>`.</span><span class="sxs-lookup"><span data-stu-id="a1890-115">For example, if a `Author` element belongs to the `mybook` namespace, it is declared as `<mybook:Author>`.</span></span>

## <a name="declaration-scope"></a><span data-ttu-id="a1890-116">Bildirim kapsamı</span><span class="sxs-lookup"><span data-stu-id="a1890-116">Declaration scope</span></span>  
 <span data-ttu-id="a1890-117">Bir ad alanı içinde bildirilen öğe sonuna kadar bildirim noktasında etkilidir.</span><span class="sxs-lookup"><span data-stu-id="a1890-117">A namespace is effective from its point of declaration until the end of the element it was declared in.</span></span> <span data-ttu-id="a1890-118">Bu örnekte ad alanı tanımlı `BOOK` öğesi öğeler için geçerli değildir `BOOK` öğesi gibi `Publisher` öğesi:</span><span class="sxs-lookup"><span data-stu-id="a1890-118">In this example, the namespace defined in the `BOOK` element doesn't apply to elements outside the `BOOK` element, such as the `Publisher` element:</span></span>  
  
```xml  
<Author>Joe Smith</Author>  
<BOOK xmlns:book="http://www.contoso.com">  
    <title>My Wonderful Day</title>  
      <price>$3.95</price>  
</BOOK>  
<Publisher>  
    <Name>MSPress</Name>  
</Publisher>  
```  
  
 <span data-ttu-id="a1890-119">Bir ad alanı, kullanılabilir, ancak XML belgenin üst kısmında görünmesini yok önce bildirilmelidir.</span><span class="sxs-lookup"><span data-stu-id="a1890-119">A namespace must be declared before it can be used, but it doesn't have to appear at the top of the XML document.</span></span>  
  
 <span data-ttu-id="a1890-120">Bir XML belgesinde birden çok ad alanını kullandığınızda, bir ad alanı Temizleyicisi görünümlü bir belge oluşturmak için varsayılan ad alanı olarak tanımlayabilirsiniz.</span><span class="sxs-lookup"><span data-stu-id="a1890-120">When you use multiple namespaces in an XML document, you can define one namespace as the default namespace to create a cleaner looking document.</span></span> <span data-ttu-id="a1890-121">Varsayılan ad alanı, kök öğesi bildirilmiş ve belgedeki tüm nitelenmemiş öğeleri için geçerlidir.</span><span class="sxs-lookup"><span data-stu-id="a1890-121">The default namespace is declared in the root element and applies to all unqualified elements in the document.</span></span> <span data-ttu-id="a1890-122">Varsayılan ad alanlarını öznitelikler için yalnızca öğeler için geçerlidir.</span><span class="sxs-lookup"><span data-stu-id="a1890-122">Default namespaces apply to elements only, not to attributes.</span></span>  
  
 <span data-ttu-id="a1890-123">Varsayılan ad alanı kullanılacak önek ve virgül gelen öğede bildirimi atla:</span><span class="sxs-lookup"><span data-stu-id="a1890-123">To use the default namespace, omit the prefix and the colon from the declaration on the element:</span></span>  
  
```xml  
<BOOK xmlns="http://www.contoso.com/books.dtd">  
```
